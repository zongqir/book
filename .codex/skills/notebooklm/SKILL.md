# 用 NotebookLM 逐轮贴源精修

## §1 命中判断

满足任意一条就调用：

- 文章 evidence_status 是 `llm-draft`，需要和原书核对
- 用户说"帮我贴源""核对一下原文""和书对一下""帮我刷一轮"
- 需要确认"我写的到底对不对、漏了什么、哪里和原书不一致"
- 需要把模型推断升级成贴源结论

不调用：

- 手上没有原始书籍材料可以作为 source
- 只想靠模型先验补内容，不涉及和原书核对
- 只是改文字润色，不涉及事实核对

## §2 产出与元数据

这个 skill 不产出独立文稿，而是改善已有文稿。

改善后更新原文稿的元数据：

- `evidence_status`: `llm-draft` → `notebooklm-checked`
- `source_basis`: 补上"NotebookLM 贴源核对 N 轮"
- `updated_at`: 更新时间

如果用户要求保留 QA 补充页，额外输出 `.QA.md`，写法见 §3.6。

## §3 工作流

核心循环四步：**问 → 看 → 改 → 再问**

### 3.1 准备

确认 notebook 和 source 已就位：

```bash
notebooklm status
notebooklm source list --json
```

source 只放原始书籍，不放你的文章草稿。

### 3.2 一轮迭代

如果文章属于某个固定槽位（00-10），第一轮问题优先从那个槽位 skill 的"NotebookLM 提问约束"里取。那些问题按槽位特性设计，比通用三步问更精准。通用三步问适合第二轮及以后。

每轮做三件事，串行执行，后一步消费前一步的答案：

1. **问覆盖**：这版正文已经站住了什么、还缺什么、哪里可能不准确
2. **问深度**：哪些段落方向对但还虚、为什么不够扎实、还缺什么类型的信息
3. **问最值钱的几刀**：如果现在只改 1–3 处，最该补哪里、补什么、怎么压缩着写

看完答案后，只吸收真正有增量价值的内容，直接改正文。不攒 QA，不生成候选稿。

改完后拿新版正文重复这个循环。

默认用 `scripts/article_revision_loop.py`，一次调用完成上面三步。

### 3.3 轮次节奏

同一篇默认刷 3–4 轮：

| 轮次 | 重点 |
|------|------|
| 第一轮 | 补主干判断和事实缺口 |
| 第二轮 | 补误判、边界、证据 |
| 第三轮 | 补没说透的地方、最小动作、停手条件 |
| 第四轮 | 只补最后一两刀 |

### 3.4 什么时候停

满足任意一条就停：

- 下一轮答案只剩边角修补，不再改变正文的判断结构
- 连续两轮回答的增量信息高度重复
- 文章已经能回答它对应槽位的核心问题

不要因为"还能继续问"就继续问。增量小了就收。

### 3.5 怎么判断答案值不值得改进去

值得改：

- 能纠正正文里一个事实错误或高频误判
- 能把一个抽象判断落到具体证据
- 能补上正文没覆盖但原书确实讲了的重要内容
- 能解释一个读者很容易误解的边界

不值得改：

- 只是背景知识扩展，和文章主问题关系弱
- 换一种说法重复正文已有内容
- 太偏细节，写进去会把主线写散
- 不能转成一句有用判断或一段短说明

### 3.6 QA 模式（次要）

如果用户明确想让 NotebookLM 自己提问题、自己回答、再筛选哪些值得留，用 `scripts/article_qa_review.py`。

这个模式适合挖补充问题，不适合作为默认主流程。

如果要留 .QA.md 展示页：

- 只保留正文没展开、但确实值得补充的 3–6 个问答
- 写成可读问答，不写"候选""保留/暂不保留""建议落稿位置"
- 不与正文大段重复
- 每个问答都能帮读者多推进一步

## §4 终检清单

- [ ] source 只放了原始书籍，没把草稿上传为 source
- [ ] 每轮问完直接改了正文，没有先攒一批 QA 再统一处理
- [ ] 每轮用的是改过的新正文，不是拿旧稿重复打转
- [ ] 改进后 evidence_status 已从 `llm-draft` 更新为 `notebooklm-checked`
- [ ] source_basis 已补上贴源核对轮次信息
- [ ] 没有产出"候选稿腔"的展示文件
- [ ] 如果有 .QA.md，它是可读补充页，不是审稿记录
- [ ] .QA.md 没有出现"候选""筛选建议""仍值得继续追问"这类过程语言
- [ ] 增量小了就停了，没有为了"还能继续问"而继续刷
- [ ] 改写后的正文仍然符合它所属槽位 skill 的质量要求
- [ ] 改完的段落读起来像原文的一部分，不像一块补丁

## 硬规则

- 全局规则遵循仓库 AGENTS.md：不上传草稿为 source、过程材料不直接展示、只回写真增量
- 默认串行 ask，不并发追问
- 默认不产出过程性展示文件
- .QA.md 要避免"元解释腔"：少写"这一章真正……"，多直接回答判断、误判、边界和动作

## 失败处理

- 鉴权失败：`notebooklm auth check --test`，必要时 `notebooklm login`
- 没有当前 notebook：`notebooklm use <notebook_id>`
- 没有来源：先 `source add` 书籍材料
- prompt 被截断：检查是否绕过了 `ask_from_article.py`
- 连续两轮增量很小：停止追问，直接收稿
