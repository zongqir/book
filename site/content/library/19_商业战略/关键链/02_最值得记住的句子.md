{{< sentence >}}
项目延期的根本原因不是估算不准确，而是安全余量在错误的地方被浪费掉了。
{{< /sentence >}}

传统项目管理让每个任务都带安全余量，看起来更安全，实际上制造了浪费。任务执行者知道有余量就会拖延，余量变成了延期的借口而不是保障。

{{< sentence >}}
关键链不是关键路径，而是考虑资源约束后的最长路径。
{{< /sentence >}}

关键路径法假设资源无限，但现实中资源是有限的。同一个人不能同时做两件事，同一台设备不能同时运行两个任务。考虑资源约束后，项目的瓶颈往往不在原来的关键路径上。

{{< sentence >}}
项目缓冲放在项目末尾，资源缓冲放在非关键链任务汇入关键链的地方。
{{< /sentence >}}

缓冲不是平均分布在每个任务上，而是集中放置在关键位置。项目缓冲保护整个项目的交付时间，资源缓冲保护关键链不被非关键链任务的延期影响。

{{< sentence >}}
专注于缓冲消耗速度，而不是任务的绝对进度。
{{< /sentence >}}

传统项目管理关心每个任务是否按时完成，关键链管理关心缓冲被消耗的速度。缓冲消耗过快说明项目有风险，缓冲消耗正常说明项目健康，即使某些任务看起来延期了。

{{< sentence >}}
多任务切换是项目延期的最大杀手。
{{< /sentence >}}

让一个人同时负责多个项目的任务，看起来提高了资源利用率，实际上因为切换成本让所有项目都延期。人的注意力需要时间重新聚焦，工作状态需要时间重新建立。

{{< sentence >}}
学生综合症：如果有50%的概率提前完成，也有50%的概率延期完成，提前的时间不会被保留，延期的时间会被累积。
{{< /sentence >}}

这是项目延期的统计学根源。每个任务的提前和延期在概率上是对称的，但在项目层面，提前不能被后续任务利用，延期会被完全传递。结果是项目只能延期，不能提前。

这些句子揭示了项目管理的反常识真相：看起来更安全的做法往往更危险，看起来提高效率的做法往往降低效率。掌握这些判断，才能真正理解为什么传统项目管理方法失效，关键链方法如何解决问题。
