# 十二句提醒，把经验、规则和反馈接成系统

## 经历不会自动沉淀

{{< sentence >}}
吃过亏不自动等于学会了，只有留下规则，亏才不会白吃。
{{< /sentence >}}

这句应该放在最前面，因为它直接拆掉“经历本身会教育人”的幻觉。

{{< sentence >}}
痛苦不是资产，反思后的规则才是。
{{< /sentence >}}

上一句强调结果，这一句强调中间缺的动作。痛苦和成长之间隔着一整套处理流程。

{{< sentence >}}
反思如果只停在感想层，下次遇到类似场景，旧情绪还会先回来。
{{< /sentence >}}

这句补的是为什么很多人明明复盘过，还是会重复犯错。

## 原则必须被写下来

{{< sentence >}}
原则要外化到系统里，不能只放在记忆和感觉里。
{{< /sentence >}}

这句是《原则》和一般格言书的分水岭。

{{< sentence >}}
一条不能指导下一次决策的原则，只是表态，不是原则。
{{< /sentence >}}

这句用来筛掉那些看着像原则、其实只有态度没有操作价值的话。

{{< sentence >}}
不写下来的原则，遇到压力时最先蒸发。
{{< /sentence >}}

这句特别适合高压决策场景。越到紧张的时候，人越容易退回感觉和习惯。

## 组织决策不能只靠感觉

{{< sentence >}}
在专业判断上，意见该不该等重，本身也是决策设计的一部分。
{{< /sentence >}}

可信度加权是这本书组织版最有特点的一层。

{{< sentence >}}
把所有意见天然放成平票，很多时候不是公平，而是偷懒。
{{< /sentence >}}

这句把上一句推得更直白。它指向的是机制设计，而不是性格好恶。

{{< sentence >}}
好的系统不是不允许分歧，而是让分歧能更快暴露到真正有判断力的人面前。
{{< /sentence >}}

这句补了一个组织使用场景：可信度加权不是压制讨论，而是加快决策质量提升。

## 系统不能替代现实

{{< sentence >}}
规则可以帮你少犯低级错，但它不能替你观察现实变化。
{{< /sentence >}}

这句是整本书最该和“系统化”同时记住的刹车。

{{< sentence >}}
把原则写死，比没有原则更危险。
{{< /sentence >}}

没有原则会乱，有了僵死原则会假装不乱。后者更隐蔽。

{{< sentence >}}
真正成熟的系统，允许自己被现实打脸后更新。
{{< /sentence >}}

这句适合当总收束。原则真正成熟，不在稳定不变，而在稳定地吸收反馈。
