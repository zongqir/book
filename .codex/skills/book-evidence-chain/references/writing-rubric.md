# 写作标尺

## 目标

读完后，读者能复述：先发生什么、为什么推到下一步、最后得到什么结论、结论的边界在哪里。

重点是把推理过程讲清楚，不是给观点打分，也不是堆证据。

## 推荐做法

- 每条链都写出"前提 → 中间推理 → 结论边界"
- 每一步都回答"为什么会到下一步"
- 至少有一个日常场景串联整条链
- H2 用问题句或推理判断
- 术语后面立刻跟白话解释
- 证据连接到具体推理步骤，不单独堆放
- 先让人看懂怎么来的，再谈是不是认同

## 不要这样写

- 标题写"论证链"或"子论证 1/2/3"
- 通篇"作者认为""书中指出"
- 只给结论不给推理过程
- 跳过关键中间环节
- 例子很多但说不清它在证明哪一步
- 术语压过解释
- 把整篇写成证据评级表或观点总结
- 文章自我指涉（"这页只回答一个问题"）
- 生造复合词
- 先抽象概括再具体展开同一件事

## 门禁

命中任一条，默认不过稿：

- 读完一段仍然说不清“为什么会到下一步”
- 段首还在评价这条推理有多重要，不是在写推理本身
- 例子很多，但读者不知道它们在支撑哪一步
- 标题和小标题拿掉书名后仍像模板，说明还没落到具体推理

## 自检

- 不看术语，读者还能复述推理链吗
- 每一步都写了为什么会到下一步吗
- 例子是在服务推理链，不是在装饰页面吗
- 结论边界说清了吗
- 这篇更像推理过程复原，还是观点总结
- 读起来舒服吗
