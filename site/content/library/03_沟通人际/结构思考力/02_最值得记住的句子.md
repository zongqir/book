# 十句提醒，把表达从堆材料改成带顺序的理解路径

## 接收顺序先于修辞

{{< sentence no="1" quote="对方能不能顺着走下去，比你讲得漂不漂亮更先决定表达效果。" >}}
{{< /sentence >}}

这句是总纲。表达真正先输的地方，常常不是文采，而是接收顺序。

{{< sentence no="2" quote="表达一乱，很多时候不是内容少，而是内容同时挤出来了。" >}}
{{< /sentence >}}

这句特别适合汇报前自检。背景、观点、例子、建议一股脑出场，是最常见的结构事故。

## 纵向展开

{{< sentence no="3" quote="往下展开时，要让下一层像在回答上一层，而不是平行堆更多信息。" >}}
{{< /sentence >}}

只要回答关系断了，结构再工整，读起来也还是散。

{{< sentence no="4" quote="层级不是装饰，它决定读者会不会觉得你一直在往前推进。" >}}
{{< /sentence >}}

这句补的是体感层。很多材料单页没问题，连起来没推进感，问题就出在层级关系没有持续回答。

## 横向分类

{{< sentence no="5" quote="横向展开最怕的，不是少讲一点，而是把不同维度的东西摆成一排。" >}}
{{< /sentence >}}

现象、原因、建议混成一组，是工作表达里最高频的错法之一。

{{< sentence no="6" quote="一组内容只能按同一把尺子并列。" >}}
{{< /sentence >}}

这句适合拿来在列三点、四点时当硬检查。

{{< sentence no="7" quote="归类不是把材料收整齐，而是先决定这组内容共同在回答什么。" >}}
{{< /sentence >}}

它比“分类整理”更进一步，直接要求并列关系服务于一个共同问题。

## 开场和模板

{{< sentence no="8" quote="序言要先让对方知道你在回答什么问题，再决定他愿不愿意往下听。" >}}
{{< /sentence >}}

这句把开场从礼貌动作拉回入口控制。

{{< sentence no="9" quote="会套框架的人很多，能让框架服务内容的人很少。" >}}
{{< /sentence >}}

这是这本书很需要保留的一句。因为结构化表达最容易退化成模板腔。

{{< sentence no="10" quote="好结构最大的价值，是替对方省掉那次脑内重组。" >}}
{{< /sentence >}}

最后这句最适合当总检标准。只要对方还得自己再整理一遍，你的结构就还没真正站稳。
