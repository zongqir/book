# USE、RED、TSA 不是三套方法，是三种收窄角度

Gregg 在书中给了一组方法，适用于性能分析的不同阶段和不同颗粒度。它们之间有分工、有先后、有互补关系。

## USE 方法

**解决什么**：快速扫描所有资源，找出哪个资源域最可能存在瓶颈。

**步骤**：对每种资源（CPU、内存、磁盘、网络接口、总线……），依次检查：

- Utilization（利用率）：忙碌程度
- Saturation（饱和度）：排队程度
- Errors（错误）：失败次数

**前提**：你能列出系统涉及的资源清单。遗漏一种资源就会留下盲区。

**强项**：防遗漏。5 分钟内覆盖主要资源域，避免一上来就钻进某个局部。

**弱项**：USE 只做资源级扫描。它不回答「为什么这个资源忙」。定位到域之后，需要切到 drill-down 或延迟分析继续深入。

## RED 方法

**解决什么**：从服务视角快速判断健康状态。

**步骤**：对每个服务，检查：

- Rate（请求速率）
- Errors（错误率）
- Duration（响应时间）

**和 USE 的关系**：USE 看资源，RED 看服务。两者互补。一个告诉你底层发生了什么，一个告诉你用户感受到了什么。

排障时常见的组合：先用 RED 确认哪个服务出了问题，再用 USE 找到是哪个资源导致的。

## TSA（线程状态分析）

**解决什么**：延迟升高但资源指标不明显时，定位线程的时间去向。

**步骤**：把线程时间分解为：

- on-CPU（执行中）
- runnable（等待调度）
- sleeping（等待 I/O、锁、定时器）
- 其他状态（不可中断等待等）

**前提**：需要内核级观测工具（perf、BPF）。纯应用层指标做不到这个粒度。

**使用时机**：USE 扫完没有明显饱和资源，但请求延迟确实在升。问题往往在等待和调度层面，TSA 直接切进去。

## drill-down 分析

**解决什么**：在某个资源域或子系统内逐层深入，从统计量到分布到事件。

**步骤**：

1. 计数器和统计量（mpstat、vmstat、iostat）
2. 分布和直方图（biolatency、runqlat）
3. 事件级追踪（perf trace、BPF 程序）

逐层递进。上一层锁定方向，下一层提供细节。

**关键约束**：每一层都有观测成本。计数器几乎免费，profile 有采样开销，trace 有运行时开销。证据成本顺序要和问题收窄顺序匹配。

## 延迟分析

**解决什么**：把请求或操作的时间拆开，找到延迟的主要贡献者。

**核心技术**：

- on-CPU / off-CPU 分解
- 延迟直方图（看分布形态，识别双峰、长尾）
- 分层延迟（应用层、系统调用层、内核层、设备层）

**和其他方法的关系**：USE 和 RED 指出「哪里有问题」，延迟分析回答「时间花在了哪里」。drill-down 提供逐层工具，延迟分析提供拆解框架。

## 工作负载特征化

**解决什么**：在排障之前，先理解正常状态下的工作负载长什么样。

**关注项**：

- 请求类型和比例
- 请求大小分布
- 到达模式（稳态、突发、周期）
- 并发度

**用途**：baseline 建立、容量规划、异常检测。不知道「正常是什么样」，就无法判断「现在是不是异常」。

## 方法之间的关系

```
告警 / 问题报告
     │
     ▼
  RED（服务健康）
     │
     ▼
  USE（资源扫描）──→ 饱和资源
     │                   │
     ▼                   ▼
  TSA（线程状态）   drill-down（逐层深入）
     │                   │
     ▼                   ▼
  延迟分析 ◄─────────────┘
     │
     ▼
  根因定位 → 优化 → before/after 验证
```

USE 做广度扫描，drill-down 做深度挖掘，TSA 和延迟分析专攻时间去向，RED 保持用户视角。组合使用时，排障效率显著高于单方法。

## 第二版的方法增量

第二版把整套方法扩展到了两个新领域：

1. **BPF 可编程观测**。BPF 让 drill-down 的第三层（事件级追踪）变得更安全、更灵活。以前需要重启或插内核模块的场景，BPF 在生产环境可以直接做。

2. **可见性受限环境**。云、容器、托管服务——底层数据拿不全。Gregg 引入了一组替代证据方法：对照分析（实例间、AZ 间、版本间对比）、边界指标（cgroup 配额、steal time）、统计推断。方法论骨架不变，但证据来源从完整视角扩展到了残缺视角。
