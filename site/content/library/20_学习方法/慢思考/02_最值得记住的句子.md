## 快模式的特征与风险

{{< sentence >}}
自动化反应省掉的不是时间，是检查。
{{< /sentence >}}

快模式给答案的速度快到你来不及质疑。走进会议室，听了三分钟汇报，心里已经有结论了。这个结论可能是对的——但你没检查过它为什么对，也没检查过它是不是真的对。省掉检查的代价不在每一次，在某一次。

{{< sentence >}}
当你觉得答案"显而易见"时，最值得问的是：是真的显而易见，还是你的大脑替你跳过了几步？
{{< /sentence >}}

"显而易见"是快模式最擅长制造的幻觉。越觉得答案显然，越应该暂停。不是因为答案一定错，而是因为"显然"这个感觉本身就在阻止你检查。

## 从快模式到慢模式的切换

{{< sentence >}}
慢思考的起点不是"想更久"，而是"先停下来问自己在想什么"。
{{< /sentence >}}

停下来不是为了推翻快模式的结论。多数时候快模式的答案检查完之后依然成立。停下来是为了把"无意识的反应"变成"有意识的选择"。

{{< sentence >}}
在判断和行动之间插入一个问题，就足以改变决策质量。
{{< /sentence >}}

那个问题可以非常简单："我现在要做的这件事，三个月后回看会后悔吗？"一个问题不能保证正确，但它能打断自动化反应链。

## 什么值得慢，什么不值得

{{< sentence >}}
不可逆的决策值得慢下来，可逆的决策不值得——难点在于多数人分不清哪些是不可逆的。
{{< /sentence >}}

开除一个员工是不可逆的。买一本书是可逆的。签一份三年合同是不可逆的。选一家餐厅吃饭是可逆的。慢思考的资源有限，花在可逆决策上是浪费。

{{< sentence >}}
用于检查的时间应该和决策的后果成正比，而不是和你的焦虑成正比。
{{< /sentence >}}

焦虑会让小事也变得看起来"必须慎重"。区分焦虑驱动的犹豫和必要的审慎：后者有具体的检查项，前者只有反复的担心。

## 慢思考的产出标准

{{< sentence >}}
如果慢下来之后得出的结论和快反应一样，但你能说出检查了什么、排除了什么——这就不是浪费时间。
{{< /sentence >}}

慢思考的价值不在于翻转结论，在于让结论变得可追溯。快模式给的结论你说不出理由，慢模式给的结论你说得出理由。理由本身就是资产——下次遇到类似情况可以复用。

| 卡住的时刻 | 调用哪句 | 做什么 |
|---|---|---|
| 觉得答案很明显 | "显而易见"那句 | 暂停三秒，问自己跳过了哪几步 |
| 不确定该不该慎重 | 不可逆那句 | 判断这个决策是否难以逆转 |
| 想了很久但没结论 | 停下来问在想什么 | 区分"在检查"还是"在焦虑" |
| 做完决策心里不踏实 | 可追溯那句 | 回看自己能不能说出检查了什么 |
