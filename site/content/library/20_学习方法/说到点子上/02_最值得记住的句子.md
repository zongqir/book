# 说到点子上：最值得记住的句子

## 开口之前的提醒

{{< sentence >}}说不清楚，大多数时候不是表达的问题，是思考的问题。{{< /sentence >}}

整本书最底层的判断。拿它校准一下自己的归因习惯——下次觉得"我不会说话"的时候，先问自己是不是还没想清楚要说什么。

{{< sentence >}}如果你自己说不出核心观点，听的人更说不出来。{{< /sentence >}}

这条拿来做写邮件之前的检查最好用。写完初稿，把正文盖住，问自己"一句话概括我要说什么"。概括不出来，说明还没想透。

{{< sentence >}}对方的注意力只有开头几句话的窗口。错过了，后面说什么都在打折。{{< /sentence >}}

注意力窗口不是"好的开场白"那种修辞问题。它的意思更硬：如果前三句没让对方明白"这件事跟我有什么关系"，后面你花再多时间解释也是在补救。

## 意图与结构的提醒

{{< sentence >}}你这次开口，到底是想让对方知道、同意、还是做一件事？{{< /sentence >}}

三种意图对应完全不同的话术结构。很多沟通混乱就是因为自己都没分清楚——一会儿在通报信息，一会儿在请求决策，听的人被拉来拉去。

{{< sentence >}}先说结论，再说原因。不要让对方猜你的终点在哪里。{{< /sentence >}}

违反直觉的简单规则。大多数人习惯按自己的思考顺序说话，但听的人不需要跟着你走一遍推导过程。先把终点亮出来，他们才知道后面的信息该怎么归类。

{{< sentence >}}一段话只说一件事。你以为的"顺便提一下"，在对方耳朵里是"突然岔到另一个话题"。{{< /sentence >}}

会议效率低、邮件被追问，很多时候就栽在"顺便"两个字上。每多塞一件事，对方对主线的记忆就被稀释一次。

## 检验与校准的提醒

{{< sentence >}}你说完以后，让对方用自己的话复述一遍。他复述不出来的部分，就是你没说清楚的部分。{{< /sentence >}}

最朴素的闭环检验方法。不需要任何工具，任何场合都能用。如果你不敢让对方复述，通常说明你自己心里也没底。

{{< sentence >}}简洁不是少说话，是每句话都在推进对方的理解。{{< /sentence >}}

"简洁"经常被误解成"说得短"。简洁的本质是去掉不推进理解的部分——重复的铺垫、不必要的背景、和主题无关的例子。剩下的每一句都在帮对方往前走一步。
