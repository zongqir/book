## 关于定位问题比解决问题更重要

{{< sentence >}}
改不好一个故事，通常不是改的方法不对，是在改错误的层面。
{{< /sentence >}}

用细节层面的修改去处理结构层面的问题，是修稿中最隐蔽的浪费。先判断问题属于哪个层面，再选工具。

{{< sentence >}}
读者说"无聊"，百分之八十的时候是节奏问题，不是内容问题。
{{< /sentence >}}

收到"读不下去"的反馈时，先检查场景切换频率和信息释放密度，再考虑换情节。无聊的根源常常是节奏失控，不是故事不够精彩。

## 关于读者信任的技术基础

{{< sentence >}}
读者不需要喜欢你的角色，但必须相信你的角色会做出那些事。
{{< /sentence >}}

人物动机的一致性是读者信任链的基础。角色突然做出不合动机的行为，信任链断裂，故事就失去了代入感。

{{< sentence >}}
悬念的本质不是藏信息，是让读者知道有信息被藏了。
{{< /sentence >}}

纠正"让读者什么都不知道就是悬念"这个误解。完全不知道会让人困惑而不是好奇。悬念需要读者意识到"有东西还没说"——这种意识才是继续翻页的动力。

## 关于视角与叙事控制

{{< sentence >}}
视角跳了一次，读者的代入感就从那个裂缝里漏走了。
{{< /sentence >}}

视角一致性是所有沉浸感的前提。每一次无意识的视角滑动，读者都必须重新建立和故事的连接。最危险的是作者自己没注意到的跳跃。

{{< sentence >}}
好结尾不是故事的终点，是读者理解发生了什么的起点。
{{< /sentence >}}

结尾的功能不是"把故事收住"，是给读者一个重新审视整个故事的新角度。结尾之后，读者回头想，整个故事才真正在脑子里完成。
