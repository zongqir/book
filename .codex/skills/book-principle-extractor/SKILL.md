# 原则提取稿

原则是判断准则：帮读者在面对这类问题时知道优先守住什么、不过早做什么、在方法细节变了以后哪些方向感仍然成立。

提取原则的目标不是把全书缩写一遍，而是从一整本书里收束出 3 到 7 条能帮读者不读偏、不用偏的持久判断方向。

四层内容各有归属：

- **观点**：作者怎么看世界或问题
- **原则**：面对这类问题时该守住的判断方向或约束
- **方法**：针对某类问题的可复用处理路径
- **步骤**：具体执行的动作序列

原则层只收第二类。观点留给作者介绍，方法留给 05，步骤留给 04。

## §1 命中判断

当任务涉及以下意图时命中：

- 用户要求提取一本书的核心原则
- 用户想把观点、原则、方法、步骤区分清楚
- 用户需要一组"不读偏"的阅读罗盘
- 用户要这本书最值得继续带着走的判断准则
- 当前材料把原则和方法混在一起，需要拆层

不命中的任务：

- 写方法结构和方法流：去 `book-methodology`
- 写具体执行步骤：去 `book-action-guide`
- 写全书摘要：不属于原则提取
- 做单章复习卡或章节记忆卡：去 `chapter-memory-cards`

这里的"原则卡"是**整本书层面**的判断准则卡，不是章节复习卡。

## §2 元数据

通用必填字段遵循 `AGENTS.md` base schema。扩展字段：

- `principle_count`：提取的原则总数
- `core_principle_count`：核心原则数量（通常 1–2 条）

`title` 从内容来，不写"核心原则"。`summary` 说清楚提取了几条什么层次的原则。

## §3 工作流

### NotebookLM 提问约束

当你不确定原书里到底有没有稳定原则层，或担心会把观点、方法、步骤混写成原则时，先按这个 skill 的标准去问 NotebookLM。

- 先问成立性：原书里有没有跨章节、跨场景稳定重复的判断准则，足以提成原则？
- 再问筛选性：哪些内容是真原则，哪些只是观点、方法或一次性技巧？
- 最后问退出性：如果原则层很薄，是原书本来就弱原则，还是当前 source 还没覆盖反复强调的部分？

优先问这种问题：

- 哪些判断在方法细节忘了以后，仍然能帮读者不读偏、不用偏？
- 哪 1 到 2 条最适合作为核心原则，哪些更适合作为支撑原则？
- 哪些候选表达看似像原则，其实只是方法步骤或口号？

不要问：`这本书的核心观点有哪些？`

通用判定方法见 [../references/notebooklm-slot-gating.md](../references/notebooklm-slot-gating.md)。

### 3.1 找候选原则

在全书中找这些信号：

- 作者反复强调"不要直接……而要先……"
- 作者反复调整优先级"真正重要的不是……而是……"
- 同一个默认错误被反复批评
- 同一个判断方向跨多个章节或案例反复出现
- 某些约束解释了后面的方法为什么是这样设计的

先多收，再收束。

### 3.2 过滤非原则

一个快速判断：如果方法细节全忘了，这句话还能帮你不读偏吗？能的话它大概率是原则。

去掉或降级这些内容：

- 世界观陈述（观点层）
- 章节标题改写成的建议
- 伪装成原则的方法
- 鼓励性口号
- 执行步骤
- 不能迁移到相邻场景的一次性技巧

### 3.3 只保留持久原则

保留标准：跨章节、跨场景稳定、能约束判断、能生成或解释下方的方法、在执行细节忘了以后仍然有用。

通常保留 3 到 7 条。不要为了数量硬凑。

### 3.4 区分核心与支撑

不要把所有原则平铺在同一层。

- 找出 1 到 2 条最能组织全书的核心原则
- 其余标为支撑原则——它们澄清、约束或保护核心原则

如果这本书确实不支持层级关系，直接说明，不硬造。

### 3.5 落稿

每条原则用 `principle` shortcode 写成卡片：

```
{{</* principle no="1" type="core" statement="原则一句话" */>}}
约束什么误判、优先守住什么、常见误读、边界——用自然段落写，不用标注字段名。
{{</* /principle */>}}
```

参数说明：

- `no`：编号，按价值从高到低
- `type`：`core`（核心原则，通常 1–2 条）或 `support`（支撑原则）
- `statement`：原则本身，一句话

卡片 body 是自然段落，不是表单。写清约束什么误判、常见误读、边界就够了，不需要逐项加粗标注。

**⚠️ 卡片格式是固定的，但文章结构不是。**

引言段怎么写、原则之间怎么过渡、要不要加收尾段，都由这本书的原则特点决定。写完后检查：如果你的引言段和同分类下另一本书的 01 文稿开头一样，必须重写引言。

正文结构：引言段 → 核心原则卡片 → 支撑原则卡片。不需要 H2 分隔，卡片的 type 自动区分层级。

引言段直接写这些原则在约束什么误判，不用 `这本书真正能穿出……`、`作者反复纠正的是……` 这种先给内容搭台的句子。

证据边界在 frontmatter 的 evidence_status 里统一标注，正文不逐条标注。

## §4 终检清单

### 反同质化（必查）

- [ ] H2 标题序列和同分类下已有书的同槽位不同
- [ ] 开头第一句和同分类下已有书的开头不同
- [ ] 全篇对比句式（"不是X而是Y""与其A不如B"）合计不超过 2 次
- [ ] 没有使用可以套到任何书上的万能句式作为标题或开头


### 原则质量

- [ ] 每条原则都能约束判断，不只是描述现象
- [ ] 每条原则跨章节成立，不是局部技巧
- [ ] 每条原则在方法细节忘了以后仍然有用
- [ ] 核心原则和支撑原则有明确层级
- [ ] 原则总数在 3–7 条之间

### 四层分清

- [ ] 没有把观点当成原则
- [ ] 没有把方法当成原则
- [ ] 没有把步骤当成原则
- [ ] 没有把口号当成原则
- [ ] 容易跟原则混淆的内容（方法、步骤等）已在相关原则的"常见误读"中提及

### 表达与证据

- [ ] 标题是内容判断，不是"核心原则"
- [ ] 每条原则使用 principle shortcode 卡片格式
- [ ] 不需要 H2 分组（核心/支撑由卡片 type 区分）
- [ ] evidence_status 在 frontmatter 标清，正文没有逐条标注
- [ ] 无评价句或重要性定级句（"最底层的""一切都建立在它上面"）
- [ ] 无泛指代词反复
- [ ] 读起来舒服

## 输出规则

- 正文 80–150 行
- frontmatter 通用字段 + 扩展字段齐全
- 原则条数宁少勿多，不为数量硬凑

## 校准与例外

- 原则层极薄的书（主要是案例集或操作手册），可以压缩到 3 条以下，并明确标注"原则性有限"
- 原则层极厚的书（如哲学著作），原则数量上限可放宽到 10 条
- 如果和 05 方法论总结的边界模糊，原则层只回答"该守住什么方向"，方法层回答"怎么组织和执行"
