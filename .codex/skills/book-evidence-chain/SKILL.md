# 论证链稿

08 回答的不是"这本书讲了什么"，也不是"我同不同意"，而是：核心结论到底怎么从前提一步步推出来的。

一篇好的论证链让读者能复述推理过程，而不只是记住结论。先让人看懂怎么来的，再谈是不是认同。

## §1 命中判断

当用户要求"写论证链""拆推理过程""分析这本书的逻辑""补齐 08"时命中。

如果用户要求写的内容更像"方法结构和组织逻辑"，优先转给 `book-methodology`。如果更像"具体怎么做"，优先转给 `book-action-guide`。如果更像"适用边界"，优先转给 `book-boundaries-and-failure-modes`。

## §2 元数据

通用必填字段遵循 `AGENTS.md` base schema。

扩展字段：

- `evidence_basis`：论证链复原的依据来源

`title` 写推理判断的内容，不写"论证链"。`summary` 只写推理主线，不写推荐语。

## §3 工作流

### NotebookLM 提问约束

当你不确定 `08` 能不能成立，或怀疑自己只是在做摘要而不是复原推理时，先按这个 skill 的标准设计 NotebookLM 问题。

- 先问成立性：原书里有没有足够稳定的前提、推理跳步和结论，能复原成一条论证链？
- 再问筛选性：哪 1 到 3 条主链最能支撑全书，而不是只是若干局部观点？
- 最后问退出性：如果论证链不稳，是原书本来就弱推理，还是 source 还没覆盖关键论证段？

优先问这种问题：

- 哪些结论最值得复原它们的推理过程？
- 哪些推理步骤是作者明说的，哪些只能做归纳或重构？
- 哪些地方存在明显跳步，必须补中间环节才能成稿？

不要问：`这本书的逻辑是什么？`

通用判定方法见 [../references/notebooklm-slot-gating.md](../references/notebooklm-slot-gating.md)。

### 3.1 找到推理主线

写之前先回答：这本书最核心的结论是什么？它是怎么从前提推到这个结论的？

区分总结论、关键子结论和支撑性结论。默认优先复原最能支撑全书主张的 1 到 3 条主链，不追求把所有小链条平铺。

一条主链只回答一个核心问题，围绕一个主结论展开。如果中间已经切换了问题定义或结论目标，拆成子链。

### 3.2 复原每条链的推理过程

每条链至少讲清三件事：

- 前提是什么
- 中间怎么推到下一步（至少 3 句，每句回答"为什么会到下一步"）
- 结论的边界在哪里

证据不要单独堆放。写清它支撑的是哪一步推理、补的是哪个跳步、限制了哪种结论强度。

至少用一个日常场景把整条链串起来。例子的作用是帮读者看懂"为什么会到下一步"，不是让页面更生动。

### 3.3 降术语

优先用白话词。保留必要术语时，后面立刻跟一句白话解释。

白话化是降低理解门槛，不是改写逻辑关系。凡是会改变前提、因果方向或结论强度的简化，都不要做。

正文不要反复出现"作者认为""书中指出""从书内依据看""理论上"——改成直接陈述句。

### 3.4 落稿

H2 标题用问题句或推理判断，不用"子论证 1/2/3"。

不要在正文写这些：

- `论证链`
- `子论证 1 / 子论证 2`
- `这页在复原哪条推理`
- `为了方便阅读，这页每条都按同样顺序写`
- `关键推理步骤`

这些是你的写作检查框，不是读者要看的内容。

**⚠️ 以下结构建议是默认参考，不是固定模板。**

每本书的论证方式不同，链条的组织方式也应该不同。归纳型论证、演绎型论证、历史叙事型论证，适合不同的展开方式。写完后检查：如果你的结构和同分类下另一本书的论证链一样，必须重新组织。

正文结构：开头用一两句话说清楚全书的核心推理方向 → 按主链逐条展开 → 可以补一段"最容易误读的地方"收尾，但不要写成独立的"误读清单"节。

### 3.5 说透检查

完稿后逐段检查：

- 这段是否真的写出了"前提 → 推理 → 结论边界"
- 读者是否能复述"为什么会推到下一步"
- 是否还存在明显跳步

只要有一段"没说透"，就继续补，不直接结束。

### 3.6 证据标注

论证链天然带重构成分。标清三层：

- 作者明说的推理：直接按作者强度写
- 基于全书结构的归纳：标明归纳
- 为帮助理解补的中间环节：标明重构

## §4 终检清单

### 反同质化（必查）

- [ ] H2 标题序列和同分类下已有书的同槽位不同
- [ ] 开头第一句和同分类下已有书的开头不同
- [ ] 全篇对比句式（"不是X而是Y""与其A不如B"）合计不超过 2 次
- [ ] 链条的组织方式和同分类下已有书的论证链不同


### 推理完整性

- [ ] 有明确的总链方向（一两句话能说清全书推理主线）
- [ ] 主链不超过 3 条，不是小链条平铺
- [ ] 每条链都有：前提 → 中间推理（≥3 句）→ 结论边界
- [ ] 至少有一个日常场景串联整条链
- [ ] 不存在明显跳步（每一步都回答了"为什么会到下一步"）
- [ ] 证据连接到具体推理步骤，不是单独堆放

### 可懂性

- [ ] 不懂术语也能跟上
- [ ] 必要术语后面有白话解释
- [ ] 无"作者认为""书中指出"反复出现
- [ ] 每段只讲一个推理动作
- [ ] 读者能复述推理过程，不只是记住结论

### 结构与标题

- [ ] 标题是推理判断的内容，不是"论证链"
- [ ] H2 用问题句或推理判断，不用"子论证 1/2/3"
- [ ] H1 和 frontmatter title 一致
- [ ] 无模板提示语

### 证据与表达

- [ ] 作者明说 vs 归纳 vs 重构已标清
- [ ] evidence_status 准确
- [ ] 无评价句或重要性定级句
- [ ] 无泛指代词反复
- [ ] 无 SKILL 内部术语泄露
- [ ] 无文章自我指涉
- [ ] 无过程叙述句
- [ ] 无生造复合词
- [ ] 无先抽象概括再具体展开同一件事
- [ ] 读起来舒服

## 输出规则

- 正文 80–150 行
- frontmatter 通用字段 + 扩展字段齐全
- 主链宁少勿多，不为覆盖面硬凑

## 校准与例外

- 推理极其密集的书（如哲学著作），主链上限可放宽到 5 条，正文上限 200 行
- 推理性很弱的书（主要是案例集或经验谈），08 可以压缩到 50 行以内，只复原最核心的一条推理
