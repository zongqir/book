# 贴源精修 写作校准

## 推荐做法

**改正文时**

- 只吸收有增量价值的内容。NotebookLM 说的和你已经写了的意思一样，不用改
- 贴源后的内容保持原有文风。不要因为"有了原书依据"就突然换成学术腔
- 新补的判断用最短的话写完。一句话能说清的不要展开成一段
- 如果原书确实有这个说法，直接写结论。如果只是 NotebookLM 的推断，标记边界
- 改完的段落要和前后文节奏统一，不要突然冒出一块"补丁质感"的文字
- 改完后整体通读一遍上下文，确认衔接自然

**写 .QA.md 时**

- 每个问答都能独立阅读，不依赖"上一轮审查"的上下文
- 问题用读者会问的方式写，不用审稿人的口吻
- 答案直接给判断，不绕弯子
- 如果答案里有不确定的地方，在答案里直接说

## 不要这样写

**改正文时**

- 把 NotebookLM 的原始回答直接粘到文章里。要消化后用自己的话写
- 为了"更有依据感"加一堆"原书指出……""根据原文……"的标注句。正文不是论文
- 把一个小补丁写成一整段。如果只是补一个边界条件，加一句话就够了
- 改了事实但没改周围的上下文衔接。补丁要融入原文，不能读出接缝
- 因为 NotebookLM 提到了一个细节就觉得必须加进去。不是所有原书内容都值得写

**写 .QA.md 时**

- 写成"候选稿""筛选建议"格式。.QA.md 是给以后翻阅的，不是给审稿人看的
- 问题用"请评估……的重要性"这种命令式。换成读者真会问的方式
- 大段重复正文内容。只补正文没展开的
- 出现"保留/暂不保留""建议落稿位置""仍值得继续追问"这类过程语言

## 自检

- [ ] 改完的段落读起来像原文的一部分，不像一块补丁
- [ ] 新补的内容比原来更扎实，不只是换了个说法
- [ ] 没有为了"显得贴源"加入不必要的"原书说"前缀
- [ ] .QA.md 里的每个问答都比正文多推进了一步
- [ ] .QA.md 没有过程语言
- [ ] 没有把 NotebookLM 的推断当成原书的原话
