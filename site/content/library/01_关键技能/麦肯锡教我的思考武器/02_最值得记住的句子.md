# 十句提醒，把分析从乱搜资料拉回切题、拆解和收束

## 先切题

{{< sentence no="1" quote="一个切得不准的问题，会吞掉后面几乎所有努力。" >}}
这句该放在分析最前面。因为知识工作里最贵的浪费，往往不是算错，而是题目就不值得这样做。
{{< /sentence >}}

{{< sentence no="2" quote="真正的起点不是资料，而是题目值不值得做。" >}}
很多分析一上来就进资料堆，这句的作用是先把手刹拉住。
{{< /sentence >}}

## 再拆解

{{< sentence no="3" quote="不能拆的题目，通常也很难被真正回答。" >}}
这句的价值在于把复杂感变成结构工作。题目一旦拆开，答案的路径会开始显影。
{{< /sentence >}}

{{< sentence no="4" quote="复杂问题不是靠更大的力气推进，而是靠更清楚的切块推进。" >}}
这句适合拿来纠正另一种常见错法：以为复杂就该上更重的研究，实际上常常先该拆。
{{< /sentence >}}

## 用假说拉住搜集

{{< sentence no="5" quote="没有假说的搜集，最后常常只是勤奋地堆信息。" >}}
这句是整本书里最该反复调用的一句之一。它直接决定分析会不会失控。
{{< /sentence >}}

{{< sentence no="6" quote="先有一个暂时性答案，不是为了证明自己对，而是为了知道先去查什么。" >}}
这句能挡住“假说驱动=先入为主”的误读。假说是方向盘，不是终审判决。
{{< /sentence >}}

## 找关键信息

{{< sentence no="7" quote="真正该找的，不是更多信息，而是最可能推翻当前判断的那部分信息。" >}}
这句把搜集从“全面”改成“会改判断”。这是轻量分析链的核心节奏。
{{< /sentence >}}

{{< sentence no="8" quote="一条关键事实常常比十页背景资料更值钱。" >}}
这句很适合拿来对抗资料焦虑。不是资料越厚，分析就越深。
{{< /sentence >}}

## 最后收束

{{< sentence no="9" quote="如果不能压成一句清楚答案，多半说明前面的分析还没真正收束。" >}}
很多工作会停在“我大概懂了”，这句逼你把“大概”压成明确结论。
{{< /sentence >}}

{{< sentence no="10" quote="分析里的高效率，很多时候来自少做无效动作，而不是把每个动作做更快。" >}}
这句适合作为整条分析链的总结。它把“思考武器”从聪明感拉回动作节约。
{{< /sentence >}}
