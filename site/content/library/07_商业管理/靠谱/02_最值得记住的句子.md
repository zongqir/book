# 十句提醒，把靠谱从模糊评价拆成可执行动作

## 接任务时先把边界讲清

{{< sentence no="1" quote="答应得快不等于靠谱，把结果标准先对齐才算真的接住事情。" >}}
这句专门治“先答应再说”的习惯。很多失信不是做不出来，而是起点对的根本不是同一件事。
{{< /sentence >}}

{{< sentence no="2" quote="任务一到手先确认目标、时间、边界和验收，不要靠各自脑补合作。" >}}
适合拿来做接任务时的检查卡。只要这几项没对齐，后面的努力很容易都在替误解买单。
{{< /sentence >}}

## 推进过程里别让别人靠猜

{{< sentence no="3" quote="真正让人不安的，常常不是延期本身，而是延期之前一直没有真实信号。" >}}
它提醒的不是延期管理，而是过程透明。很多人怕说坏消息，最后反而把信任直接做没了。
{{< /sentence >}}

{{< sentence no="4" quote="风险越早暴露，越像负责；越晚解释，越像甩锅。" >}}
这句最适合拿来纠正“我再扛一下就好”的错觉。早报码不是示弱，而是在给协作方留动作空间。
{{< /sentence >}}

{{< sentence no="5" quote="进度汇报不是报平安，而是让别人判断接下来该怎么配合你。" >}}
它能把“状态更新”一下拉回真实用途。不能支持别人判断的汇报，通常都只是情绪安抚。
{{< /sentence >}}

## 交付不是把东西扔出去

{{< sentence no="6" quote="交付完成不只看你做完，还看下一个人是不是能顺着接下去。" >}}
很多人以为把文件发出去就算交付，这句正好把误解拉回来。交付能不能接力，才是真正的分水岭。
{{< /sentence >}}

{{< sentence no="7" quote="没有上下文的成果，常常只是把整理工作转嫁给别人。" >}}
这句适合放在交付前最后看一眼。你如果没把口径、假设和下一步说清，下游就得重新替你整理一遍。
{{< /sentence >}}

## 靠谱靠长期积，不靠单次表演

{{< sentence no="8" quote="职业信用不是靠一次惊艳表现建立的，而是靠长期可预测的交付积出来的。" >}}
它纠正的是“偶尔救火很漂亮，就等于值得信任”的误判。真正的信用来自长期稳定，不来自单次高光。
{{< /sentence >}}

{{< sentence no="9" quote="别人愿不愿意继续把关键事交给你，取决于和你合作时要不要总是多留一层防备。" >}}
这句最适合拿来做外部视角自检。你不要只问自己努力不努力，要问别人和你合作时累不累、慌不慌。
{{< /sentence >}}

{{< sentence no="10" quote="靠谱不是把所有事都揽下来，而是知道什么该接、接了怎么交、交完怎么让人省心。" >}}
用它收尾很合适。它把“靠谱”从性格词拉回边界、动作和交付方式。
{{< /sentence >}}
