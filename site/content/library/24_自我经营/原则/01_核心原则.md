# 七条准则：经验什么时候才开始值钱

{{< principle no="1" type="core" statement="吃过亏不等于长了教训，只有把痛苦追问到规则层，经验才开始值钱。" >}}

达利欧管理桥水四十年，最早想明白的就是这件事。痛苦只是信号。不追问、不记录、不命名错法，痛苦过完就蒸发了。

{{< /principle >}}

{{< principle no="2" type="core" statement="原则不是座右铭，是经过现实检验、下次还能拿来用的如果—那么规则。" >}}

「以后要更小心」不是原则。「如果关键判断只有单一信息源，先暂停并补一路验证」才是原则。能写成动作指令的，才值得留进系统。

{{< /principle >}}

{{< principle no="3" type="support" statement="判断过程必须被记录到自己能回看、别人能追问的地方。" >}}

不写下来，复盘就只能靠记忆。记忆和自尊会合谋美化当初的判断。外化不是形式，是让经验可检验的前提。

{{< /principle >}}

{{< principle no="4" type="support" statement="在专业问题上，不同人的判断权重不该天然相等。" >}}

谁离事实更近、谁在这类问题上有更长的正确记录，谁的意见就该多听。可信度加权不是压制讨论，是让讨论更快逼近真相。

{{< /principle >}}

{{< principle no="5" type="support" statement="错误必须被公开暴露，反馈才能进入系统。" >}}

如果失误只在私下承认，规则系统就只能抓到表层信号。透明不是价值观表演，是让反馈链条不断掉的机制设计。

{{< /principle >}}

{{< principle no="6" type="support" statement="任何原则都必须保留被推翻的可能。" >}}

规则写死的那一刻，就从帮你贴近现实变成帮你回避现实。成熟不是原则越来越多，是原则越来越经得起现实敲打。

{{< /principle >}}

{{< principle no="7" type="support" statement="规则是用来辅助判断的，不是用来替代判断的。" >}}

流程、指标、打分表都是工具，不是地面。一旦迷恋系统的整齐感而忽视现场反馈，原则系统就走到了自己的反面。

{{< /principle >}}
