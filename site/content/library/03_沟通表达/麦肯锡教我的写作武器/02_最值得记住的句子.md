# 写的是主题还是信息，一秒就能判断

## 信息与主题

{{< sentence >}}
主题只圈范围，信息才传递判断。
{{< /sentence >}}

"Q2 市场情况"是主题。"Q2 华东区销售额同比下降 12%"是信息。写报告时自查一遍小标题，如果全是主题标签，正文再长也没在说话。

{{< sentence >}}
写不出信息的地方，通常是还没想清楚。
{{< /sentence >}}

把主题硬改成信息时会卡住——卡住的原因不是文字功力不够，是判断还没做完。承认这一点比强行凑句子有用。

## 三种信息类型

{{< sentence >}}
记述、评价、规范，三种信息回答三个不同的问题。
{{< /sentence >}}

记述回答"发生了什么"，评价回答"这意味着什么"，规范回答"该怎么办"。从事实跳到建议时中间缺评价，读的人一定会追问"凭什么"。

{{< sentence >}}
同一段里混着三种信息类型，读的人得自己拆逻辑。
{{< /sentence >}}

一段话先说数据、再说判断、再给建议，感觉很紧凑。但对方读的时候必须自己分辨哪句是事实、哪句是判断、哪句是请求。拆开写，每句的功能更清楚。

## 逻辑连接

{{< sentence >}}
So What 从下往上检查：这些加在一起说明什么。
{{< /sentence >}}

列了三条事实，问自己一句"所以呢"。答不上来，说明上面缺一层归纳。不是内容不够，是还没把内容收成判断。

{{< sentence >}}
Why So 从上往下检查：凭什么你能这么说。
{{< /sentence >}}

写了一句结论，问自己"理由呢"。拿不出两三条站得住的支撑，说明结论还不够稳。先补理由，再决定结论用多大力气。

## 说服结构

{{< sentence >}}
并列型摆证据，解说型讲推导——写之前先选一种。
{{< /sentence >}}

推荐方案时三条理由各自独立，用并列型。分析问题成因时前一条推出后一条，用解说型。没选就开写，写到一半容易两种混在一起，读的人跟不上。

{{< sentence >}}
结构选错不会报错，但对方的理解路径会跑偏。
{{< /sentence >}}

该用因果链条的地方摆了并列理由，对方看到的是"三个平行原因"而不是"一条推导线"。逻辑结构传递的不只是内容，还有内容之间的关系。
