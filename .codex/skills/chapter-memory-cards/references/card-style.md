# 记忆卡片文风口径

## 总体要求

- 先像判断，再像解释
- 先短句，再补必要信息
- 读起来像有经验的人在提醒，不像 AI 在总结

## 标题口吻

优先写成这种形状：

- `X 不等于 Y`
- `先……再……`
- `X 只能当线索`
- `别把 X 直接当成 Y`

避免这种形状：

- `关于 X 的重要说明`
- `X 的核心原理`
- `为什么 X 很重要`
- `警惕……` 后面跟很长的解释性尾巴

## 字段口吻

### `phenomenon`

- 先写现场最先看到的现象
- 一般 1 句
- 不要先解释原理

### `dont`

- 直接写最容易下错的结论
- 句子要硬，少绕
- 优先用“不要……”开头

### `check`

- 直接写先补哪组证据
- 尽量用并列短语，不要展开成长段

### `upgrade`

- 写“满足什么条件才配升级成结论”
- 优先用“当……时，……才……”这种结构
- 不要在这里补背景课

### `why`

- 只补一句为什么值得留这张卡
- 说清它纠正什么误判、补什么盲区、为什么值得反复看
- 不要夸张，不要赞美式收尾

## 长度要求

- `title`：尽量 8 到 18 个字
- `phenomenon` / `dont` / `check` / `upgrade`：尽量各 1 句
- 单字段超过 45 个汉字时，优先再压

## 现成风格参考

更接近目标的例子：

- `文件慢，不等于设备慢`
- `await 和 util 只能当线索`
- `CPU 高，先分做工、排队、降频和失真`

它们的共同点：

- 标题短
- 误判先行
- 读者一眼知道这张卡在纠正什么
