# NotebookLM 槽位判定

NotebookLM 在这些写作 skill 里，不是用来"帮你想内容"，而是用来验证：

- 按当前 skill 的标准，这个槽位到底能不能成立
- 原书里哪些材料足以支撑正式文稿
- 如果不能成立，缺的是原书本来没有，还是当前 source 覆盖不够

## 基本原则

1. 先由 skill 定义"什么叫成立"
2. 再把这个标准翻译成 NotebookLM 问题
3. 先问成立性，再问筛选性，最后问退出性

是把 skill 的判定标准压成一组更窄、更可核对的问题。

## 三类问题

### 1. 成立性问题

用来判断这个槽位能不能写。

典型问法：

- 原书里有没有足够稳定的材料支撑这个槽位？
- 这些材料是作者明说、结构归纳，还是只能重构？
- 按这个槽位的要求，现在已经够成正式稿，还是只够内部诊断？

### 2. 筛选性问题

用来判断如果能写，到底该留什么，不该留什么。

典型问法：

- 如果只能留 3 到 8 个内容，最该留下哪几个？
- 哪些候选内容最符合当前槽位标准，为什么？
- 哪些内容看起来相关，但不够格进入这个槽位？

### 3. 退出性问题

用来判断如果不能写，问题到底出在哪里。

典型问法：

- 是原书本来就不提供这一层内容，还是当前 source 覆盖不够？
- 是材料太弱，还是槽位本身不匹配？
- 如果不该写当前槽位，更应该转去哪一页？

## 反模式

不要问这些：

- `这本书讲了什么？`
- `帮我总结一下这本书。`
- `还有哪些值得写的内容？`
- `有哪些名句/案例/方法？`

这些问题会把 NotebookLM 拉回摘要模式，而不是槽位判定模式。

## 输出要求

优先让 NotebookLM 返回这些东西：

- 明示材料 / 归纳材料 / 重构材料的区分
- 哪些章节、段落、表达最接近当前槽位要求
- 哪些候选内容应该保留，哪些应该剔除
- 如果不能成立，最主要的原因是什么

如果判定仍然不够稳，正式稿先不要硬补，保留同名 `.QA.md` 作为内部诊断。
