# 读书库

这个仓库现在的重心，不是收集电子书本身，而是围绕 `site/content/library/` 持续生产可发布的单书文稿、章节拆解和站点内容。

## 当前真实结构

当前真正长期使用的目录有：

```text
book/
  00_inbox/
  site/content/library/
  site/static/
  astro-site/
  scripts/
  .codex/skills/
```

- `00_inbox/`：新收集的原始资料。先放进来，再决定是否入库
- `site/content/library/`：正式资料库正文。默认围绕这里工作
- `site/static/`：站点静态资源和导出数据
- `astro-site/`：Astro 展示层
- `scripts/`：内容校验、导出、构建和辅助脚本
- `.codex/skills/`：项目级写作和研究 skills

`app/` 目前仍在仓库里，但不是当前主展示链路。当前站点和发布主线以 `astro-site/` 为准。

当前准备把一级分类重构为 17 类。目标不是“分得越多越好”，而是把本来就不该混在一起的问题域拆开。

目标一级分类如下：

1. `自我经营`
2. `学习方法`
3. `沟通表达`
4. `职场协作`
5. `管理领导`
6. `商业战略`
7. `资产投资`

这 17 类还没有批量迁移完。当前阶段先统一分类模型，再逐步迁目录、脚本和站点文案。

## 这次重分的核心判断

这次重分不是为了追求更细，而是为了解决几个根本错位：

- `个人`、`职场`、`管理` 不是一类
- `投资`、`金融`、`经济` 不是一类
- `政治`、`历史`、`思想` 不是一类
- `历史传记` 这种“问题域 + 体裁”混合命名不适合继续做一级分类
- `健康生活` 当前没有形成稳定书目簇，暂不作为目标一级分类保留

## 目标分类边界

下面这组定义用于后续迁移判断。

### `自我经营`

处理习惯、执行、自控、时间安排、个人秩序。

收：

- 习惯养成
- 自我管理
- 执行系统
- 时间与精力管理

不收：

- 纯学习方法
- 纯心理机制理论
- 纯职场关系处理

### `学习方法`

处理阅读、提问、思考、学习、问题定义。

收：

- 阅读方法
- 学习策略
- 提问与思辨
- 问题分析

不收：

- 个人习惯训练
- 哲学原理书
- 专业技术手册

### `沟通表达`

处理对话、演讲、表达、说服、谈判。

### `职场协作`

处理组织中的位置判断、向上管理、协作关系、办公室博弈。

### `管理领导`

处理带人、授权、机制、分工、组织运行和领导责任。

### `商业战略`

处理商业判断、竞争、增长、创业、行业策略。

### `资产投资`

处理资产配置、估值、选股、投资判断。

### `金融体系`

处理金融市场结构、财报、银行体系、金融史、金融运作机制。

### `经济分析`

处理增长、制度、周期、危机、宏观与微观经济解释。

### `国家治理`

处理国家能力、制度设计、权力运行、国内治理框架。

### `国际政治`

处理国际关系、地缘政治、大国博弈、国际秩序。

### `中国历史`

处理中国史、中国社会结构和制度变迁。

### `世界历史`

处理世界史、全球史、文明比较、跨区域历史演化。

### `哲学思想`

处理哲学、政治思想、原则判断、世界观问题。

### `心智成长`

处理情绪调节、意义感、自我修复、自我整合。

### `心理机制`

处理人格结构、社会心理、行为偏差、影响机制。

### `技术工程`

处理计算机、工程实践、系统设计，以及 AI 与数字工具。

`个人GPT` 这类内容，当前先归入 `技术工程`，再在二级分类里细分为 `AI工具` 或 `数字工作流`，暂不单独升级为一级分类。

## 工作方式

默认工作顺序：

1. 原始资料先进入 `00_inbox/`
2. 正式正文落到 `site/content/library/`
3. 先确定归类，再补固定槽位
4. 提交前跑校验脚本
5. 通过后再导出站点数据或构建 Astro

## 常用命令

查看库内结构：

```powershell
Get-ChildItem site/content/library -Depth 2
```

搜索书名或文稿：

```powershell
rg "关键词" site/content/library
```

检查 frontmatter：

```bash
python scripts/check-frontmatter.py
```

检查写作反模式：

```bash
python scripts/check-writing-patterns.py
```

检查槽位覆盖：

```bash
python scripts/check-slot-presence.py
```

检查同质化：

```bash
python scripts/check-homogenization.py
```

导出站点内容：

```bash
python scripts/export_site_content.py
```

构建 Astro：

```bash
python scripts/check-astro-build.py
```

本地预览：

```powershell
./scripts/start-site.ps1
```

## 维护原则

- 默认围绕 `site/content/library/` 工作，不在根目录再维护平行正文树
- 一级分类优先表达“问题域”，不表达“体裁”
- 一级分类控制在 20 个以内，当前目标为 17 个
- 热点主题先放二级分类，不轻易升成一级分类
- `_index.md` 只做入口，不写成长正文
- 分类重构先定模型，再迁目录，不把“目标状态”伪装成“当前事实”
