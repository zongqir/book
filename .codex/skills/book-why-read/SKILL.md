# 单书入口稿

把 `00_为什么读这本书.md` 当成"阅读入口判断 + 情绪点火 + 核心价值入口稿"。

这篇稿子优先回答两件事：
- 这本书最该从什么入口进入读者判断系统
- 读完以后，后面还会继续带着走的判断是什么

但不要把“入口判断”理解成纯理性判断。00 页不是采购评审，也不是选品报告。它除了告诉读者“为什么该读”，还要让读者在开头就感觉到：这本书碰到了我的某个痛点、欲望、亏欠、羞耻、松动或盼头。



这里的“入口”不只是一种。常见有四类：
- 当前语境入口：为什么现在进入阅读队列更有价值
- 误判入口：它最该拿来纠正什么高代价错法
- 问题入口：它最擅长处理什么问题或困惑
- 情绪入口：它先碰到了读者哪一种真实感受，才让后面的判断开始成立

不要把“为什么现在读”误写成唯一入口。它只是常见入口，不是每篇 00 都必须正面展开的唯一轴线。
也不要把 00 写成纯理性判断稿。很多书之所以值得进入，不是因为你先算明白了价值，而是因为它先把某种说不清的卡点说中了。

## 1. 先判定是否命中

出现下面这些情况，就按这个 skill 处理：
- 用户直接提到 `00_为什么读这本书.md`、`00` 文件、单书入口稿、开篇稿
- 用户要新建、重写、改写、润色、审阅、去重单书开篇文稿
- 用户说入口稿太像书评、太像摘要、推荐味太重、开头两篇重复、想把"为什么读"和"主要价值"合并
- 任务语境明确在读书库或研究资料库里处理单书开篇文档

下面这些任务不要落到 `00`：
- 讲方法结构和部件关系：去 `05_方法论总结.md`
- 讲具体步骤和执行顺序：去 `04_行动指南.md`
- 讲适用前提、不要硬套、失效信号和停退换判断：去 `06_适用边界与失效条件.md`

## 2. 先补元数据

通用字段与语义统一遵循仓库 `AGENTS.md`。这个槽位只额外关注：
- `why_now`

同时先把这两个通用字段写对：
- `title` 不要直接写 `为什么读这本书` 或 `为什么读《书名》`，而要把正文最重要的入口判断压成一句内容标题
- `summary` 直接概括"入口判断 + 读完后会继续带着走什么判断"，不要写成 `帮助读者判断……`、`这篇文章会……` 这类任务说明

最小示例：

```yaml
why_now: "信息过载阶段最容易把输入错当理解，这本书能先把阅读流程和判断顺序立起来。"
```

`why_now` 与开头段的分工：
- `why_now` 是元数据字段，写给筛选和索引用，一两句话交代时机判断即可
- 开头段是正文入口，要先点中一个真实入口，再自然带出后面会留下什么
- 两者可以有重叠，但开头段不应该是 `why_now` 的逐字扩写，`why_now` 也不应该是开头段的压缩摘要
- 如果正文主入口是误判入口或问题入口，`why_now` 可以只在元数据中成立，不要求正文每段都围着“现在读”打转
- 如果正文主入口是情绪入口，`why_now` 也不要强行在首段里抢主位；情绪点中了，时机判断可以放在后面补稳

再确认当前证据边界是否写清：
- 只基于目录、抽样、既有材料或模型先验时，保持 `evidence_status: llm-draft`
- 不把未核对判断写成已经贴源的结论

## 3. 按这个骨架落稿

### NotebookLM 提问约束

当你不确定 `00` 能不能成立、怕它滑成摘要或推荐稿，或拿不准入口判断到底该落在哪时，先按这个 skill 的标准去问 NotebookLM。

- 先问成立性：原书里有没有足够材料支撑“当前语境入口 / 误判入口 / 问题入口 / 情绪入口”中的至少一种？
- 再问筛选性：哪几个问题、误判或价值最值得放进入口稿，而不是把目录和内容复述一遍？
- 最后问退出性：如果 `00` 写不稳，是原书入口判断本来就薄，还是 source 还不足以支撑入口判断和核心价值？

优先问这种问题：

- 这本书最该从哪一种入口进入阅读队列：当前语境、常见误判，还是某个长期问题？
- 它最容易先碰中读者哪一种真实感受：羞耻、亏欠、疲惫、渴望、松一口气，还是别的什么？
- 它最值得拿来纠正什么高代价误判？
- 如果不从“为什么现在读”切入，还有什么入口更稳？
- 如果读完只能留下 3 到 5 个最核心的价值判断，哪些值得继续带着走？
- 哪些内容看似相关，实际上更应该留给 `04`、`05` 或 `06`？

不要问：`这本书讲了什么，为什么值得读？`

通用判定方法见 [../references/notebooklm-slot-gating.md](../references/notebooklm-slot-gating.md)。

先看 [writing-rubric.md](./references/writing-rubric.md) 和 [gold-standard.md](./references/gold-standard.md)。

如果你发现自己开始写“这篇文章会”“这页主要回答”“更适合哪种人现在读”这种句子，先停下来，再看 [../../references/writing-anti-patterns.md](../../references/writing-anti-patterns.md)。00 页最常见的坏味道，不是信息不够，而是开始解释文稿功能、读者分群和内容加工流程。

默认按这组顺序推进：
1. 先选一个最稳的入口，不要三个入口一起上
2. 开头先点中这个入口，再展开它在解决什么问题
3. 正文拆出 3 到 5 个真正该留下来的核心价值；如果少于 3 个，要能说明为什么

执行时用硬规则，不用靠感觉：
- 第一段必须先让读者感觉到“这事碰到我了”，再带出"最值得关注什么"
- 每个主体段落只承载一个核心判断
- 标题和 `summary` 都要从内容判断里长出来，不要回到槽位名或任务说明
- 小标题优先写成问题判断式、价值判断式或误判纠正式
- 避免泛化标题，如 `为什么读这本书`、`主要价值`、`适合谁`
- 把 `适合谁`、`更适合哪种人`、`哪些人适合读` 当成硬禁例；它们属于导购式结构壳，不是内容判断，终稿里不要保留
- 不要用元说明句领起段落，如 `这篇文章会`、`这页主要回答`、`这一节会先……再……`
- 不要把内容生产动作写成正文重心，如 `把这本书压成……`、`把这页转成……`、`入口稿 / 判断稿`；00 页要交付的是判断，不是加工过程
- 不把正文写成 `价值一 / 价值二 / 价值三` 的空壳列表
- 不要默认用 `如果你……这本书值得……` 起手；条件句只能偶尔用，不能成为整篇骨架
- `summary`、首段、H2 不要同时都写成条件句；三者里最多保留一处明显条件句
- `值得现在读`、`值得早点读`、`值得尽快进入书库` 这类短语，全篇最多出现一次
- 开头不要只有道理，没有情绪温度；至少要让读者感到一种真实代价、真实渴望，或一种“终于有人把这件事说出来了”的松动
- 情绪不是抒情表演。不要故作煽情，不要空喊“人生”“热爱”“勇气”；要把情绪落在具体处境里

如果拿不准结构和文风，直接参照 [gold-standard.md](./references/gold-standard.md)。重点看：
- `title` 怎么从内容判断里来，而不是写成槽位名
- 开头段怎么先点中一个真实入口，再自然带出"读完后会继续带着走什么判断"
- 小标题怎么直接落在判断上
- 每段怎么只承载一个核心判断，不做空壳编号
- 不同开头范式怎样写，避免机械复用单一样板句法
- 情绪入口怎样写得有温度，但不滑成鸡汤、煽情或口播推荐

## 4. 用两个问题筛正文

每一段都要主要回答下面两个问题之一：
- `它最该从什么入口进入我的判断系统？`
- `如果我读了，读完后还会继续带着走什么判断？`

如果某段主要不在回答这两个问题，就删掉、改写，或移到别的槽位。

优先删除这些内容：
- 按章节复述目录或内容
- 只是在换说法重复"这本书很重要"
- 前半篇是推荐稿，后半篇才开始列价值
- 只有"适合谁"，没有"读完留下什么"
- 只有一排价值点，没有"为什么现在读"
- 大量 `这本书讲了`、`作者介绍了`、`它会让你` 这类转述腔

## 5. 终检清单

提交前逐条过一遍：

**核心**
读者读完以后，能回答“它为什么该进入我的判断系统”以及“读完后会继续带着走什么”
**元数据**
- [ ] `title` 是内容判断，不是 `为什么读这本书` 或 `为什么读《书名》`
- [ ] `summary` 直接写内容判断，不是 `帮助读者判断……` 这类任务说明
- [ ] `why_now` 不是占位空句，有具体的时机判断
- [ ] `created_at` / `updated_at` 是完整带时区的 ISO 时间，不是纯日期
- [ ] `evidence_status` 与实际来源一致；没有贴源就写 `llm-draft`

**结构**
- [ ] 第一段先下了入口判断，而不是先搭台或先分群
- [ ] 正文有 3–5 个核心价值段落，每段只承载一个判断
- [ ] 小标题直接落在内容判断上，没有保留结构壳标题
- [ ] 没有出现 `适合谁`、`更适合哪种人`、`哪些人适合读` 这类导购式标题
- [ ] 正文 H1 与 frontmatter `title` 一致
- [ ] 没有把“为什么现在读”写成唯一入口，除非这本书确实最适合这样进入
- [ ] 开头不是只有理性判断，还点中了一个真实处境里的情绪入口

**槽位边界**
- [ ] 没有滑成书评、摘要或推荐稿
- [ ] 没有混入方法论结构（→ 05）、执行步骤（→ 04）或边界条件（→ 06）
- [ ] 没有按章节复述目录

**文风**
- [ ] 没有连续出现"这本书讲了""作者介绍了""它会让你"这类转述腔
- [ ] 没有用 `这篇文章会`、`这页主要回答` 这类元说明句解释文稿功能
- [ ] 没有把 `把这本书压成……`、`入口稿 / 判断稿` 这类内容生产语言写进正文
- [ ] 没有空泛夸赞（"经典""有启发""值得一读"）
- [ ] 标题和段首句读起来像自然中文，不像模板填空
- [ ] 没有把 `如果你……这本书值得……` 复用成 `summary`、开头和 H2 的统一模板
- [ ] 开头 5 句里 `如果` 不超过 1 次；H2 标题里条件句不超过 1 个
- [ ] 读起来不是纯理性说明书，开头能让人感到被碰到，但又没有滑成煽情推荐文

**长度**
- [ ] 正文在 80–130 行之间；低于 80 行说明判断没展开，超过 130 行检查是否混入了别的槽位内容

## 输出规则

- 多用短段落、直接判断、具体场景和有温度的真实感受
- 先选最稳的入口，再说它到底留下什么
- 文风保持平实，但不要把平实误写成干巴巴的论证摘要
- 改现有稿子时，优先保留已经成立的有效段落，不做机械式全量重写
- 正文典型长度 80–130 行；低于 80 行通常说明判断没有展开，超过 130 行要检查是否混入了相邻槽位的内容
