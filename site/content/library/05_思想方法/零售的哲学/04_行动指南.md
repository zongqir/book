# 四个经营动作，从一个货架开始验证

周会上只有人报数字、聊竞品，没人说得出"顾客最近到底哪里不一样了"——出现这个信号，就该停下来跑一轮。下面四个动作可以在一周内走完。

## 站在收银台后面看半小时

观察顾客变化的起手式不是调报表，是看人。

关掉电脑，站到收银台后面或货架通道旁边，连看半小时。看的时候只问自己三个问题：

1. 最近哪类顾客来的时间变了？早高峰的人少了，还是晚8点后突然多了一拨？
2. 哪个货架前停留时间变了？是看得更久但不买，还是根本不停了？
3. 有没有人拿起来又放回去，或者到柜台问"你们有没有某某"？

三个问题答完，挑出一条最具体的变化。"顾客变了"不算。"工作日晚8点后的年轻女性明显多了，但即食区的停留时间反而变短了"才算。

**判断标准：** 你找到的信号能落到一个时段、一类人、一个货架。三个要素缺一个，就还不够具体。

**常见陷阱：**

- 把自己的焦虑当顾客变化。"最近生意不好"是经营感受，不是顾客行为。
- 只看POS不看人。报表告诉你结果，但看不到顾客犹豫了三秒之后拿了旁边那个。
- 信号太大。"消费降级"没法验证；缩到一个品类、一个时段才能动手。

## 写一句能被下周数据打脸的假设

观察到变化之后，把它压成一句话。这句话必须满足一个硬条件：下周的结果能把它推翻。

**操作步骤：**

1. 用这个句式起草——"在[时段/场景]里，[这类顾客]更想要[具体需求]，而不是[当前供给]。"
2. 检查：这句话是不是同时猜了两三个原因？如果是，只留一个，其余的另写。
3. 再检查：如果数据完全相反，这句话算被推翻了吗？如果怎么都能自圆其说，还不够锐利。

**示例：**

- ❌ "顾客更重视品质了"——太宽，不管卖什么都能套上去。
- ✅ "工作日晚8点后进店的顾客，宁可多花5块钱买一份小份即食，也不愿意从冷柜里自己组合晚餐。"

写完之后问自己："接下来我要看哪个商品在哪个时段的什么数据？"答得出来，假设就够用了。

**常见陷阱：**

- 写成愿望。"推出XX顾客一定喜欢"是计划，不是假设。
- 塞太多变量。时段、人群、品类同时猜，最后验不清楚哪个对了。
- 只为印证已有判断。如果你已经"知道"答案，你写的只是说服材料。立刻换一个人来挑战这句话。

## 选一个单品，在一个货架上跑五天

假设成形之后，动作要小到不需要审批。不改门店，改一个货架；不调品类，调一个单品。

**第一步：选品。** 选和假设直接挂钩的商品。假设说"顾客想省决策时间"，就去决策成本最高的品类里挑一个最有代表性的单品——比如那个SKU最多、顾客拿起来比较最久的货架。

**第二步：定位置。** 把它放到假设指向的位置。假设指向"晚8点后的即食需求"，就让它出现在晚间动线最顺手的地方。

**第三步：只改这一刀。** 价格不动，促销不加，旁边的陈列不换。一次只动一个变量，才能看清反馈来自哪里。

**第四步：连记五天。** 每天记三个数：

- 目标单品在目标时段卖了多少
- 顾客拿起来又放回去多少次（需要现场观察，POS看不到）
- 连带购买了什么

**判断标准：**

- 连续3天以上同方向变化 → 信号初步成立，值得扩大范围。
- 只有1天好看 → 偶然波动，不做判断。
- 5天后完全没变化 → 回到上一步，重新检查假设。

**常见陷阱：**

- 同时改了三件事。位置换了、价签调了、旁边还加了POP——到底哪个起了作用？
- 只看销售额。总额涨了5%说明不了什么，要看"目标时段的目标单品有没有变"。
- 太早下结论。两天好就全店推，三天差就放弃——都太急。给自己定死五天再决策。

## 便利的边界该不该再往外推一步

铃木敏文做过的很多决策——代收水电费、装ATM、卖现磨咖啡——本质都在回答同一个问题：顾客心里"便利"的边界是不是又移了。

当团队提出"我们能不能也做XX"时，用这四条过一遍：

**一、和顾客已有的到店动线矛盾吗？** 如果需要专门绕路或多花十分钟，推不动。便利扩展必须顺着现有习惯走，不能让顾客改路线。

**二、替代方案的体验有多差？** 便利店ATM的对手不是其他便利店，是银行排队。替代方案越笨重，你切进去的空间越大。如果替代方案已经很好用了，硬推没意义。

**三、现有团队不加人能交付吗？** 一个新服务如果需要额外培训两周、多加一个岗位，启动成本太高。先找不需要增加人手就能跑起来的做法。

**四、失败了损失可控吗？** 能不能在一家店、一个月内试完？如果试错成本低到可以承受，就值得开一轮。

**判断标准：** 四条里至少三条过，开一轮小试。只过了一两条，先搁着等更好的时机。

**常见陷阱：**

- 为创新而推边界。"别人没做过"不是理由，"顾客到店时确实需要"才是。
- 忽略交付复杂度。很多扩展在顾客端看着轻，在执行端很重。
- 把"没人投诉"当成"有人需要"。没有拒绝不等于有需求。

## 三个做偏了的信号

跑验证的过程中出现下面任何一条，停下来回溯：

**团队开始找证据支持已有结论。** 验证变成了立场辩护——大家只挑对自己有利的数据。重写假设，或者换一个人来主导。

**改了很多东西，说不清在验证什么。** 动作和假设脱节了，每天的调整已经偏离了最初那句话。回到第二步，把假设再压一遍。

**所有人都在盯大盘数字。** 总销售额、总客流、总毛利——这些指标太粗，变化信号被平均值稀释了。回到第三步，重新盯住一个单品、一个时段。

## 一页清单

### 开始前

- [ ] 找到一个具体的顾客行为变化信号（时段 + 人群 + 货架，三要素齐）
- [ ] 写出一句能被下周数据推翻的假设
- [ ] 选定一个单品、一个货架、一个时段

### 试跑时

- [ ] 只改一个变量
- [ ] 每天记三个数：目标单品动销、拿起放回次数、连带购买
- [ ] 连续记录至少5天

### 收口时

- [ ] 写清假设是否成立
- [ ] 留下3条具体反馈（哪个品、哪个时段、哪类顾客）
- [ ] 如果便利边界值得推，四条判断清单至少过三条
- [ ] 有效做法写进订货规则或陈列原则，不留在口头
