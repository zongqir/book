# 历史临场解读

这个 skill 不负责替代 `AGENTS.md` 决定最终落哪个固定槽位。

它负责先回答一件事：这本历史书到底在给读者什么。  
如果主要价值是 `处境进入 + 现象感知 + 后见之明纠偏`，就先用这套镜头，不要直接套 `04_行动指南` 或泛化 `05_方法论总结`。

## 1. 什么时候命中

出现下面这些情况，就先用这个 skill 收紧解读方向：

- 用户说历史类、国家治理类、战争史、帝国兴衰类书不适合硬写行动和方法
- 用户强调代入感、现场感、历史位置、上下文、前夜感、现象、压力、限制
- 用户说“我想知道当时到底像什么”“我无法理解二战/秦始皇/晚明/东汉末年的真实情况”
- 用户要解读历史书，但当前写法明显太像观点摘要、制度结论、现代分析报告

下面这些情况不要只靠这个 skill：

- 用户要逐章拆解：走 `engineering-book-chapter-breakdown`
- 用户要核对原文、贴源比对：走 `notebooklm`
- 用户明确只要某个固定槽位的产出：由槽位 skill 主导，这个 skill 只提供历史镜头

## 2. 先回答六个问题

动笔前先把下面六个问题答出来，至少在工作上下文里明确，不要边写边猜：

1. 这本书最想恢复的，是哪一种已经失真的历史现场？
2. 读者如果站在不知道结局的当下，先会看到什么现象？
3. 当时最真实的压力是什么：财政、战争、秩序、合法性、派系、时间，还是别的？
4. 书里的人手里有哪些路？这些路为什么其实越走越窄？
5. 作者最强的地方，是人物困局、崩塌前夜、单一事件穿刺，还是制度切片？
6. 今天的读者最容易拿什么后见之明去误读这段历史？

## 3. 先判书型，再决定展开方式

以下书型只是分析工具，不是固定标题模板：

- `人物困局型`：从不同角色的受困方式照出同一堵墙，例如《万历十五年》
- `崩塌前夜型`：重点是秩序还没正式倒，但已经越来越没人信它还能收拾局面，例如《三国前夜》
- `事件穿刺型`：从一场具体事件一路向里钻，逼出更深层结构，例如《叫魂》
- `制度切片型`：从某个位置、流程、层级切开制度，看它如何在日常中运转或失灵
- `战争生存型`：把大历史压到战场、后方、逃难、补给、普通人生存上

同一本书可以兼具两类，但必须先判主型。主型决定：

- 开头先从现场还是先从人物进入
- 案例按人物排、按压力排，还是按事件链排
- `04`、`05` 默认是留正文、留 `.QA.md`，还是谨慎保留

## 4. 解读顺序

历史类默认按这个顺序推进，不要反过来：

1. 先写表面上看见了什么
2. 再写人卡在什么位置上
3. 再写限制如何收缩选择
4. 最后才命名结构问题和因果链

也就是说，`叙事不是包装，叙事本身就是理解发生的地方`。

如果你发现自己一上来就在写：

- 制度失灵
- 系统性危机
- 技术管理不足
- 结构性崩塌

先停下来。  
这些词可以用，但只能在读者已经看见现象之后再用。

## 5. 路由到固定槽位时的默认判断

对历史叙事、治理史、思想史、战争史这类书，且主要价值在处境重建、现象辨认和认知纠偏时，默认优先考虑：

- `00`
- `02`
- `03`
- `06`
- `08`
- `10`

必要时再补：

- `01`：原书确实反复验证一组稳定判断准则

不要默认补：

- `04`
- `05`

只有在原书明确给出可迁移的观察法、分析步骤、判断流程或实践框架时，才保留 `04` / `05`。否则按仓库规则留 `.QA.md`，把“不留”的原因写清楚。

## 6. 文风硬规则

- 先让读者进入局里，再让读者得出判断
- 先写眼前现象，不先写抽象结论
- 不把人物压成论点工具
- 不把历史压成“观点 + 例子”
- 不把后来的现代术语塞回当时人的脑子里
- 不把历史误写成组织管理手册
- 明确镜头边界：这本书看的是中枢、地方、战场、民间，还是其中一层

## 7. 终检

- [ ] 读者先看到现场，再看到结论
- [ ] 人物或事件被写成了处境入口，不只是观点论据
- [ ] 文章里保留了“读者不知道结局时会怎样理解局面”的空间
- [ ] 没把历史书硬翻译成行动建议或通用方法
- [ ] 已说明这本书的镜头边界，没有把它写成“历史全貌”
