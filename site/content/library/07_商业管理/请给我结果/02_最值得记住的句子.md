# 十句提醒，把忙碌感拉回结果感

{{< sentence no="1" quote="动作很多不等于结果很多，组织最会被过程热闹欺骗。" >}}
这句最适合放在会很多、群很热闹、进度感很强的时候看。它先把“忙”从“有结果”里拆出来。
{{< /sentence >}}

{{< sentence no="2" quote="结果不清，责任一定会散；责任一散，执行就会很忙却很空。" >}}
拿它去照任务起点。只要没人能一句话说清最后交什么，后面再怎么分工都容易虚。
{{< /sentence >}}

{{< sentence no="3" quote="真正的执行起点，不是把任务分下去，而是把要交回来的结果说出来。" >}}
这句最该在布置任务前翻出来。它纠正的是那种以为“已经分下去了”就等于启动了执行的错觉。
{{< /sentence >}}

{{< sentence no="4" quote="如果工作最后无法被验收，它就很容易在中途被各种努力感替代。" >}}
一旦验收口不清，过程就会疯狂膨胀。大家都能证明自己很努力，但没人能证明结果真的成立。
{{< /sentence >}}

{{< sentence no="5" quote="结果导向不是只看最后一秒，而是让每一步都能对着结果差距说话。" >}}
它纠正的是把结果导向读成“只在最后看结果”的粗暴理解。真正的方法是中途也一直对着结果差距。
{{< /sentence >}}

{{< sentence no="6" quote="很多解释之所以越说越多，是因为一开始没人把结果定义清楚。" >}}
这句适合放在复盘时看。解释多不一定因为人坏，很多时候只是因为起点就没把结果说硬。
{{< /sentence >}}

{{< sentence no="7" quote="过程本身不能自动证明价值，只有它最终产出的结果能证明。" >}}
当你又想拿过程完整感安慰自己时，把这句翻出来。它逼你回到最硬的问题：最后到底交回了什么。
{{< /sentence >}}

{{< sentence no="8" quote="结果不是一个更凶的词，它是组织执行里最基本的坐标。" >}}
这句拿来纠正“结果导向就是压人”的误听。没有这个坐标，过程、责任和反馈都会发散。
{{< /sentence >}}

{{< sentence no="9" quote="不能被确认收回的成果，常常只是在某个人那里看起来做完了。" >}}
它最适合放在交付后用。发出去不等于收回来，没有确认动作，结果就还悬着。
{{< /sentence >}}

{{< sentence no="10" quote="这本书最该留下来的，不是逼人更紧，而是逼人把结果说得更清楚。" >}}
用它压总纲最合适。以后忘掉别的，也别把这本书只记成一句更凶的话。
{{< /sentence >}}
