# 读书库

这个目录按“长期稳定分类 + 当前专题研究”来组织。

目标有两个：

1. 平时收书、找书、归档时结构稳定，不需要反复搬文件。
2. 做专题研究时，可以快速把相关材料整理到一个专题目录，再交给 NotebookLM 持续分析。

## 目录结构

```text
book/
  00_inbox/
  01_关键技能/
  02_专业技术/
  03_沟通人际/
  04_理财/
  05_思想方法/
  90_专题研究/
    _模板/
```

## 一级目录说明

### `00_inbox`

新进来的 PDF、电子书、文章、截图、网页导出先放这里。

如果暂时不确定归类，也先放这里，不要因为分类犹豫而中断收集。

### `01_关键技能`

放最值得长期反复调用的关键技能类内容，例如：

- 学习方法
- 写作与表达
- 知识管理
- 系统思考
- 问题解决
- 信息检索

### `02_专业技术`

放你主业或重点研究方向里的专业书、技术资料、行业材料。

你是程序员的话，建议按下面这套稳定结构继续细分：

- `00_待归类`
- `01_计算机基础`
- `02_编程语言`
- `03_工程实践`
- `04_系统设计`
- `05_DevOps与基础设施`
- `06_AI与数据`

对应说明见：

- `02_专业技术/README.md`

### `03_沟通人际`

放与人协作相关的内容，例如：

- 沟通
- 谈判
- 管理
- 组织协作
- 关系处理

### `04_理财`

放资产、投资、商业、财务认知相关内容。

### `05_思想方法`

放哲学、认知、社会观察、决策框架、价值观这类“长期影响判断方式”的内容。

这个名字比“思想大文件”更稳定，也不容易变成杂物间。

### `90_专题研究`

这里不是按书的“所属类别”来放，而是按你“正在研究的问题”来组织。

一个专题就是一个独立目录。后续需要用 NotebookLM 时，优先从各大类里挑材料，复制或整理到对应专题目录。

## 推荐工作流

### 日常收集

1. 新材料先进入 `00_inbox`。
2. 有空时再移动到长期分类目录。
3. 文件名尽量写清楚作者、书名、年份或版本。

### 做专题研究

1. 在 `90_专题研究` 下复制 `_模板`，新建一个专题目录。
2. 把相关 PDF、摘录、网页导出、自己的笔记放进专题目录。
3. 优先把“真正与问题相关”的材料放进去，不要整库都塞给 NotebookLM。
4. 在专题目录里持续沉淀问题、结论、引用和行动项。

NotebookLM 的角色和专题工作流说明见：

- `90_专题研究/NOTEBOOKLM_WORKFLOW.md`
- `RESEARCH_WORKFLOW.md`

## 命名建议

- 一级目录保留数字前缀，便于固定顺序。
- 专题目录建议使用简短、问题导向的名字，例如：
  - `AI产品方法`
  - `个人知识管理`
  - `长期投资框架`
- 文件名尽量避免只写“笔记1”“总结最终版”这类名称。

## NotebookLM 使用建议

- NotebookLM 更适合“围绕一个问题组织一组资料”，不适合直接对整个书库做混合分析。
- 一个专题里尽量只放强相关资料，这样回答质量更高。
- 你的结论、提问、提炼后的摘要，建议也保留在本地专题目录，不要只留在对话里。

## 项目级 Skills

这个仓库已经开始把重复出现的研究动作沉淀成项目级 skills，例如：

- `topic-bootstrap`：初始化专题目录、`topic.yml` 和首轮骨架
- `book-why-read`、`book-key-sentences`、`book-best-cases`、`book-action-guide`：生成单书目录里的固定文档
- `notebooklm-source-check`：把贴源分析和回写格式固定下来
- `dida-research-closure`：把研究推进拆成连续动作

完整技能分层、覆盖范围和文档格式约定见：

- `SKILLS_MAP.md`

## 维护原则

- 先收集，后分类。
- 长期分类保持稳定，不频繁折腾。
- 研究按专题推进，不把所有问题都塞进同一个目录。
- 同一份资料可以服务多个专题，不必强求只归属一个问题。
