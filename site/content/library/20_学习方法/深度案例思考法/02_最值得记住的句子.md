## 关于结果导向的误区

{{< sentence >}}
如果因为结果成功就反推这个决策是对的，你会忽视运气在其中的作用。
{{< /sentence >}}

同样的决策，在不同时期可能导致完全不同的结果。分析时要问：这个结果有多少来自决策的质量，有多少来自外部条件的巧合？

{{< sentence >}}
失败的案例有时教不会你什么，反而是"虽然失败但几乎成功"的案例最有价值。
{{< /sentence >}}

从"差一点就赢了"的案例中，能看清什么才是真正的关键要素。

## 关于信息选择的误区

{{< sentence >}}
分析案例时，你忽视的细节往往和你关注的细节一样重要。
{{< /sentence >}}

如果你说不出为什么某个信息无关，那它可能其实很相关。

{{< sentence >}}
"这个人之前的经历"这个看似无关的背景信息，往往解释了"为什么只有他想到了这个方案"。
{{< /sentence >}}

案例分析时，要有意识地问：遗漏的背景信息有哪些？

## 关于因果关系的误区

{{< sentence >}}
A 导致了 B，和 A 是 B 的最重要因素，是两回事。
{{< /sentence >}}

某个因素可能有贡献，但不是主要因素。分析时要排序：哪个因素贡献最大？

{{< sentence >}}
"所有成功的企业都做了 X"，不等于 X 是成功的原因，可能是成功的结果。
{{< /sentence >}}

因果的方向很容易反了。

## 关于推广的误区

{{< sentence >}}
一个案例的教训，往往只对在类似约束条件下的人有效。
{{< /sentence >}}

把一个成功案例推广到完全不同的场景，往往会失败。

{{< sentence >}}
"他成功了"最多只能告诉你有这样一条路可行，不能告诉你这条路对你也可行。
{{< /sentence >}}

从个案推导规律时，一定要明确：这个规律的适用范围是什么。

## 关于隐含假设的误区

{{< sentence >}}
大多数案例分析都基于一个隐含假设：决策者是理性的，而且拥有充分的信息。
{{< /sentence >}}

实际上，很多决策来自有限理性、有限信息、甚至是错误信息。分析时要问：决策者当时知道什么？不知道什么？
