# 复盘自检稿

## §1 命中判断

用户要求新建或重写 `10_复盘指标.md` 时启用。

典型触发：
- "帮我写这本书的复盘"
- "补一份 10 号槽位"
- "这份复盘读起来像绩效考核，重写"
- "指标太多了，帮我收一下"

## §2 元数据

通用必填字段遵循 `AGENTS.md` base schema。

扩展字段（按需保留，不强制）：
- `audience`

不设 `metric_basis` 这类考核型扩展字段——这不是评估工具。

补充约束：
- `confidence` 不是“绩效打分字段”，但当前仓库的固定槽位 schema 会要求保留；按仓库 schema 正常填写即可

## §3 工作流

### NotebookLM 提问约束

当你不确定 `10` 到底该不该写、哪些复盘场景最有区分度、或怀疑自己会把复盘页写成绩效表时，先按这个 skill 的标准去问 NotebookLM。

- 先问成立性：原书的方法有没有明确的行为变化，足以转成场景式自检？
- 再问筛选性：哪些日常场景最能区分"真的在用"和"其实没进入动作"？
- 最后问退出性：如果 `10` 不成立，是原书方法性太弱，还是当前 source 还不足以看清行为变化？

优先问这种问题：

- 读完这本书以后，哪些具体动作会和没读前明显不同？
- 哪 5 到 8 个场景最适合拿来做读完一段时间后的自检？
- 哪些候选问题其实在考知识记忆或态度表态，不适合进 `10`？

不要问：`这本书可以怎么复盘？`

通用判定方法见 [../references/notebooklm-slot-gating.md](../references/notebooklm-slot-gating.md)。

**⚠️ 以下写作流程是思考工具，不是输出模板。**

每本书的行为变化点不同，场景不同，提问方式也应该不同。写完后检查：如果你的 H2 标题、提问句式和同分类下另一本书的复盘指标一样，必须重新组织。尤其注意：不要每本书都用"三个月后"开头。

### §3.1 确定核心行为变化

先回答一个问题：**读完这本书，如果方法真的进入了执行，读者的什么日常行为会变？**

不是"读者应该知道什么"（那是方法论总结的事），而是"读者做事的动作会有什么不同"。

找到 3-5 个最有区分度的行为变化点。区分度的意思是：用了和没用，在这个场景里的反应明显不同。

### §3.2 设计场景式提问

每个行为变化点对应一个具体场景。场景要足够日常，读者能立刻对号入座。

提问方式：
- "你上一次遇到 X 的时候，做了什么？"
- "当 X 出现时，你的第一反应是什么？"
- "过去一个月里，有没有一次 X 的情况？你怎么处理的？"

以上只是示例句式，不要每本书都用这三种。根据这本书的内容特点，自行设计最贴合的提问方式。


不要写成：
- "你的 X 执行率是多少"
- "你在 X 方面的表现属于哪个等级"
- "请评估你的 X 水平"

### §3.3 写行为对比

每个场景 2-4 句。核心是通过具体动作让读者自己认出自己的状态。

写法：描述"在用"是什么动作，"没在用"是什么动作。不贴"好/坏"标签，不打分，不分等级。

对比要落到动作层面，不能停在态度层面。"你觉得自己做到了"是态度，"你上一次做 X 时先做了 Y 再做了 Z"是动作。

### §3.4 标题与结构

**标题**：用一个能触发自检欲望的问题句或判断句，不用"复盘指标"。

**H2**：每个 H2 是一个场景或一个行为变化点的自然语言描述，不是编号、分类名或结构壳。

**结构自由度**：不强制"过程/结果"二分。如果场景本身自然涵盖了"有没有在用"和"用了有没有效"，就不需要额外分类。分类服务于阅读，不服务于填表。

**收尾**：可选。如果有需要，可以加一段帮读者判断下一步的短段落。不要加 checkbox 清单、5 问模板或"一页判断"——整篇文章本身就是一页判断。

### §3.5 篇幅

**基线：60-100 行**（含 frontmatter）。

这是自检工具，不是文章。能快速扫完、反复拿出来用，比写得全面更重要。

超过 100 行的常见原因：场景太多（砍到 5-8 个）、每个场景写太长（收到 2-4 句）、混入了方法论解释（移到 05）。

### §3.6 反模式黑名单

以下表述不进正文：

- "好信号""坏信号"或任何二分标签对
- "执行率""成功率""达标率""命中率"等 KPI 词汇
- `- [ ]` checkbox 格式
- "这套方法""这本书的方法"反复出现（超过两次换掉）
- "指标一""指标二"编号式标题
- "过程指标""结果指标"作为 H2 分类标题
- "请评估""请打分""属于哪个等级"
- 泛指"你觉得自己做到了吗"——态度不算行为

## §4 终检清单

### 反同质化（必查）

- [ ] H2 标题序列和同分类下已有书的同槽位不同
- [ ] 开头第一句和同分类下已有书的开头不同
- [ ] 全篇对比句式（"不是X而是Y""与其A不如B"）合计不超过 2 次
- [ ] 没有使用可以套到任何书上的万能句式作为标题或开头


提交前逐条过：

1. 标题是内容句，不是"复盘指标"
2. 每个 H2 是场景或行为变化点，不是编号或分类名
3. 每个提问锚定了一个日常场景
4. 对比落在动作层面，不是态度层面
5. 没有"好信号/坏信号"或任何二分标签
6. 没有"执行率""成功率"等 KPI 词汇
7. 没有 `- [ ]` checkbox 格式
8. 没有"指标一/二/三"编号式标题
9. 没有"过程指标""结果指标"作为 H2
10. "这套方法""这本书"没有反复出现
11. 没有混入方法论解释（那是 05 的事）
12. 没有评价句或重要性定级句
13. 没有替作者解释心理、意图或关注点的句子
14. 没有自我指涉（"这页是""这份工具"）
15. 没有导游词或过程叙述
16. 没有生造复合词
17. 读起来像一次诚实的自我对话，不像年终述职
18. 总行数 60-100
19. frontmatter 完整，通用字段齐全
20. `evidence_status` 准确
21. 时间字段写昨天

## 输出规则

- 整篇是场景式自检对话，不是分类评分表
- 5-8 个场景式提问，每个 2-4 句
- 不设"最小复盘模板""一页判断"等固定尾巴段落——整篇文章本身就是复盘工具
- 结尾可选，不强制

## 校准与例外

- 如果一本书的方法论非常弱或几乎没有可执行方法，这个槽位可以跳过
- 技术类书籍的行为变化点通常更具体（"遇到性能问题时你先做了什么"），非技术类通常更偏判断习惯（"你看到 X 时第一反应是什么"）
- 行为变化点的数量取决于书的方法密度，不需要凑数
