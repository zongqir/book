# 减阅读疲劳重写

把这个 skill 当成正文减负器。

它只做一件事：
- 把已经有内容的稿子，改到读者第一遍就能跟上

它不做三件事：
- 不发明新观点
- 不重定槽位边界
- 不把原来的硬判断磨成软话

## 0. 先吃透“通俗易懂”

这里的“通俗易懂”不是幼儿化，也不是把内容写浅。

它真正指的是：
- 读者第一遍就能跟上
- 不需要把书面话翻译成人话
- 不需要作者一路搀着走
- 读完一句，主干已经清楚

反过来说，下面这些都不算通俗易懂：
- 句子太长，读到结尾才知道主语和重点
- 词太书面，读者得先在脑子里翻译一遍
- 解释欲太强，一句话讲完还要再讲两句
- 什么都说透，不给读者自己接上的空间

通俗易懂不是把读者当白痴。
恰好相反。

它是相信读者听得懂，所以作者知道什么时候该停。

## 1. 触发条件

出现下面这些说法时，用这个 skill：
- “这篇读起来累”
- “太绷了”
- “像 AI 写的”
- “像总结，不像正文”
- “像说明书”
- “帮我降模板感”
- “别改意思，重写句子”
- “加一点人的处境”
- “留一点白，别句句顶满”

下面这些任务不要只用这个 skill：
- 需要决定写哪个槽位：先按 `AGENTS.md` 路由
- 需要补结构、补案例、补方法：先用对应槽位 skill
- 需要贴源核对：走 `notebooklm`
- 只改 frontmatter summary：走 `book-summary-writing`

如果任务同时命中槽位 skill 和本 skill，默认顺序是：
1. 先让槽位 skill 把内容写对
2. 再用本 skill 给正文减负

## 2. 先查五类问题

重写前先判断，文章累在哪一类。不要一上来全篇抹平。

### A. 词壳过重

命中任一条，先改词壳：
- 句子先评价，后交内容
- 段首总是 `这本书`、`它`、`这套方法`
- 正文在解释“这篇在做什么”
- 正文在暴露加工动作

常见坏句：
- `这本书真正有价值的地方，是...`
- `最值得注意的是...`
- `这篇文章会...`
- `把这本书压成...`

处理原则：
- 直接写后半句的内容
- 能写动作就别写评价

### B. 句子太长

命中任一条，先拆句：
- 单句超过 35 字
- 25 字以上的句子连续出现三句
- 一句里塞了 3 个以上概念
- 连续三句同一开头
- 连续出现同一种转折壳

重点警惕：
- `不是...而是...`
- `如果...就...`
- `因为...所以...`

这些句式不是禁用。
问题在于一连用，句子就会又硬又长。

### C. 只有判断，没有处境

命中任一条，补处境露出：
- 全篇都在讲道理，没有人
- 只有方法，没有代价
- 只有抽象词，没有具体局面

这里补的不是抒情，是处境：
- 人卡在哪里
- 人为什么会默认那么做
- 再往下走会付什么代价

### D. 书面味太重

命中任一条，先换词：
- 同一句里连续出现“使用 / 进行 / 以及 / 由于 / 因此 / 从而”这类书面连接
- 明明能用常用词，却用了公文词
- 句子像说明书，不像人在说话

优先替换：
- `使用` -> `用`
- `进行分析` -> `分析`
- `对于` -> `对`
- `由于` -> `因为`
- `因此` -> `所以` 或直接删
- `以便` -> `这样就`

原则不是口语化表演。
是少让读者在脑子里做第二次翻译。

### E. 节奏板结

命中任一条，先打散节奏：
- 连续三段长度差不多
- 每段都在“判断 + 解释 + 结论”
- 同一判断解释两次以上
- 小标题已经把正文说完了

打散方式只有两种：
- 删掉重复解释
- 把长段和短段拉开

通用反模式对照看 [../../references/writing-anti-patterns.md](../../references/writing-anti-patterns.md)。
具体替换动作看 [rewrite-playbook.md](./references/rewrite-playbook.md)。

## 3. 按这个顺序重写

### 第一步：先删坏句壳

先删下面这类壳，不要急着润：
- 评价壳
- 元说明壳
- 加工流程壳
- 泛指代词壳

执行方法：
1. 圈出段首里不传递新信息的词
2. 直接看删掉后半句能不能独立站住
3. 能站住就删前半句

例子：
- `这本书真正有价值的地方，是它一直逼你先看顾客变化。`
- 先删成：`它一直逼你先看顾客变化。`

### 第二步：再拆长句和同构句

只做三种改法：
- 一句拆两句
- 先场景，后判断
- 把并排句改成长短交替

默认偏短，不要偏长。
能用短句说清，就别留成长句。

但下面这些情况必须拆：
- 一句里既有判断，又有边界，又有结论
- 同一句里 4 个以上抽象词连排
- 读完句子后，主干还不清楚

### 第三步：去书面化，再补处境

先改掉书面词，再补局面。
不要一边想写得轻，一边还满篇“使用 / 进行 / 由于 / 因此”。

不要硬加“焦虑”“痛苦”“热爱”这类词。
先补局面，再让感觉自己出来。

优先补这三种东西：
- 一个会发生在读者身上的卡点
- 一个默认反应
- 一个继续这样下去的代价

例子：
- 不写：`这会让人更焦虑。`
- 改写成：`你已经把日程排满了，托底动作还是第一个被删掉。`

### 第四步：删重复解释，留下停顿

“留白”不是故作文学感。
它就是两件事：
- 同一判断别解释两遍
- 关键句后允许停一下

执行方法：
- 长段后补一个短段收住
- 连续两句意思差不多，删更空那句
- 小标题只管定方向，不提前写完正文

如果一句话已经到了，就停。
不要抢着把第二层、第三层都说透。

## 4. 保真门禁

命中任一条，默认不过稿：
- 为了顺口删掉限定条件
- 把 `llm-draft` 写成定论
- 把“可能 / 往往 / 更像”改成绝对判断
- 把具体内容改成更空的漂亮话
- 把槽位边界改坏

## 5. 最低终检

提交前至少确认：
- 坏句壳明显下降
- 长句明显减少，但限定词还在
- 35 字以上的句子没有连排
- 书面连接词明显下降
- 同一种转折壳没有连排成骨架
- 文中已经出现至少一个具体局面
- 连续三段不再一样长
- 一句话说完后，没有紧跟一个“换说法再说一遍”的尾巴
- 读完以后，原来的核心判断没有被冲淡

## 输出规则

- 默认直接改正文，不只给建议
- 优先局部重写，不机械全稿推倒
- 如果全篇都被同一种句壳占满，再整体重排
- 向用户汇报时，说清这篇主要累在什么地方，改动重点放在哪两类问题
