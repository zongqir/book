{{< sentence >}}
问题解决的第一步不是找答案，而是弄清楚问题到底是什么。
{{< /sentence >}}

大多数人在问题面前焦虑，不是因为问题太难，而是因为问题太模糊。把"我该怎么办"翻译成一个具体的、可以检验的问题陈述，问题就已经解决了一半。

{{< sentence >}}
把大问题拆成小问题，小到你可以立刻动手的程度。
{{< /sentence >}}

"拆"这个动作本身就是行动。不需要一次性拆到底——先拆一层，做完第一个小块，剩下的自然会变清楚。

{{< sentence >}}
不要等到收集了所有信息才开始行动。
{{< /sentence >}}

信息永远收集不完。七步框架的核心假设是：行动本身会产生新信息。先动起来，在过程中修正方向，比等待完美方案更有效。

{{< sentence >}}
假说不需要一开始就对，但必须一开始就有。
{{< /sentence >}}

没有假说的分析是散步，有假说的分析是赶路。即使最初的假说完全错了，它也帮你节省了时间——因为你知道这条路不通，可以换一条。

{{< sentence >}}
用"是/否"来检验你的方案，不要用"好/坏"。
{{< /sentence >}}

"好不好"是个无底洞。"是不是解决了问题"是个可以回答的问题。判断标准从主观偏好转向客观检验，决策速度会快很多。

{{< sentence >}}
解决问题的能力不是天赋，是可以训练的技能。
{{< /sentence >}}

渡边健介写这本书的出发点就是这句话。解决问题不需要特殊智力，需要的是一套可重复的步骤和反复练习的意愿。
