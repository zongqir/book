## 关于开头的真实功能

{{< sentence >}}
前五页不是开场白，是面试。
{{< /sentence >}}

开场白可以慢慢来，面试不行。面试者在决定你能不能进下一轮之前，只给你几分钟。开头的每一段都必须在争取下一段被读到的权利。

{{< sentence >}}
读者不欠你耐心。你的故事必须自己赚取被读下去的资格。
{{< /sentence >}}

新手最大的幻觉是以为读者会"给机会"——先看看再说。事实是，读者的默认状态就是随时准备放下。每翻一页都是一次重新选择。

## 关于空转与紧迫感

{{< sentence >}}
引擎在响但车没动——这就是大多数开头的问题。
{{< /sentence >}}

有文笔、有氛围、有描写，就是没有故事开始。空转的开头最难自我诊断，因为写作者觉得自己在铺垫。编辑觉得他在浪费篇幅。

{{< sentence >}}
不要告诉读者天气，除非天气正在杀人。
{{< /sentence >}}

开头的每一个细节都必须服务于紧迫感。不是说不能写环境——而是环境描写必须在推动故事，不是在填充页面。

## 关于自检的方法

{{< sentence >}}
把你的前五页给一个陌生人看，问他主角想要什么。他答不上来，你就还没写完开头。
{{< /sentence >}}

这个测试比任何技巧分析都管用。因为它直接检测的是开头最核心的功能：让一个什么都不知道的人迅速关心你的主角。

{{< sentence >}}
开头的语气承诺了一种阅读体验——后面你必须兑现。
{{< /sentence >}}

语气是一种隐性合同。轻快的开头承诺了一段愉快的阅读；沉重的开头承诺了严肃的内容。不兑现比开头差更让读者愤怒。
