# 先定听众和目标，再碰一个字的内容

你已经知道下周要做一次发言。时间不多，不想再把成败押给临场状态。

下面这套流程解决的核心问题只有一个：怎样把一次表达准备得更稳，上台后更不容易失控。先走一遍最小版，再看完整流程。

## 时间紧就保住这四步

1. 写清这次演讲的目标、听众和一句核心结论。
2. 把内容压成"开场抓注意力 + 中间两到三个主点 + 结尾收行动"。
3. 预演一次开场和结尾，改掉空话和过长句。
4. 列五个最可能被问到的问题，提前写出回答方向。

这四步做了，"讲着讲着就散了"的问题会明显减轻。如果时间只够做一步，先做第一步。目标和听众没定，后面全是猜。

## 写清四件事再动手组织内容

开始准备前，先在一张卡片上写清楚：

- 这次讲给谁
- 他们为什么来听
- 希望他们听完后记住什么
- 希望他们听完后做什么

四件事写不出来，先别堆材料。不然很容易出现两种结果：要么内容太满，什么都想讲；要么内容太空，听完没人知道重点。

这张卡片就是整场演讲的锚。后面每个准备动作都围绕它展开。

## 按听众重排内容，删掉他们不需要的

把听众真正关心的放前面，背景和细节往后压。

准备时连续问自己四个问题：他们最关心什么？可能听不懂什么？为什么要现在听？哪些内容不讲也不影响主线？

一直按"我知道什么"堆材料，现场就容易自说自话、信息过量、重点不明。按"他们需要什么"排，能砍掉至少三分之一的内容。

如果发现自己舍不得删——停下来看一眼任务卡。不支持核心结论的，都可以删。

## 把一场演讲压成两到四个主点

结构不贪多，先保一条主线。

对多数工作型演讲，更稳的做法是：开场先交代问题或利益点；中间只保留两到四个主点，每个主点配一个例子或事实；结尾明确收束判断或行动。

做到什么程度算够：你不看长稿，只看一页提纲，也知道自己往哪走。如果一页纸写不下，说明主点太多。

## 单独打磨开场和结尾

开场和结尾不要顺带准备，要单独拿出来练。

开场至少做到一件事：让听众知道为什么现在值得听。结尾至少做到一件事：让听众知道整场话最后落在哪里。

重点删掉这些常见问题：开场先讲套话和感谢，迟迟不进主题；一上来就铺背景，听众还没建立兴趣；结尾重复前面内容却没收束判断；结束时语气发虚，整场失去落点。

如果只能重点练一段，练开场。因为开场一旦空，后面要花三倍力气才能拉回注意力。

## 提前处理紧张、提问和现场突发

不要把临场问题留到现场。

正式上台前至少做三件事：按真实节奏讲一遍，找出会卡壳的段落；列五到十个最可能被问的问题；预想被质疑、被打断时怎么拉回主线。

怯场明显的，不要靠"告诉自己别紧张"。靠熟悉结构、提前走台词、缩短句子、控制起步速度来减压。紧张不能靠意志消失，但能靠准备降级。

## 上台前最后检查现场条件

最后一轮检查不是补内容，是确认环境和节奏。

- 时长和实际一致吗
- 投影、翻页、麦克风有没有问题
- 哪段超时最容易拖垮整体
- 时间被压缩的话，哪部分可以立刻删

准备一版压缩方案。压缩版只保留"结论—两点依据—行动"。有了这版垫底，突发压时不会慌。

## 场景不同，重心不同

向上汇报：结论和影响放前面，不先铺完整背景。培训分享：多安排例子、提问和参与节点，别一直单向输出。说服型演讲：重视听众动机和顾虑，别只讲自己要表达什么。即兴发言：最少想清目标、一句结论和一个结束句。

不管什么场景，任务卡、结构压缩和问答预案这三步都不能省。

**任务定义**

- [ ] 压成两到四个主点

- [ ] 留下一页提纲

**关键段落**
- [ ] 开场单独练过
- [ ] 结尾单独练过
- [ ] 改掉套话和空话

- [ ] 按真实节奏模拟一遍
- [ ] 预判五到十个问答
- [ ] 准备压缩版方案
- [ ] 检查时长和现场条件
