{{< sentence >}}
回复邮件的速度，就是你在对方心里的响应速度。
{{< /sentence >}}

你可能觉得"晚两小时回也没什么"。但发邮件的人在等你的那两小时里，已经默默把你的响应优先级降了一档。不需要立刻给出答案——但"收到，我看看"这五个字，三分钟内发出去，效果和两小时后回复完全不一样。

{{< sentence >}}
发消息不要只写"在吗"。
{{< /sentence >}}

"在吗"后面通常跟的是一个需要对方花时间处理的请求。但你先发一个"在吗"，等对方回"在"，你再发请求——你多花了对方一轮等待时间。直接写"在吗？想确认一下周五的会议是在A还是B会议室"。

{{< sentence >}}
你交出去的东西，就是你在别人眼里的样子。
{{< /sentence >}}

一份排版乱七八糟的文档、一封没有主题行的邮件、一个文件名叫"未命名(3).docx"的附件——每一件都在说"我不太在意质量"。不是说要完美，是最低限度的整洁。

{{< sentence >}}
开会迟到两分钟和迟到十分钟，在别人眼里是一样的：迟到了。
{{< /sentence >}}

你觉得"就迟了一点点"。但对已经坐在会议室里的人来说，迟到就是迟到，两分钟和十分钟的区别只是"这个人不守时的程度不同"。提前到是唯一不扣分的选项。

{{< sentence >}}
领导说"不急"，不是真的不急——是给你留面子。
{{< /sentence >}}

信太明提醒了一个新人最容易踩的坑：领导说"这个不着急，你有空做就行"。然后你真的等"有空"了再做——一周后领导来问进度，你说还没开始。在领导心里，这件事的优先级就是你对工作态度的体现。

{{< sentence >}}
做笔记不是为了以后看，是为了让对方看到你在认真听。
{{< /sentence >}}

很多人觉得自己记忆力好、不用记笔记。但在会议或谈话中拿出本子记录这个动作本身，传递的信号是："我把你说的话当回事。"不管你以后翻不翻那个本子。

{{< sentence >}}
向别人请教之前，先花五分钟自己查一下。
{{< /sentence >}}

问"这个表格的公式怎么写"之前，先搜一下——如果搜得到，你省了别人的时间，也避免了被贴上"不会自己解决问题"的标签。请教的正确打开方式是："我查了一下，试了A和B两种方法都不行，您看是不是应该用C？"
