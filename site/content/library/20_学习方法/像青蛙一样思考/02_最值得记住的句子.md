{{< sentence >}}
先看见发生了什么，再解释为什么发生。
{{< /sentence >}}

很多错判断不是智力问题，是顺序问题。你一上来就解释，后面看到的事实都会被解释牵着走。

{{< sentence >}}
一个反常的小细节，往往比一套顺滑的大道理更值钱。
{{< /sentence >}}

顺滑的解释很容易让人舒服，异常点才逼你修模型。真正推进思考的，通常不是一致，而是卡住你的那一下。

{{< sentence >}}
离现场越远，越容易把想象当成理解。
{{< /sentence >}}

在桌面上讨论问题很容易，在一线看问题才知道限制条件是什么。很多“早该想到”的建议，离现场最近的人最知道为什么做不到。

{{< sentence >}}
比较不是找谁对谁错，是看不同条件会把结果推向哪里。
{{< /sentence >}}

只比结果很容易流于态度，连同起点、约束和路径一起比，结论才不至于太轻。

{{< sentence >}}
你最熟的解释，往往也是最该晚一点相信的解释。
{{< /sentence >}}

熟悉感会伪装成正确感。越是你一眼就觉得“这肯定就是这样”，越要多找一个反面样本。

{{< sentence >}}
好的思考不是更快下结论，而是更早发现自己可能看错了。
{{< /sentence >}}

如果一个思考过程没有留下修正入口，它最后通常只会变成自我确认。给自己留退路，才有继续接近事实的机会。
