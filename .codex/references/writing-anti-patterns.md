# 写作反模式

这份参考只处理一个问题：哪些表达会让正文从“在说内容”滑成“在说内容的包装、使用方式或生产流程”。

它不是禁词表，也不是要把文字写得越来越僵。核心判断只有一句：

如果一句话的重心落在文稿类型、使用方式、加工动作，而不是落在判断、问题、边界或动作本身，这句话就值得高度警惕。

默认先按 7 类看。

## 1. 元说明腔

这类句子在解释“我正在写什么”，不在推进内容。

常见表达：

- `这份指南适合用在这样的时候`
- `这篇文章会`
- `这一节会先……再……`
- `这页主要回答`
- `这里重点讲`

为什么让人烦：

- 重心落在“文档功能”，不落在“问题本身”
- 写作者站在说明文位置，不在内容内部
- 很像 README、说明书或提示词

替换方向：

- 直接写情境，不介绍文档功能
- 直接写问题，不介绍结构角色

改写前：

```md
这份指南适合用在这样的时候：团队开始只会谈目标、经验和竞品动作。
```

改写后：

```md
当团队开始只会谈目标、经验和竞品动作，真正关于顾客变化的判断往往已经缺席了。
```

## 2. 内容生产腔

这类词先暴露加工流程，再暴露内容。

常见表达：

- `把……压成……`
- `拆成`
- `转成`
- `沉淀成`
- `判断稿`
- `执行稿`
- `入口稿`
- `结构总结稿`

为什么让人烦：

- 像在汇报内容生产流程，不像成稿语言
- 让读者先看到“工序”，看不到“判断”
- 容易带出 prompt、工单、流水线味

注意：

`压成` 不是绝对禁词。它在内容内部有时成立，例如“把主观抱怨压成客观延时”。真正要警惕的是下面这种写法：

- 把书压成什么
- 把一页压成什么
- 把产物压成什么

也就是句子的重心落在“加工动作”和“产物类型”，不是落在“问题如何被处理”。

固定槽位正文里再多加一条硬规则：

- 把书压成什么
- 把材料压成什么
- 把这一页压成什么
- 把现有内容收束成一句什么

这几类在正式正文里默认都算坏句，不用再看它有没有“部分推进内容”。因为一旦写出来，读者先看到的就是加工动作，不是判断。

还有一类要一起处理：

- `从现有材料里能看出`
- `按当前可见材料重构`
- `基于现有材料可以推到`

这类句子表面在交代边界，实际是在把 frontmatter 该做的工作塞进正文。

替换方向：

- 少说文稿类型，多说读者会得到什么
- 少说加工动作，多说完成后的判断或动作

改写前：

```md
把《零售的哲学》压成一轮可执行零售诊断。
```

改写后：

```md
用一轮小规模验证，把顾客变化、经营假设和现场反馈重新接起来。
```

再看一个固定槽位里更常见的坏句：

改写前：

```md
从现有材料里能稳定看出的，是一条反复出现的经营顺序。
```

改写后：

```md
零售经营真正反复出现的，是一条稳定的经营顺序。
```

## 3. 导购分群腔

这类句子像推荐页，不像分析稿。

常见表达：

- `适合谁`
- `更适合哪种人`
- `哪些人适合读`
- `哪种人现在最该读`

为什么让人烦：

- 先做读者分群，再谈内容判断
- 有导购、推荐、口播转场味
- 容易把动态问题写成静态人群标签

替换方向：

- 写“什么问题状态下它有用”
- 写“什么卡点出现时该想起它”

改写前：

```md
## 更适合哪种人现在读
```

改写后：

```md
## 如果你缺的不是动作，而是判断顺序
```

## 4. 模板化 AI 腔

这类表达单看不一定重，但一连排就很像模型在转述。

常见表达：

- `这本书讲了`
- `作者介绍了`
- `它会让你`
- `值得注意的是`
- `需要指出的是`
- `不难发现`
- `首先 / 其次 / 最后`

为什么让人烦：

- 段落像在转述目录，不像在下判断
- 节奏像标准答案，不像自然中文
- 主语总在绕内容走，不直接落到内容上

替换方向：

- 直接说结论
- 直接说误判
- 直接说读者会遇到的真实问题

## 5. 空心评价壳

这类句子看起来像在下判断，其实只是在给下一句搭台。

常见表达：

- `最耐读的一条线`
- `真正能穿出具体历史处境的`
- `反复纠正的是`
- `最怕的错法`
- `真正厉害的地方`

为什么让人烦：

- 句子还没落到具体内容，只先宣布“后面很重要”
- 写作者站在讲评席上，不在内容内部
- 同一套句壳可以套任何书、任何段落，说明它本身没有信息量

替换方向：

- 直接说那条线是什么
- 直接说误判是什么
- 直接说动作和边界，不先宣布“这里很重要”

改写前：

```md
卷一最耐读的一条线，是它对书本崇拜和空洞推演的持续警惕。
```

改写后：

```md
卷一一直在拆书本崇拜和空洞推演。
```

再看一个：

改写前：

```md
卷一最怕的错法，是材料太薄就先下重结论。
```

改写后：

```md
材料太薄就先下重结论，是最常见的走偏起点。
```

## 6. 结构壳标题

标题在说“这是哪一栏”，不是在说“这一栏的判断是什么”。

常见表达：

- `为什么读这本书`
- `主要价值`
- `本章定位`
- `问题驱动`
- `一句话总结`
- `小结`
- `概述`

为什么让人烦：

- 读者看完标题仍然不知道这节在说什么
- 标题管理的是结构，不是内容
- 容易把文章写成模板填空

替换方向：

- 直接把标题写成判断
- 让读者只看标题就知道这节在说什么

## 7. 内部简称主语腔

这类句子把卷次、章节号、槽位号、目录代号推到句首当主语，读起来像工作笔记，不像成稿。

常见表达：

- `卷一里最值得反复调出来的`
- `卷一本来处理的是`
- `这也是为什么卷一的案例价值`
- `第一章真正厉害的地方`
- `03 这页最值钱的是`

为什么让人烦：

- 把定位编号当正文主语，像工作笔记，不像成稿
- 主语语义太薄，承受不了抽象判断
- 一旦再叠加 `最值得`、`真正`、`价值`、`地方` 这类评价壳，句子会同时发空、发硬

替换方向：

- 能省主语就省
- 需要指代时改成自然对象，如 `这一卷文章`
- 直接说内容，不先说“它最值钱的地方”

改写前：

```md
卷一里最值得反复调出来的，是几类会不断重演的误判。
```

改写后：

```md
最值得反复调出来的，是几类会不断重演的误判。
```

改写前：

```md
卷一本来处理的是社会力量、组织路线和现实条件判断。
```

改写后：

```md
这一卷文章本来就在处理社会力量、组织路线和现实条件判断。
```

改写前：

```md
这也是为什么卷一的案例价值，不在戏剧性，而在可调用性。
```

改写后：

```md
这些案例的价值不在戏剧性，而在可调用性。
```

## 使用方式

这份参考默认这样用：

1. 先判断一句话的问题属于哪一类
2. 再问重心落在内容，还是落在包装、说明和加工
3. 如果后者更重，就优先改写

## 门禁

下面这些命中任一条，默认不能算过稿，继续改：

1. 段首句仍在解释文档功能，而不是直接推进判断
2. 正文开头还在交代工序边界，例如 `基于……提炼`、`不是逐字引文`
3. 段首句还是空心评价壳，例如 `最耐读的一条线`、`最怕的错法`、`真正厉害的地方`
4. 小标题还是结构壳，读者看完标题仍不知道这一节在说什么
5. 正文高频拿 `卷一`、`第一章`、`03` 这类内部简称顶在句首承载抽象判断
6. 同一类句壳可以无损替换到别的书上，说明这句话本身没有信息量

适合放进 `AGENTS.md` 的，是这些类别本身。

适合放进具体 skill 的，是某个槽位最容易犯的具体句式和替换方式。
