## 关于优先级

{{< sentence >}}
不要问"今天必须做什么"，要问"今天做完什么对这个月的结果影响最大"。
{{< /sentence >}}

紧急的事会自己跳出来抢你的注意力。重要的事不会。如果你的一天是被"必须做"驱动的，到月底你会发现时间全花在灭火上，没有一件推动了核心目标。

{{< sentence >}}
你做的事有一半不值得做，但你觉得每一件都不可或缺。
{{< /sentence >}}

感觉"什么都重要"本身就是效率低的信号。如果你真的对每件事都做了"做完对结果影响多大"的判断，至少有一半会被降级。波曾的建议是每周日晚上花十五分钟列出下周三件最重要的事——只许三件。

## 关于时间分配

{{< sentence >}}
把你最好的时间段给最重要的任务，而不是给最先到来的任务。
{{< /sentence >}}

多数人的上午——精力最好的时段——是被邮件和消息占掉的。波曾的做法是每天前两个小时不开邮箱，先做当天最重要的那件事。邮件不会因为晚两小时回就出事，但你的深度思考能力只在上午最充沛的那几个小时里才好用。

{{< sentence >}}
短任务不是碎片时间做的，是批量处理的。
{{< /sentence >}}

回邮件、签审批、处理日常消息——这些五分钟以内能完成的事，不要穿插在大任务之间做。每次切换都有认知成本。波曾的方案是每天设两到三个固定时段集中处理这些短任务。

## 关于阅读和写作

{{< sentence >}}
任何文件，先读最后一段——那里通常有结论。结论不值得深究的，整份文件就不需要细读。
{{< /sentence >}}

职场里百分之七十以上的阅读是可以跳过的。问题是你不知道哪些能跳，所以全读了。波曾给了一个过滤器：先看结论，结论跟你的工作无关或不影响你的判断，就停在这里。

{{< sentence >}}
写东西的第一句就写结论。展开只做一件事：让结论更可信。
{{< /sentence >}}

跟金字塔原理的结论先行一脉相承，但波曾的表述更极端——不只是报告和汇报，连邮件也这么写。"建议我们改用 B 方案，因为……"比"关于 A 和 B 方案的比较分析"的沟通效率高三倍。

## 关于会议

{{< sentence >}}
没有明确议题和预期产出的会议，不值得参加。
{{< /sentence >}}

你每周有多少会议是"了解一下情况""同步一下进展"？波曾的判断标准：会议邀请里如果不写"这次会要解决什么问题、结束时要产出什么决定"，直接拒绝或要求补充信息后再决定。

{{< sentence >}}
会议的成本不是你坐在那里的一个小时，是你被打断的整个上午。
{{< /sentence >}}

一个一小时的会放在上午十点，实际上毁掉的是九点到十二点的整块时间——九点不敢开始大任务，十一点已经没剩多少深度工作时间。会议的真实成本远高于会议本身的时长。
