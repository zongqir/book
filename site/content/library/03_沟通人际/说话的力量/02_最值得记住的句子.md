# 开口之前先读一下对方的状态

## 判断先于开口

{{< sentence no="1" quote="说话的效果由听的人的反应决定，不由说的人的意图决定。" >}}
你觉得自己说清楚了，不等于对方听明白了。衡量一句话有没有用，看对方说完后的动作，不看你自己的感觉。
{{< /sentence >}}

{{< sentence no="2" quote="开口之前先判断对方是在听还是在等你说完。" >}}
对方眼神飘了、点头加速、身体转向别处——大概率在等你停嘴。这时候每多说一句，都在消耗你的信用。
{{< /sentence >}}

{{< sentence no="3" quote="你觉得自己说清楚了，是最危险的错觉。" >}}
说的人按自己的逻辑觉得完整。听的人用自己的框架在理解。中间那道缝隙只有你主动去确认才能发现。
{{< /sentence >}}

## 说服从回应开始

{{< sentence no="4" quote="先回应对方的担心，再说你的方案。" >}}
顺序一换，同样的方案落地效果完全不同。对方的担心没被接住之前，他没有空间去听你的想法。
{{< /sentence >}}

{{< sentence no="5" quote="提问比陈述更容易让对方自己想通。" >}}
陈述给压力，提问给空间。一句"你觉得最大的风险是什么"比十句"我觉得这样做更好"有穿透力得多。
{{< /sentence >}}

{{< sentence no="6" quote="说服力来自你说的恰好是对方需要听的。" >}}
音量和气势都是错误的放大器。对方只会被一种话打动——他正在想的问题，你替他说出来了。
{{< /sentence >}}

## 停下来才能听到回声

{{< sentence no="7" quote="说完一句话停一下，看对方什么反应再决定下一句。" >}}
大多数人说话像打连发，一口气把想说的全倒出去。停顿是最被低估的说话工具——停下来，你才能看见对方有没有跟上。
{{< /sentence >}}

{{< sentence no="8" quote="安慰人的第一句话不是建议，是重复对方刚说的。" >}}
朋友说"最近工作压力太大了"。如果你马上接"要不要换个方式休息"，他会觉得你没听。先说"压力确实大"——让他知道你接住了。建议以后再说也来得及。
{{< /sentence >}}
