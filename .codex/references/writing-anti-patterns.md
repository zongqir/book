# 写作反模式

这份参考只处理一个问题：哪些表达会让正文从“在说内容”滑成“在说内容的包装、使用方式或生产流程”。

它不是禁词表，也不是要把文字写得越来越僵。核心判断只有一句：

如果一句话的重心落在文稿类型、使用方式、加工动作，而不是落在判断、问题、边界或动作本身，这句话就值得高度警惕。

默认先按 7 类看。

## 1. 元说明腔

这类句子在解释“我正在写什么”，不在推进内容。

常见表达：

- `这份指南适合用在这样的时候`
- `这篇文章会`
- `这一节会先……再……`
- `这页主要回答`
- `这里重点讲`

为什么让人烦：

- 重心落在“文档功能”，不落在“问题本身”
- 写作者站在说明文位置，不在内容内部
- 很像 README、说明书或提示词

替换方向：

- 直接写情境，不介绍文档功能
- 直接写问题，不介绍结构角色

改写前：

```md
这份指南适合用在这样的时候：团队开始只会谈目标、经验和竞品动作。
```

改写后：

```md
当团队开始只会谈目标、经验和竞品动作，真正关于顾客变化的判断往往已经缺席了。
```

## 2. 内容生产腔

这类词先暴露加工流程，再暴露内容。

常见表达：

- `把……压成……`
- `拆成`
- `转成`
- `沉淀成`
- `判断稿`
- `执行稿`
- `入口稿`
- `结构总结稿`

为什么让人烦：

- 像在汇报内容生产流程，不像成稿语言
- 让读者先看到“工序”，看不到“判断”
- 容易带出 prompt、工单、流水线味

注意：

`压成` 不是绝对禁词。它在内容内部有时成立，例如“把主观抱怨压成客观延时”。真正要警惕的是下面这种写法：

- 把书压成什么
- 把一页压成什么
- 把产物压成什么

也就是句子的重心落在“加工动作”和“产物类型”，不是落在“问题如何被处理”。

固定槽位正文里再多加一条硬规则：

- 把书压成什么
- 把材料压成什么
- 把这一页压成什么
- 把现有内容收束成一句什么

这几类在正式正文里默认都算坏句，不用再看它有没有“部分推进内容”。因为一旦写出来，读者先看到的就是加工动作，不是判断。

还有一类要一起处理：

- `从现有材料里能看出`
- `按当前可见材料重构`
- `基于现有材料可以推到`

这类句子表面在交代边界，实际是在把 frontmatter 该做的工作塞进正文。

替换方向：

- 少说文稿类型，多说读者会得到什么
- 少说加工动作，多说完成后的判断或动作

改写前：

```md
把《零售的哲学》压成一轮可执行零售诊断。
```

改写后：

```md
用一轮小规模验证，把顾客变化、经营假设和现场反馈重新接起来。
```

再看一个固定槽位里更常见的坏句：

改写前：

```md
从现有材料里能稳定看出的，是一条反复出现的经营顺序。
```

改写后：

```md
零售经营真正反复出现的，是一条稳定的经营顺序。
```

## 3. 导购分群腔

这类句子像推荐页，不像分析稿。

常见表达：

- `适合谁`
- `更适合哪种人`
- `哪些人适合读`
- `哪种人现在最该读`

为什么让人烦：

- 先做读者分群，再谈内容判断
- 有导购、推荐、口播转场味
- 容易把动态问题写成静态人群标签

替换方向：

- 写“什么问题状态下它有用”
- 写“什么卡点出现时该想起它”

改写前：

```md
## 更适合哪种人现在读
```

改写后：

```md
## 缺的不是动作，而是判断顺序
```

## 4. 模板化 AI 腔

这类表达单看不一定重，但一连排就很像模型在转述。

常见表达：

- `这本书讲了`
- `作者介绍了`
- `它会让你`
- `如果你……这本书值得……`
- `值得注意的是`
- `需要指出的是`
- `不难发现`
- `首先 / 其次 / 最后`

为什么让人烦：

- 段落像在转述目录，不像在下判断
- 节奏像标准答案，不像自然中文
- 主语总在绕内容走，不直接落到内容上

替换方向：

- 直接说结论
- 直接说误判
- 直接说读者会遇到的真实问题

## 5. 空心评价壳

这类句子看起来像在下判断，其实只是在给下一句搭台。

常见表达：

- `最耐读的一条线`
- `真正能穿出具体历史处境的`
- `反复纠正的是`
- `最怕的错法`
- `真正厉害的地方`

为什么让人烦：

- 句子还没落到具体内容，只先宣布“后面很重要”
- 写作者站在讲评席上，不在内容内部
- 同一套句壳可以套任何书、任何段落，说明它本身没有信息量

替换方向：

- 直接说那条线是什么
- 直接说误判是什么
- 直接说动作和边界，不先宣布“这里很重要”

改写前：

```md
卷一最耐读的一条线，是它对书本崇拜和空洞推演的持续警惕。
```

改写后：

```md
卷一一直在拆书本崇拜和空洞推演。
```

再看一个：

改写前：

```md
卷一最怕的错法，是材料太薄就先下重结论。
```

改写后：

```md
材料太薄就先下重结论，是最常见的走偏起点。
```

## 6. 结构壳标题

标题在说“这是哪一栏”，不是在说“这一栏的判断是什么”。

常见表达：

- `为什么读这本书`
- `主要价值`
- `本章定位`
- `问题驱动`
- `一句话总结`
- `小结`
- `概述`

为什么让人烦：

- 读者看完标题仍然不知道这节在说什么
- 标题管理的是结构，不是内容
- 容易把文章写成模板填空

替换方向：

- 直接把标题写成判断
- 让读者只看标题就知道这节在说什么

## 7. 内部简称主语腔

这类句子把卷次、章节号、槽位号、目录代号推到句首当主语，读起来像工作笔记，不像成稿。

常见表达：

- `卷一里最值得反复调出来的`
- `卷一本来处理的是`
- `这也是为什么卷一的案例价值`
- `第一章真正厉害的地方`
- `03 这页最值钱的是`

为什么让人烦：

- 把定位编号当正文主语，像工作笔记，不像成稿
- 主语语义太薄，承受不了抽象判断
- 一旦再叠加 `最值得`、`真正`、`价值`、`地方` 这类评价壳，句子会同时发空、发硬

替换方向：

- 能省主语就省
- 需要指代时改成自然对象，如 `这一卷文章`
- 直接说内容，不先说“它最值钱的地方”

改写前：

```md
卷一里最值得反复调出来的，是几类会不断重演的误判。
```

改写后：

```md
最值得反复调出来的，是几类会不断重演的误判。
```

改写前：

```md
卷一本来处理的是社会力量、组织路线和现实条件判断。
```

改写后：

```md
这一卷文章本来就在处理社会力量、组织路线和现实条件判断。
```

改写前：

```md
这也是为什么卷一的案例价值，不在戏剧性，而在可调用性。
```

改写后：

```md
这些案例的价值不在戏剧性，而在可调用性。
```

## 8. 长句与术语过密

这类句子把太多信息塞进一句话，或者把多个术语挤在一起，让读者必须反复回看才能理解。

常见表现：

- 单句超过 40 字，逗号接逗号不断句
- 同一句里 4 个以上专业词汇连续出现，没有日常语言穿插
- 术语首次出现时没有自解释，默认读者已经懂了

为什么让人累：

- 读者需要在脑子里同时保持 3-4 个概念才能走完这句话
- 术语堆叠让外行直接放弃，让内行觉得在读教科书
- 句子越长，读者走神的概率越高

替换方向：

- 把 40 字以上的句子拆成 2-3 个短句，每句只处理一个概念
- 术语第一次出现时，紧跟一句日常语言解释
- 同一句里最多 2-3 个专业词，多了就拆句

改写前：

`md
债务与金融机制把矛盾向后延时，把成本向下转移，让系统短期维持扩张，但被延后的矛盾会在某些阶段以危机形式集中兑现。
`

改写后：

`md
债务和金融工具能把问题往后拖。短期看起来一切正常，扩张还在继续。

但被拖住的问题不会消失。它会在某个时点集中爆发——这就是危机。
`

## 9. 段落节奏均匀

这类文章每段长度差不多，读起来像在看同一尺寸的砖块码在一起。

常见表现：

- 连续 4 段以上都是 3-5 行
- 没有 1-2 行的短段做节奏变化
- 每段都是 "判断 + 解释 + 举例" 的三段式

为什么让人闷：

- 节奏均匀意味着没有重点
- 读者找不到“哪句话最重要”
- 视觉上没有停顿点，眉毛胡子一把抓

替换方向：

- 关键判断后面插一个 1-2 行的短段收住
- 偶尔用单句成段，让那句话单独站住
- 允许短段（1-2行）和长段（4-6行）交替出现

改写前（均匀节奏）：

`md
第一部分先谈学习态度。作者认为态度是方法生效的前提，
不是鸡汤，而是地基。没有对的态度，后面的技巧连开始的机会都没有。

第二部分是全书核心。作者把学习设计成一连串问题，
而不是抽象概念。这把学习从被动吸收改成了主动发问。

第三部分处理知识分类。不同类型的知识要用不同的
记忆策略，不能一招鲜吃遍天。这正是很多人学习无效的原因。
`

改写后（长短交替）：

`md
态度不是鸡汤。它是地基。

作者把态度放在第一部分，是因为没有对的态度，后面的技巧连开始的机会都没有。

全书最核心的设计是第二部分。

学习不是被动吸收——而是主动发问。作者设计了一连串问题，让你带着问题进入每个知识点。

第三部分说的是：不同类型的知识，记法不一样。一招鲜吃遍天的学习方法不存在。
`


## 使用方式

这份参考默认这样用：

1. 先判断一句话的问题属于哪一类
2. 再问重心落在内容，还是落在包装、说明和加工
3. 如果后者更重，就优先改写

## 门禁

下面这些命中任一条，默认不能算过稿，继续改：

1. 段首句仍在解释文档功能，而不是直接推进判断
2. 正文开头还在交代工序边界，例如 `基于……提炼`、`不是逐字引文`
3. 段首句还是空心评价壳，例如 `最耐读的一条线`、`最怕的错法`、`真正厉害的地方`
4. 小标题还是结构壳，读者看完标题仍不知道这一节在说什么
5. 正文高频拿 `卷一`、`第一章`、`03` 这类内部简称顶在句首承载抽象判断
6. 同一类句壳可以无损替换到别的书上，说明这句话本身没有信息量

适合放进 `AGENTS.md` 的，是这些类别本身。

适合放进具体 skill 的，是某个槽位最容易犯的具体句式和替换方式。
7. 单句超过 35 字且没有用分号或破折号拆开
8. 同一句里超过 3 个专业词汇且没有自解释
9. 连续 3 段以上长度均匀，没有短段做周期变化
10. `不是…而是…` 及其变体（`不只是…更是…`、`并不是…其实是…`、`不在于…在于…`）全篇超过 3 次
11. 一个判断后面跟了 2 层以上防御性补充（常见误读 + 边界 + 约束连排）
12. 同一句里 4 个以上并列项没有分组

## 10. 反向建构过密

这类句式强迫读者先建一个理解再推翻它，反复使用会让读者一直在"被纠正"的状态里。

常见表达：

- `不是 A，而是 B`
- `不只是 A，更是 B`
- `并不是 A，其实是 B`
- `不在于 A，在于 B`
- `X 之所以 Y，不是因为 A，而是因为 B`
- `与其说 A，不如说 B`

为什么让人累：

- 读者必须先在脑子里装入 A，再否定 A，再装入 B——一次认知负担翻三倍
- 偶尔用一次有纠偏效果，每段都用就变成句式模板
- 连排三次以上，读者会觉得作者在跟一个想象中的对手辩论，而不是在跟自己说话

硬指标：

- 全篇不超过 3 次
- 连续两段不能都出现
- 超过 3 次的，优先替换成直接陈述 B

替换方向：

- 直接说 B，删掉对 A 的否定
- 如果 B 写得足够清楚，读者自然能感知到它跟常见看法不同

改写前：

```md
问题通常不在书不够好，也不在你读得不够多。更常见的卡点，是"读过"和"读懂"一直没分开。
```

改写后：

```md
大多数人的阅读卡点在同一个地方："读过"和"读懂"一直没分开。
```

## 11. 防御性写作

这类段落在每个判断后面堆两三层"防止被误读"的补充，读者不是在接收内容，而是在旁听作者自辩。

常见表达：

- `这条原则约束的是…`
- `常见误读是把它读成…`
- `边界在于…`
- `当然，这不是说…`
- `不要把 A 理解成 B`

为什么让人累：

- 它的隐含假设是"你很可能会理解错"，读者被反复当作需要纠正的人
- 判断 + 误读 + 边界 + 约束的四件套，让段落膨胀到核心意思反而沉下去
- 读者的注意力被拉到防守层上，主判断的力量被稀释

硬指标：

- 一个判断后面最多跟一层补充
- `常见误读是…` 全篇最多出现 1 次
- 如果觉得需要两层以上防守，先回去改判断本身

替换方向：

- 把边界条件写进判断本身，而不是判断之后再加补丁
- 砍掉"常见误读"层——如果判断本身写清楚了，读者不会误读

改写前：

```md
理论只有回到调查和实践里，才有资格继续被相信。这条原则约束的是书本替代调查、概念替代接触、引用替代理解。常见误读是把它读成"经验比理论高明"，于是顺手滑到反智。其实它约束的是脱离现实的理论，不是否定理论本身。
```

改写后：

```md
理论当然需要。但理论只有在现实里碰过、被修正过，才从借来的确定感变成自己的判断力。一旦现实给出反证，你还只拿原来的概念去压，理论就已经僵了。
```

## 12. 并列项过载

一句话里 4 个以上并列项堆在一起，没有分组，读者的工作记忆必须同时 hold 住全部项目才能走完这句话。

常见表达：

- `选址、配送、订货、单品管理、信息系统、门店沟通，这些…`
- `准备太晚、话说太满、开头空、结尾弱、互动差、怯场重、遇到提问发懵…`
- `先…再…随后…接着…然后…最后…`

为什么让人累：

- 工作记忆容量大约是 3-4 个单元，超过以后读者开始丢信息
- 并列项越多，读者越不确定哪个更重要
- 到第 5 个以后，大脑开始做模式匹配而不是语义处理

硬指标：

- 同一句里并列项不超过 3 个
- 超过 3 个的，先分组再展开

替换方向：

- 把 6 个并列项按内在逻辑分成 2-3 组，每组起一个角色名
- 或者拆成几句话，每句只列 2-3 个

改写前：

```md
准备太晚、话说太满、开头空、结尾弱、互动差、怯场重、遇到提问发懵，这些都不是抽象问题。
```

改写后：

```md
准备和结构上的问题——准备太晚、开头空、结尾弱。现场表现上的问题——话说太满、互动差、怯场重、提问接不住。

哪一类都不抽象，都能单独练。
```
