# 任务质量评分标准

## 好任务的 5 条硬标准

1. **够短**：一条任务只对应一个动作。"读目录并摘录并写草案"是三件事，应该拆成三条。
2. **够具体**：读完标题就知道要做什么。"继续整理"不合格，"把 chunk 策略判断收敛成规范草案的检索层规则"合格。
3. **完成可判断**：完成标记必须指向可观测的结果。"有更多理解"不可判断，"关键段落已摘录到 exports/02_key-quotes.md"可判断。
4. **绑定文件路径**：每条任务至少关联一个仓库文件，做完知道回写到哪里。
5. **不靠隐形上下文**：拿掉上下文对话，单看这条任务也能执行。

## 常见坏味道

| 坏味道 | 为什么差 | 改法 |
|--------|----------|------|
| `研究一下 RAG` | 太大，不知道从哪开始 | 拆成具体的阅读、核对、收敛动作 |
| `做一轮 NotebookLM` | 没有问题焦点 | 写清楚围绕什么问题核对 |
| `完成这个专题` | 口号，不是任务 | 拆成当前轮最短的 4 条闭环动作 |
| `深入理解向量检索` | 完成标记不可判断 | 改成可观测产出，如"摘录 3 条关键对比" |
| `整理输出` | 没有文件路径 | 写清回写到哪个文件的哪个部分 |
| `读完这本书` | 太大，没有具体产出 | 拆成浏览目录标出 X 章、摘录 Y 段 |

## 闭环检查

- 四类任务（reading / research / spec / review）是否至少各一条
- 是否有回写动作落回仓库
- 下一轮动作是否已经写进 `notes/next-actions.md`
- 依赖关系是否标清楚

## 完成标记的好坏判断

好的完成标记：
- `关键段落已摘录到 exports/02_key-quotes.md`
- `规范条目已写，每条有验收标准`
- `差异点列表已写，每条标明来源`

差的完成标记：
- `有更多理解`
- `已经深入研究`
- `进展顺利`
