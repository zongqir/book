# 先判断再深读，读完一定要落盘

## 拿到新书，不想再盲读的时候

你拿到一本新书，不想再从第一页机械读到最后一页的时候，可以打开这份指南。

尤其是这几种情况：

- 经常读完以后只剩零散印象，复述不出作者主张
- 笔记摘了很多，但说不清这本书的结构和论证
- 想把单本书阅读接进自己的书库或专题

如果只是随手翻翻，不需要跑完整流程。

## 先跑这四步

只想先跑一版最小闭环，做这四步：

1. 花十分钟看目录、序言和结尾，写下"这本书解决什么问题"和"值不值得深读"。
2. 第一遍只做检视阅读，标出关键章节和暂时不懂的问题，不在细节上卡住。
3. 第二遍只深读关键章节，写出三个核心主张和主干论证。
4. 读后落一版本地文档，至少留下主题判断、核心观点、个人判断和下一步动作。

跑完这四步，一轮阅读就算闭环了。后面是在这个基础上展开。

## 从判断到落盘

### 先花十分钟判断值不值得认真读

不要急着读正文。先看这几个地方：书名和副标题、目录、序言、封底、结尾。

然后回答四个问题：这本书在解决什么问题？对我当前有什么直接价值？值得深读还是只需要知道大意？如果只读 20%，最该读哪几章？

至少留下三行预判记录：我为什么读这本书、我预计它能帮我解决什么、我准备读到什么程度。写不出来的话，不要开始精读。

大多数人效率低不是因为不努力，而是因为跳过了这一步——还没搞清值不值得就已经扎进细节了。

### 完成一轮检视阅读，建全书地图

这一轮不是为了读懂细节，而是建立全书地图。

按目录走一遍，给每章写一句用途判断。快速翻读每章开头和结尾。遇到不懂的先标记，不当场深挖。抓重复出现的关键词和概念。

这一轮结束后，至少能写出：核心问题是什么、作者怎么展开的、哪几章最重要、第二轮重点看什么。写不出来说明检视阅读还没做完。

### 进入分析阅读，把书拆开

这时才开始认真读。重点不是摘更多句子，而是把书真正拆开。

盯住四件事：作者到底在主张什么、为什么这么主张、怎样组织论证、哪里成立哪里可能不成立。

读重点章节时反复问自己：这一章最核心的结论是什么？用什么理由支撑？和全书其他部分什么关系？用自己的话说，我会怎么表达？

这一阶段最少留下三类东西：三个核心主张、一组关键概念、一段主干论证摘要。如果只能摘句子不能复述结构，通常说明还没真正读懂。

### 理解站稳了再评价

不要太早批评。真正有效的评价至少满足两个前提：你能准确复述作者的观点，你知道他的结论怎样推出来的。

然后再写自己的判断：认同什么、保留什么、反对什么。

能区分"我不同意"和"我其实还没读懂"，这一步就算站住了。对于明显信息垃圾的文本，快速识别并退出比硬做完整分析更合理。

### 落盘成文档和下一步动作

不落盘的阅读很快蒸发。

读完一本值得保留的书，至少留下：这本书在解决什么问题、三个最重要的观点、作者怎么论证的、我的判断、我的下一步动作。

后续要确认原书怎么说或做多书对照，再接 NotebookLM 做贴源。已经进入行动层的内容，拆到 dida-cli。

## 不是每本书都走完整流程

- 入门书或概览书：缩短分析阅读，重点放在结构地图和三个核心观点。
- 难书：不要在第一遍查完所有问题，先完成一轮粗读再决定哪些值得深读。
- 后续要做多书比较或确认原文：再接 NotebookLM，不要一开始就把工具当前置。
- 已经明确关联某个专题：在最后一步直接写明要进入哪个专题目录，不要停在"以后再说"。

## 跑完后手里至少有这五样

- 一句话说明这本书在解决什么问题
- 三个最重要的主张
- 一段主干论证摘要
- 一段个人判断
- 一条明确的下一步动作

做到什么程度才算完成：你能脱离原书，用自己的话向别人讲清楚这本书的核心主张和论证结构，并且说得清自己的判断和下一步打算。如果只是"读过了，有印象"，还不算完成。

## 一页清单

### 读前

- 看目录、序言、结尾
- 写清阅读目的
- 判断值不值得深读

### 第一遍

- 建立全书地图
- 标记重点章节
- 记录关键词和疑问

### 第二遍

- 找三个核心主张
- 找关键概念
- 找主干论证
- 用自己的话复述

### 读后

- 写个人判断
- 写下一步动作
- 需要时接 NotebookLM 或 dida-cli
