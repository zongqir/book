# 所有问题七步解决：最值得记住的句子

## 问题定义和推进节奏

{{< sentence >}}问题定义写不成一句话，说明你还没准备好开始分析。{{< /sentence >}}

七步法的第一步就是最硬的门槛。很多团队把"我们大概知道问题是什么"当成起点，但"大概知道"和"能写成一句带边界的陈述"之间，差的不是文字功夫，是对问题的深层理解。写不出来就先别动手。

{{< sentence >}}在前三步多花一天，能在后四步省一周。{{< /sentence >}}

定义、拆解、排优先级——这三步最容易被跳过，因为它们看起来"不像在干活"。但每次项目结束后复盘，最大的时间浪费几乎都可以追溯到前三步的草率。

{{< sentence >}}不是所有分支都值得深挖。砍掉七成，留下三成，才是优先级的意思。{{< /sentence >}}

排优先级不是"给每条线打个分然后从高到低排"。优先级的本质是主动放弃。如果拆解出来十个子问题，最后还是十个都在做，等于没排过。

## 分析质量和假设驱动

{{< sentence >}}先有假设，再做分析。没有假设的数据收集是在大海里捞针。{{< /sentence >}}

假设驱动是七步法的核心引擎。不是"想好了答案再去找支持"，而是"先有一个方向性判断，再去找支持或推翻它的证据"。方向错了可以换。没有方向，收集再多数据也只是在绕圈。

{{< sentence >}}一天之内能回答的问题，不要用一周去完美。{{< /sentence >}}

"一日答案"是七步法里最实用的加速器。在信息不完备时，强迫自己在一天内给出方向性判断。判断可以粗糙，但必须有。它让团队快速校准方向，而不是等到信息齐全再开始思考。

{{< sentence >}}逻辑树的唯一质检标准是 MECE——不重叠、不遗漏。{{< /sentence >}}

MECE 是"相互独立、完全穷尽"的缩写。画逻辑树时，如果分支之间有重叠，分析就会重复；如果有遗漏，关键问题就会被漏掉。不符合 MECE 的树，画了等于没画。

## 综合与交付

{{< sentence >}}综合不是把每个人的发现拼在一起。综合是回答一个问题：所以结论到底是什么？{{< /sentence >}}

汇总和综合的区别，是七步法后半段最关键的分水岭。汇总是把材料摆出来让别人自己判断；综合是你已经替他们判断完了，并且能讲清楚为什么。做不到这一步，前面五步的分析全部浪费。

{{< sentence >}}每一页汇报都要经得起一个追问：所以呢？{{< /sentence >}}

"所以呢"是最终交付的质检工具。如果一页幻灯片展示了数据和图表，但读完之后听众还是要问"所以呢"——这页就没做完。每页都应该自带结论，而不是把判断权留给听众。
