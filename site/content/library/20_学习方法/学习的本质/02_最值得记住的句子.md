# 大脑的学习方式和你以为的几乎相反

下面每句话都指向一个关于学习的常见误判。读的时候先问自己：我原来的做法和这句冲突吗？冲突在哪里？

## 关于学习到底是什么

{{< sentence >}}
学习不发生在你听懂的时刻，发生在你的预测被打破的时刻。
{{< /sentence >}}

听懂意味着当前模型够用，不需要更新。只有预测和现实对不上——犯错或困惑的瞬间——大脑才启动模型修改。追求"听一遍就懂"的人追求的恰恰是不需要学习的状态。

{{< sentence >}}
熟悉感是记忆最危险的替身。
{{< /sentence >}}

反复看笔记让你觉得"我知道这个"。这种感觉来自识别通道，考场上需要的是提取通道——从空白状态把答案建构出来。两条通路独立运作，识别能力强不代表提取能力也强。

## 关于注意力

{{< sentence >}}
注意力不是你投入多少的问题，是一次只能通过一件事的问题。
{{< /sentence >}}

注意力像窄门，加大力度不会让门变宽。同时做两件需要深度加工的事，每件的质量都会下降。多任务处理不提升效率，它降低每个任务的加工深度。

{{< sentence >}}
手机放在视野里，即使没响，注意力已经在分流了。
{{< /sentence >}}

大脑持续监控环境中可能需要响应的东西。手机的存在本身就占用了一部分注意力资源，不需要真的有通知。移走比静音更有效——问题不在声音，在于大脑知道那里有一个随时可能打断你的源头。

## 关于错误和反馈

{{< sentence >}}
先猜再看答案，比先看答案再记，学得更牢——哪怕每次都猜错。
{{< /sentence >}}

猜测迫使大脑生成预测，看答案时产生预测误差，预测误差驱动模型更新。跳过猜测直接看答案，等于跳过了学习引擎的启动步骤。全猜错也比不猜强。

{{< sentence >}}
即时反馈不总是最优解，延迟反馈有时候学得更深。
{{< /sentence >}}

立刻知道对错让纠错效率最高，但延迟反馈迫使大脑在等待期间维持不确定状态，这种不确定本身会加深编码。场景不同，最优反馈时机不同。

## 关于睡眠和时间

{{< sentence >}}
睡眠是学习流水线的固化工序，砍掉它等于砍掉成品率。
{{< /sentence >}}

白天标记需要更新的连接，睡眠期间完成结构性修改。慢波睡眠负责把短期痕迹搬到长期存储区，快速眼动睡眠负责在已有知识网络中为新内容找接口。砍睡眠等于砍掉流水线最后一道工序。

{{< sentence >}}
同样的内容，分三天各学二十分钟，比一口气学一小时记得更久。
{{< /sentence >}}

间隔让每次重新学习都经历一次小规模遗忘和重新提取。每次提取都是一次预测-纠错循环，每次循环都加固一次连接。集中学习省掉了这些循环，看起来快，但连接没有被反复加固过。

## 什么场景调哪句

| 场景 | 调这句 |
|------|--------|
| 听完课觉得全懂了，不想做题 | 第 1、2 句 |
| 学习时总忍不住看手机 | 第 3、4 句 |
| 做错题觉得是浪费时间 | 第 5 句 |
| 急着知道做得对不对 | 第 6 句 |
| 想熬夜多学一会儿 | 第 7 句 |
| 试图一次性学完所有内容 | 第 8 句 |
