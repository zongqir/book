{{< sentence >}}
不要去煮沸整个海洋。
{{< /sentence >}}

面对复杂问题的第一反应往往是收集所有信息。这行不通——信息是无穷的，时间是有限的。先形成假设，再去收集能证实或推翻假设的关键信息。

{{< sentence >}}
以假设为驱动，而不是以数据为驱动。
{{< /sentence >}}

传统做法是先收集数据再得出结论。麦肯锡的做法是先提出初始假设，然后收集数据来验证或修正假设。假设可以是错的——但有假设比没假设高效十倍。

{{< sentence >}}
MECE：相互独立，完全穷尽。
{{< /sentence >}}

拆解问题时确保每个部分之间没有重叠、所有部分加起来没有遗漏。听起来简单，做到很难。大多数分析的问题不是分析能力差——是问题拆解的时候就漏了或者重了。

{{< sentence >}}
在电梯里能讲清楚的方案才是好方案。
{{< /sentence >}}

电梯测试：如果你只有30秒向CEO汇报——你能讲清楚吗？如果不能，说明你还没有真正理解自己的结论。不是表达能力的问题，是思考清晰度的问题。

{{< sentence >}}
80/20法则：80%的价值来自20%的工作。
{{< /sentence >}}

不是所有分析都值得做。找到那20%——最能影响结论的关键分析——集中火力。做完80%就够了，最后20%的精益求精往往不改变结论。

{{< sentence >}}
先说结论，再说原因。
{{< /sentence >}}

不要像写论文一样从背景写到分析再到结论——没人有耐心等你铺垫完。先说"我建议做X"，然后回答"为什么"。
