## 先说结论，其他都是注脚

{{< sentence >}}
职场说话的第一句如果不是结论，后面说的全是噪音。
{{< /sentence >}}

领导问"项目进展怎么样"，你从上周的需求变更说起，讲了五分钟还没到重点。领导已经在看手机了。结论先行不是偷懒，是尊重对方的时间和注意力。

{{< sentence >}}
汇报不是讲过程，是预判领导会问什么，提前把答案备好。
{{< /sentence >}}

汇报的常见错误：把自己做了什么按时间顺序讲一遍。领导不关心你的时间线，关心的是结论、风险和下一步建议。孙路弘把汇报结构压成四个模块：结论→依据→风险→建议。

## 谈判不是讲道理，是算交换

{{< sentence >}}
你要什么不重要，重要的是你能给什么。
{{< /sentence >}}

职场谈判和正式商务谈判的区别在于：你通常没有退场的自由，你还得跟对方继续合作。孙路弘的谈判建议不是"怎么多拿"，是"怎么让对方觉得不亏"——这才是能反复用的逻辑。

## 说服卡住了，先找对方的卡点

{{< sentence >}}
说服失败不是因为你的道理不够硬，是因为你没找到对方卡在哪里。
{{< /sentence >}}

你准备了十个理由来推动一个方案。同事说"我再想想"。你以为是理由不够，又补了五个。其实对方的卡点不在你的方案本身，在于他担心自己部门的资源被抽走。十五个理由没有一个对准了他的真实顾虑。

{{< sentence >}}
说服不是单方面输出，是先诊断对方的决策障碍。
{{< /sentence >}}

孙路弘把说服拆成两步：第一步不是说，是问。问到对方的卡点在哪里。第二步才是拿出解决卡点的方案。跳过第一步直接开讲的人，大概率在对着空气说话。

## 练习比翻完更重要

{{< sentence >}}
说话能力不是读出来的，是用嘴巴练出来的。
{{< /sentence >}}

孙路弘在每章结尾放的不是总结，是练习题——"用下面这个模板重新组织你上周的汇报"。他的判断很明确：说话是肌肉记忆，不是知识储备。你读一百本沟通书，不如把一次汇报按结构改写三遍。
