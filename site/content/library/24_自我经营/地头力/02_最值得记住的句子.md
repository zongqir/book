# 没有假说的人到了现场只会收集信息

{{< sentence >}}没有假说的人到了现场，只会收集信息，不会发现问题。{{< /sentence >}}

带着问题去和不带着问题去，看到的东西完全不同。假说不需要正确，但需要存在——它是让观察有方向的前提。下次去现场之前，先问自己一个判断性的问题，哪怕是错的。

{{< sentence >}}答案不在办公室里，在现场。{{< /sentence >}}

会议室讨论了三小时得不出结论的事，到车间走一圈可能五分钟就清楚了。信息经过汇报层层过滤之后，失真是必然的。直接看到的东西，和别人转述给你的东西，判断质量差一个量级。

{{< sentence >}}结论力不是拍脑袋，是在不完整信息下快速形成可验证的假说。{{< /sentence >}}

"信息不够所以不做判断"是一种常见的责任回避。有效的结论力是：信息只有七成，先做一个判断，然后立刻设计验证方式。错了改比不判断要好。

{{< sentence >}}抽象力是从十件事里看出同一个结构。{{< /sentence >}}

车间三个班组犯同一类错误，表面看是培训问题，抽象一层可能是工序设计问题，再抽象一层可能是排班逻辑和工序复杂度不匹配。能跳几层，决定了你能解决多深的问题。

{{< sentence >}}换一个视角看同一件事，你会发现完全不同的因果链。{{< /sentence >}}

客户投诉质量问题——从生产部看是良品率达标，从客户角度看是使用场景不匹配。同一组数据，视角一换，问题定义就变了。转换力的价值不在于"全面"，在于发现单一视角里看不到的因果关系。

{{< sentence >}}框架是路标，不是道路本身。{{< /sentence >}}

框架帮你快速定位方向，但走路还得靠自己的脚。用框架替代思考的人，最后变成了框架的搬运工——输入不同场景，输出相同结论。地头力要求的是：到了路标指不到的地方，你还能自己找路。
