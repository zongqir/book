# 八句话，把演讲从自我表演拉回听众认知设计

{{< sentence >}}
你演讲结束后，听众脑子里只会留下一个东西。先决定那个东西是什么。
{{< /sentence >}}

准备演讲时最容易犯的错是什么都想说。维曼斯的纪律是先砍到只剩一个核心信息，然后用全部时间围着它展开。

{{< sentence >}}
演讲不是你在台上做了什么，是听众在椅子上经历了什么。
{{< /sentence >}}

视角翻转。大多数演讲者盯着自己的稿子、自己的紧张、自己的表现。维曼斯的思路是：忘掉自己，跟着听众的认知状态走。

{{< sentence >}}
开场不是吸引注意力，是松动听众脑子里原来那个想法。
{{< /sentence >}}

"吸引注意力"太模糊，容易变成讲笑话。"松动默认状态"是一个更精准的目标——让听众原来的判断产生裂缝，他才会往你的方向走。

{{< sentence >}}
故事不是调味品，是负责把听众从一个理解层运到下一个理解层的交通工具。
{{< /sentence >}}

这句话会彻底改变你选故事的方式。你不再找"感人的""有趣的"故事，而是找"在认知路径上最高效的"故事。

{{< sentence >}}
如果你讲完一个道理还要再讲一个故事来解释，说明道理本身没讲明白。
{{< /sentence >}}

故事和道理不是并列关系。好的故事里道理已经自明了，不需要你再念一遍。

{{< sentence >}}
结尾不是总结。结尾是给听众一个能带走的东西。
{{< /sentence >}}

总结不创造新的认知价值。好的结尾让听众觉得"我拿到了一样东西"，而不是"终于讲完了"。

{{< sentence >}}
紧张不是你的敌人，无聊才是听众的敌人。
{{< /sentence >}}

演讲者花在克服紧张上的精力，远超花在"听众会不会走神"上的精力。但让一场演讲失败的，几乎从来不是紧张，而是听众坐了十分钟还不知道你要说什么。

{{< sentence >}}
一场演讲能排练多少遍，就值多少分。
{{< /sentence >}}

维曼斯方法论里最朴素也最容易被忽略的一条。设计再好，不排练就没法验证。排练不是为了背稿，是为了让你发现认知路径上哪里断了、哪个转折不自然、哪个地方听众会走神。
