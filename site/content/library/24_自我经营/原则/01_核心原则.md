# 七条判断准则，把经验变成会进化的规则系统

{{< principle no="1" type="core" statement="痛苦本身不会带来进步，只有被追问和被记录的痛苦才会变成资产。" >}}

达利欧最硬的一刀，就是把“吃亏会成长”的浪漫幻觉拆掉。情绪过去以后，什么都不留下，痛苦就只是痛苦。

{{< /principle >}}

{{< principle no="2" type="core" statement="原则不是信条，而是被现实反复检验过、仍然值得保留的如果—那么规则。" >}}

这条原则把《原则》和格言书彻底分开。原则不是为了显得有见地，而是为了在下一次类似场景里继续能用。

{{< /principle >}}

{{< principle no="3" type="support" statement="重要判断不要只留在脑内，要尽量外化到能被别人看见、也能被自己回放的地方。" >}}

一旦不外化，复盘就会被记忆和自尊共同改写。外化不是形式主义，是为了让经验可检验。

{{< /principle >}}

{{< principle no="4" type="support" statement="不同人的意见不该天然等重，可信度要进入决策机制。" >}}

达利欧把“谁更懂这件事”当成制度问题，而不是靠老板直觉处理的人情问题。

{{< /principle >}}

{{< principle no="5" type="support" statement="任何流程化决策最后都要回到现实校验，规则可以辅助现实，不能取代现实。" >}}

算法、指标、打分都只是桥，不是地面。离开现实反馈，原则系统会很快僵化。

{{< /principle >}}

{{< principle no="6" type="support" statement="原则系统要允许透明暴露错误，否则反馈只会停在表面礼貌层。" >}}

达利欧为什么那么强调公开、记录和直面分歧？因为如果错误只留在私下感受里，它就很难真的进入系统。很多组织不是没有反馈，而是反馈不够透明，导致规则更新停在很浅的位置。

{{< /principle >}}

{{< principle no="7" type="support" statement="规则服务的是更高质量的现实判断，不是把人训练成规则执行机器。" >}}

这条原则是给整套系统装刹车。原则越多，越要防止自己开始把规则当现实本身。真正成熟的使用方式，是让规则帮你更快贴近现实，而不是拿规则替代对现实的观察。

{{< /principle >}}
