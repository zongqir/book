# Summary 写法

summary 只写内容判断，不写页面说明。

如果目标是单书目录入口 `_index.md` 的 summary，额外遵守一条：
- 不要把 summary 写成“把一本……拆成……”或“把这本书整理成……”这类内容加工说明。
- `_index.md` 的 summary 要回答的是“这本书最该留下什么判断”，不是“仓库把它加工成了什么产物”。

适用时机：
- 用户要求重写摘要、收紧 summary、减少模板腔
- 现有 summary 像导语、推荐语、功能说明或目录提示
- 句子里反复出现"这本书 / 这一章 / 这页 / 这篇 / 会告诉你 / 会带你 / 最值钱"

执行规则：
1. 先提炼正文里最该留下的一个判断，优先写误判、边界、顺序或动作。
2. 直接陈述内容，不写页面用途；默认不写"这本书 / 这一章 / 第 X 章 / 这页 / 这篇"。
3. 删除"会告诉你 / 会带你 / 帮你 / 最值钱 / 最值得 / 核心 / 真正"这类抬气氛词。
4. 删除“把一本……拆成……”“把这本书压成……”“整理成文稿 / 书稿 / 卡片 / 功能文档”这类加工动作句，尤其是 `_index.md`。
5. 句子要能脱离标题单独成立，读起来像正常中文，不像推荐文案。

改写方向：
- 从"这页讲什么"改成"内容本身在说什么"
- 从"推荐语"改成"直接判断"
- 从"抽象价值词"改成"误判、边界、顺序、动作"
- 从"仓库怎么加工这本书"改成"这本书到底留下什么判断"

对照例子：
- 差：这一章会告诉你性能分析该怎么开始。
- 好：性能分析的起手动作不是抓工具，而是先把"慢"定义具体。
- 差：这本书真正能带给你的是排障方法。
- 好：排障稳定性来自顺序、证据和分层收窄，不来自工具数量。
- 差：把一本以人物和危机为线索、讲华尔街三百五十年制度演化的金融通史，拆成可判断、可执行、可复盘的理财文稿。
- 好：华尔街每次危机留下来的不只是输赢，更是规则、机构和权力边界的重写。
- 差在哪：前一句写的是仓库加工动作，不是这本书本身最该留下的判断。

最小检查：
- 这句话脱离标题还能不能成立
- 它是在说内容，还是在说页面用途
- 有没有写"这本书 / 这一章 / 第 X 章 / 这页 / 这篇"
- 有没有写"会告诉你 / 会带你 / 帮你 / 最值钱 / 最值得 / 核心 / 真正"
- 如果是 `_index.md`：有没有把“仓库怎么拆、怎么整理、怎么压成文稿”写进 summary

## 终检清单

改写完成后逐条过一遍：

1. summary 是否只有一句话，没有分号或句号拆成多句
2. 是否避免了"本书 / 本章 / 这本书 / 这一章"等指代词
3. 是否包含一个具体判断，而不是泛泛评价
4. 读者不看标题，单读这句话能不能理解它在说什么
5. 是否避免了"最值得 / 最核心 / 真正重要 / 最有价值"等抬气氛词
6. 是否避免了"会告诉你 / 会带你 / 帮你理解"等导购句式
7. 如果是 `_index.md`，是否避免了“把一本……拆成……”这类加工动作句
8. 字数是否控制在 40 字以内
9. 读起来是否像正常中文陈述句，不像文案或广告语
10. 判断的强度是否与证据匹配，没有把推测写成定论
