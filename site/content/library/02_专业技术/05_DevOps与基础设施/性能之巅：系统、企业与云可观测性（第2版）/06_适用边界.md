# 《性能之巅》的适用边界

这本书最适合用在这几类处境里: 你面对的是系统级性能问题，症状表现为延迟、吞吐、资源利用、排队、抖动、饱和或容量边界；你需要解释一个问题为什么发生，而不是只想把某个指标先压下去；你有一定系统背景，愿意跨内核、运行时、应用和基础设施边界看问题。换句话说，它最适合“需要诊断链”的任务。

它不太适合被直接拿来解决产品分析、业务漏斗、简单前端体验评估，或者只依赖 APM 看板就能处理的浅层告警。也不适合完全零基础读者把它当第一本 Linux 入门书。因为它默认你至少知道系统大致怎么运转，知道延迟、上下文切换、I/O、锁、网络这些词在说什么，否则读起来会更像在背术语。

还有一个容易误用的边界: 这本书不是“任何性能问题都必须按全套流程走”的命令。对于一次性的小问题、证据非常明显的问题，轻量路径当然更经济。它真正擅长的，是那些层次多、线索乱、容易误判、而且值得留下方法沉淀的问题。真正该用它的时候，通常不是“事情很小”，而是“错一步就会把团队带偏很多时间”。
