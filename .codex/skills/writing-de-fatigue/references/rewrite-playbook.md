# Rewrite Playbook

## 目标

把"意思没错，但读着累"的正文改轻。

按三个层面来组织：词、句、段。每个问题：怎么认、怎么改、改完什么样。

## 一、词的层面

### 1. 书面虚词——怎么认，怎么改

全文搜：`由于`、`因此`、`以便`、`从而`、`以及`、`关于`、`涉及`、`对于`。

同一段出现 2 个以上就要动手。

直接替换：由于→因为，因此→所以/删，以便→这样就，从而→这样，以及→和，关于→说到/删，涉及→碰到，对于→对。

### 2. 名词化空壳——怎么认，怎么改

全文搜：`进行`、`做出`、`加以`、`予以`、`开展`、`实施`。

看到"空壳动词 + 名词"结构，直接用那个名词的动词形式替换。

改写前：

```md
作者对传统阅读方法进行了系统性批判，并对新的阅读框架做出了详细阐述。
```

改写后：

```md
作者系统批判了传统阅读方法，然后详细讲了自己的新框架。
```

### 3. 造词和隐喻动词——怎么认，怎么改

搜不到，靠通读。判断标准：跟朋友说话时这个词说不出口。

常见造词：`方法线`、`壳句`、`改判`、`经营纪律`、`成功部件`。换成日常表达或拆开说。

常见隐喻动词做主力表述：`压回`、`打到`、`碰到顾客`、`落到系统`、`站稳`、`收成三组`。确认比字面表达更精确时才保留，否则换回去。

改写前：

```md
它的方法骨架可以压成三层。
```

改写后：

```md
方法可以分三层来看。
```

改写前：

```md
先把问题压回零售最该回答的地方。
```

改写后：

```md
先把问题拉回来：顾客最近到底变了什么？
```

## 二、句的层面

### 4. 修饰前置和的字链——怎么认，怎么改

#### 怎么认

读一句话，如果读到后半段才知道主语是谁，就是修饰前置过多。数句子里的"的"，超过 2 个就是"的字链"。

#### 怎么改

核心动作：把主语提到前面，修饰语拆成后续分句。

改写前：

```md
在当前充满不确定性的竞争格局下努力提升自身核心竞争力的中国科技企业正面临前所未有的挑战。
```

改写后：

```md
中国科技企业正面临一个大挑战。竞争格局充满不确定性，核心竞争力必须跟上。
```

"的字链"改法：删掉中间层级的"的"，把嵌套关系拆成并列或因果。

改写前：

```md
本书对于现代组织管理中的领导力发展的理论框架的适用边界的讨论尤为深入。
```

改写后：

```md
领导力发展有一套理论框架，但不是哪里都管用。这本书讨论的就是它的适用边界。
```

### 5. 因果压缩——怎么认，怎么改

#### 怎么认

一句话里出现两个以上因果连接词（因为、导致、使得、从而、于是），或者因果链超过两环。

#### 怎么改

找句子里最重的那个逗号或连接词，从那里断开。每句只处理一个因果关系。

改写前：

```md
正是因为作者坚持从实践出发收集案例，使其论证建立在坚实的经验基础上，从而使整套方法论获得了超越纯理论推演的说服力。
```

改写后：

```md
作者的案例全从实践里来，论证就有了经验基础。这比纯靠理论推演要有说服力得多。
```

### 6. 方向变化过多——怎么认，怎么改

#### 怎么认

数一句话里的方向变化次数。每个"但""却""虽然""不过""其实""不是…而是…"都算一次方向变化。超过 1 次就要拆。

"不是…而是…"及其变体是最常见的方向变化。全文搜 `不是`、`而是`、`并不是`、`不在于`、`与其说`。数总量。超过 3 次就要动手。

#### 怎么改

**多方向句：** 拆成两句，每句处理一个弯。

改写前：

```md
虽然作者的论证在大方向上基本成立，但部分关键环节的证据其实并不充分，不过这并不影响整体框架的参考价值。
```

改写后：

```md
论证的大方向基本成立，整体框架可以参考。但有几个关键环节的证据不太够。
```

**反向建构：** 删掉 A，直接说 B。

改写前：

```md
这本书最有价值的地方，不是教你怎么读得更快，而是让你分清"读过"和"读懂"。
```

改写后：

```md
这本书最有价值的地方，是让你分清"读过"和"读懂"。
```

改写前：

```md
它解决的问题是：当局面复杂、力量悬殊、意见混乱时，怎样不靠抽象激情，而靠更稳的顺序把问题看清。
```

改写后：

```md
局面复杂、力量悬殊、意见混乱——这时候靠激情没用，得靠一个更稳的判断顺序。
```

#### 保留判断

有些"不是…而是…"是必要的。保留条件：
- 这个误解读者真的会犯
- 不纠正会严重影响后面的阅读
- 全篇已经控制在 3 次以内

### 7. 重心错位——怎么认，怎么改

#### 怎么认

读完一句话，问自己：最重要的意思是在主句里还是从句里？如果是在"在…的情况下""通过…的方式""基于…的考量"后面的从句里，就是重心埋错了。

#### 怎么改

把核心判断翻到第一句做主句。条件和限定变成后面的补充。

改写前：

```md
因为结构整理真正整理的，从来不是材料，而是问题和回答之间的关系。
```

改写后：

```md
结构整理的核心是问题和答案的逻辑关系，不是材料本身。
```

改写前：

```md
如果能够深入理解作者在第三章中所构建的分析框架并将其与自身实践相结合，读者将从本书获得远超一般管理类书籍的收获。
```

改写后：

```md
这本书真正的收获在第三章的分析框架。理解它，再接上自己的实践，能学到的东西远超一般管理书。
```

### 8. 间接表达——怎么认，怎么改

#### 怎么认

句子有一个"导入壳"，删掉它意思不变。常见导入壳：

- `它解决的问题是…`
- `这套组织方式都围着一个判断转…`
- `这也是为什么…`
- `卷一还要逼你回答另一件事…`

#### 怎么改

删掉导入壳，直接从内容开始。

改写前：

```md
这套组织方式都围着一个判断转: 先把回答交出去，再把支撑它的东西一层层往下摆。
```

改写后：

```md
核心判断很简单：先给答案，再分层展开理由。
```

### 9. 模糊指代——怎么认，怎么改

#### 怎么认

搜"它""这""那个"。每遇到一个，看它的指代对象是否在同一句或紧邻上一句。不在的就是模糊指代。

#### 怎么改

换成具体名词。

改写前：

```md
这也是它在工作场景里格外耐用的原因。
```

改写后：

```md
金字塔原理在工作场景里格外耐用，原因就在这里。
```

## 三、段的层面

### 10. 记忆过载——怎么认，怎么改

#### 怎么认

数逗号。一句话里 4 个以上并列名词或短语没有分组，就是过载。

#### 怎么改

先分组，给每组起个角色名。

改写前：

```md
准备太晚、话说太满、开头空、结尾弱、互动差、怯场重、遇到提问发懵，这些都不是抽象问题。
```

改写后：

```md
准备和结构上的问题——准备太晚、开头空、结尾弱。现场表现上的问题——话说太满、互动差、怯场重、提问接不住。

哪一类都不抽象，都能单独练。
```

### 11. 防御性写作——怎么认，怎么改

#### 怎么认

搜：`常见误读`、`边界在于`、`这条原则约束的是`、`当然这不是说`、`不要把 A 理解成 B`。

看段落结构：如果一个判断后面跟了 2 层以上补充，就是防御过度。

#### 怎么改

砍到只剩一层。如果砍完判断站不住，改判断本身。

改写前：

```md
不要只看表面强弱，要看局面里有没有裂缝和可生长条件。

这条原则约束的是"弱就一定不能活，强就一定不可动摇"的线性判断。真正的局面经常不是这么走的。

常见误读是把它读成盲目乐观，好像任何弱势局面都能逆转。边界恰恰在这里：裂缝和条件要真实存在，不是靠意志补出来。

它纠正的不是悲观情绪，而是把表面强弱直接当终局的偷懒。
```

改写后：

```md
表面的强弱不等于终局。

局面里有没有裂缝，有没有可以长出来的条件——只要这些问题没问完，就不该提前收摊。当然，裂缝和条件得真实存在，靠意志补不出来。
```

#### 常见陷阱

砍防御层时最容易犯的错，是把判断也一起磨软了。防御层该砍，但判断的硬度要保住。

### 12. 节奏锁死——怎么认，怎么改

#### 怎么认

看段落长度。连续 4 段以上都是 3-5 行，节奏就锁了。看段首结构。连续 3 段以上用同一种开头，模板感就出来了。

#### 怎么改

两种动作：插短段、换开头。

改写前（连续长段）：

```md
第一类场景是替别人定义问题。你知道概念，知道立场，甚至知道正确表述，却不知道那群人每天到底被什么卡住、先在乎什么、真正会为什么动起来。改法不是"更关心"，而是先去调查。

第二类场景是一次受挫就判死。局部受挫之后，团队迅速得出"这条线根本不可能"的结论。
```

改写后（长短交替）：

```md
第一类：替别人定义问题。你知道概念和立场，却不知道那群人每天被什么卡住。改法是先去调查，从具体处境拆起。

这个错法今天到处都是。

第二类：一次受挫就判死。局部没打赢，团队马上觉得整条线不行了。
```

### 13. 抽象先行——怎么认，怎么改

#### 怎么认

看段首。连续 3 段以上都是概念或判断开头（`最容易…`、`真正要…`、`核心在于…`），读者已经在概念空转了。

#### 怎么改

在其中至少 1 段换成具体处境起笔。

改写前：

```md
这套方法最擅长处理的，不是所有复杂问题，而是那类力量分布很不均、表面判断很容易把人和问题看粗的局面。
```

改写后：

```md
团队里吵了三天，谁都觉得自己对。再一看，连"对面到底是谁"都还没搞清楚。

这套方法就是处理这种局面的——力量不均，判断又粗，谁都在凭感觉表态。
```

不是每段都要加场景。连续 3 段概念开头时，插 1 段处境就够。

## 四、复合叠加——整段重写指南

当一段话在词、句、段三个层面同时出问题时，不要逐个修补。直接重写。

重写步骤：
1. 先问自己：这段到底想说什么？用一句话说出来
2. 从那句话开始写，一次只展开一个点
3. 写完对照原段，确认核心判断没丢、限定条件没删

改写前（三层都中）：

```md
这套经营思路的起点不是"怎么卖更多"，而是"旧经验为什么开始解释不动现在的顾客"。所以它先处理的不是动作，而是对顾客的解释。零售现场最容易出现一种错觉: 生意一出问题，大家立刻讨论该降价、该换陈列、该做活动，仿佛动作越快越接近答案。
```

改写后：

```md
生意一出问题，团队的第一反应通常是：降价、换陈列、做活动。动作越快越心安。

但铃木敏文先问另一件事：顾客最近变了什么？

只要这个问题没回答，后面的动作越坚决，偏差只会越大。
```

## 五、禁做事项

改完以后，命中任一条都不算过稿：
- 更顺了，但更空了
- 多了很多"你""我们"，内容还是没落地
- 具体判断被改成了抽象漂亮话
- 限定条件被删掉了
- 全文换成另一套固定句式
- 反向建构被砍了，但换成了另一种同样密集的句式模板
- 口语化过头——"说白了就是""你猜怎么着"不是通俗，是表演
