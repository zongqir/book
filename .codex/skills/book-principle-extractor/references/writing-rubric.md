# 原则提取 · 写作标尺

## 四层快速判断

- **观点**：主要在描述世界、问题或人的行为吗？→ 观点层
- **原则**：主要在给一个判断方向或约束吗？方法变了它还成立吗？→ 原则层
- **方法**：有明确的处理路径或决策模式吗？→ 方法层
- **步骤**：具体到可以立即执行吗？→ 步骤层

## 原则质量检验

一条好的原则通常满足大部分：

1. 能约束判断
2. 跨章节成立
3. 能生成或解释它下方的方法
4. 有一个明确的常见误读
5. 有边界
6. 一句话能记住

## 常见假原则

- 模糊口号："保持耐心""独立思考"
- 章节标题改写成的建议
- 包装成智慧的执行指令
- 听起来深刻但不约束行动的漂亮话

## 可读性

- 原则总表要一眼扫完，每条一句话
- 核心原则展开时，"约束什么误判""常见误读"必须具体到读者能对号入座
- 支撑原则可以写得更紧凑，但每条至少要有一个失效点
- 段落长短交替，允许一句话成段

## 不要这样写

- 不要把全书所有重要句子都升格成原则
- 不要把方法伪装成原则（"先检视再分析"是方法顺序，不是原则）
- 不要所有原则平铺在同一层，必须区分核心和支撑
- 不要用"核心原则"当标题
- 不要逐条标注"证据强度：作者明说"——元信息放 frontmatter，不放正文
- 不要把筛选过程做成单独段落（"容易被误当成原则的内容"）；如果需要澄清，融入相关原则的"常见误读"

## 门禁

命中任一条，默认不过稿：

- 引言段还在给原则搭台，比如 `真正能穿出……`、`反复纠正的是……`
- 某条原则更像方法、步骤或口号，而不是判断方向
- 核心和支撑仍然平铺，没有层级
- 某条原则换一本到别的书上也成立，说明还不够贴书

## 自检

1. 标题是内容判断还是结构壳？
2. 原则和方法是否区分清楚了？
3. 核心 vs 支撑有没有层级？
4. 每条原则的误读写出来了吗？
5. 证据边界标清了吗？
6. 读起来舒服吗？
