# Rewrite Playbook

## 目标

把“意思没错，但读着累”的正文改轻。

不要追求花哨。
只做五件事：
- 去词壳
- 砍长句
- 去书面味
- 补处境
- 打散节奏

## 一、先删的句壳

下面这些句壳，默认先删再看：

| 坏句壳 | 怎么处理 |
| --- | --- |
| `这本书真正有价值的地方，是...` | 直接写后半句 |
| `最值得注意的是...` | 直接写后半句 |
| `这篇文章会...` | 改成正文内容本身 |
| `把这本书压成...` | 改成具体动作或具体判断 |
| `它提醒你...` | 直接写它提醒的那件事 |

例子：

改写前：

```md
这本书真正有价值的地方，是它一直逼你先看顾客变化。
```

改写后：

```md
顾客变了，判断顺序就得跟着变。

零售先看顾客，再看同行动作。
```

## 二、先拆的句子

下面这些句子，默认优先拆：
- 主干埋在句尾
- 一句里塞了判断、原因、边界三层
- 三句连着都同一开头
- 同一种转折壳连用两次以上
- 25 字以上的句子连排

例子：

改写前：

```md
很多判断之所以越做越偏，是因为问题一开始没有分清层次，后面补进来的证据和分析都会继续替一个歪掉的入口服务。
```

改写后：

```md
很多判断从起手那一问就已经偏了。

入口歪了，后面补进来的材料只会继续替那个偏差服务。
```

## 三、先换书面词

这些词不一定错。
问题是用多了，正文会很板。

优先替换：

| 偏书面 | 更自然 |
| --- | --- |
| 使用 | 用 |
| 进行分析 | 分析 |
| 对于 | 对 |
| 由于 | 因为 |
| 因此 | 所以 / 直接删 |
| 以便 | 这样就 |

例子：

改写前：

```md
由于判断顺序出现偏差，因此后续分析很容易继续围绕错误前提进行展开。
```

改写后：

```md
因为起手就偏了，后面的分析很容易继续围着错前提打转。
```

## 四、怎么补处境

别硬写情绪词。
先补一个局面，情绪会自己出来。

优先补这三类：
- 人已经怎么做了
- 人为什么会顺着这个方向滑下去
- 再滑一步会发生什么

例子：

改写前：

```md
幸福训练不能替代现实处理。
```

改写后：

```md
你已经把时间挤到没有多少腾挪空间了，托底动作还是第一个被删掉。

这时继续加练习，常常只会把该处理的现实问题改写成“我还不够会调整自己”。
```

## 五、怎么打散节奏

“留白”不是玄学。
它通常就是下面二种动作：
- 删除第二次解释
- 让关键句单独站住

一句话已经到了，就停。
不要急着替读者把下一层也说完。

例子：

改写前：

```md
经验很容易先变成盲区，因为过去有效的方法常常会被误当成稳定规律，而需求变化很快，组织如果继续依赖旧经验，就会慢慢失去对现实变化的感知能力。
```

改写后：

```md
经验很容易先变成盲区。过去有效，只说明过去有效。
可是，很多时候顾客已经变了，组织却还在照老路反应。问题通常就是从这里开始的。
```

## 六、禁做事项

改完以后，命中任一条都不算过稿：
- 更顺了，但更空了
- 多了很多“你”“我们”，内容还是没落地
- 具体判断被改成了抽象漂亮话
- 限定条件被删掉了
- 全文换成另一套固定句式
