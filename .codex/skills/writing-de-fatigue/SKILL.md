# 减阅读疲劳重写

## 0. 这个 skill 只回答一个问题

为什么读者明明对内容感兴趣，读到第三段就开始走神？

答案通常不是内容太难。是表达在消耗读者的注意力。

消耗发生在三个层面。词让读者在脑里做翻译。句子让读者读两遍才能接住意思。段落让读者的注意力一断再断，最后退出。

这三层消耗单独看不严重，但它们叠加。叠到一定密度，读者就会觉得：这篇文章在考验我，不是在帮我。

这个 skill 的工作，就是从词、句、段三个层面把消耗降到读者不会注意到的程度。

### 它不做三件事

- 不发明新观点
- 不重定槽位边界
- 不把原来的硬判断磨成软话

## 1. 通俗易懂到底是什么

先说它不是什么：

- 不是用简单词替换专业词（那叫词汇降级）
- 不是把长句全拆成短句（那叫机械限长）
- 不是把内容写浅（那叫降质）
- 不是模仿口语腔（那叫口语化表演）

**通俗易懂的真实定义：读者的大脑在处理这个句子时，不需要回头、不需要重组、不需要在脑内翻译，一遍就能接住意思。**

它对应三个认知变量：

| 变量 | 含义 | 通俗易懂的要求 |
| --- | --- | --- |
| 工作记忆负载 | 读到句尾时，读者需要同时记住几个未完成的语义单元 | 不超过 3 个 |
| 期望确认延迟 | 读者建立预期后，多久能得到确认或反转 | 尽量短，不让读者悬着 |
| 表层-深层映射成本 | 读者需要几步才能从字面意思到达实际意思 | 尽量一步到位 |

一句话同时在三个维度都低成本，读者就觉得"好懂"。任何一个维度成本高，读者就"卡了一下"。连卡三四次，读者退出。

### 通俗 ≠ 浅

通俗是降低表层的穿透成本。浅是降低内容本身的深度。

"人的一切痛苦，本质上都是对自己无能的愤怒"——意思一点不浅，但每个词都是日常词，语序就是说话的顺序，一遍就懂。穿透成本极低。

通俗易懂追求的是：保持深度，降低穿透成本。

### 为什么中文特别容易写累

中文有五个结构特性，使得书面语和口语的距离比很多语言都大：

1. **修饰语全部前置**——所有定语、状语都堆在被修饰词前面。修饰链一长，读者等核心名词的时间越长，工作记忆积压越多
2. **书面语有独立虚词系统**——口语说"因为、所以、但是、用、和"，书面说"由于、因此、然而、使用、以及"。读者在脑里要做一次翻译
3. **四字格是压缩包**——一个四字格精炼有力，三四个连排就变成一堵墙，读者要连续解压
4. **名词化制造正式感**——"讨论"变"进行讨论"，"分析"变"开展分析"，每次名词化都让读者多做一步拆包
5. **话题-评论 vs 主谓结构的张力**——口语天然是"这件事，我觉得不行"，书面强改成"我认为此事不可行"。前者处理成本更低

这五个特性叠加，就是为什么中文正式写作特别容易滑向"意思没错，但读着累"。

- "太绷了"
- "像 AI 写的"
- "像总结，不像正文"
- "像说明书"
- "帮我降模板感"

- "读不进去"
- "每段都一样"

## 3. 词的层面：不让读者在脑里翻译

每一个需要读者在脑里多做一步"翻译"的词，都在消耗注意力。

### 3.1 书面虚词替换

读者看到"由于"，脑里先翻成"因为"，再理解意思。这一步翻译本身就是消耗。

| 偏书面 | 更自然 |
| --- | --- |
| 使用 | 用 |
| 进行分析 | 分析 |
| 对于 | 对 |
| 由于 | 因为 |
| 因此 | 所以，或直接删 |
| 以便 | 这样就 |
| 以及 | 和 |
| 从而 | 这样 |
| 关于 | 说到，或直接删 |
| 涉及 | 碰到 / 遇到 |

这些词不是禁词。一篇里出现几次没问题。问题是密度——同一段里连出三四个，读者就觉得在读报告。

### 3.2 砍名词化空壳

"进行讨论"比"讨论"多一步拆包。"做出调整"比"调"多两步。

看到这些结构，直接用动词替换整个结构：

| 名词化 | 直接动词 |
| --- | --- |
| 进行讨论 | 讨论 |
| 做出调整 | 调 |
| 开展分析 | 分析 |
| 加以改善 | 改 |
| 予以支持 | 支持 |
| 实施管理 | 管 |

改写前：

```md
团队需要对当前的业务流程进行全面的梳理，并对发现的问题做出及时的调整。
```

改写后：

```md
团队得把业务流程全面梳理一遍，发现问题就及时调。
```

### 3.3 造词和隐喻动词

仓库里常见一类词：不是书面词，也不是口语词，而是作者临时造的或把隐喻当字面用的。

常见造词：`方法线`、`壳句`、`改判`、`经营纪律`、`成功部件`。这些词读者第一次遇到必须从上下文猜含义。

常见隐喻动词：`压回`、`打到`、`碰到顾客`、`落到系统`、`站稳`、`收成三组`。本来是比喻，但被当成正式表述的主力动词。读者要先解码隐喻，再理解内容。

规则：

- 造词首次出现时，同句或下句用日常话解释一遍
- 隐喻动词如果不比字面表达更精确，换回字面动词
- 判断标准：跟朋友说话时你说不出口的词，就拆成自然的表达

改写前：

```md
先把问题压回零售最该回答的地方。
```

改写后：

```md
先把问题拉回来：顾客最近到底变了什么？
```

改写前：

```md
它的方法骨架可以压成三层。
```

改写后：

```md
方法可以分三层来看。
```

## 4. 句的层面：不让读者读两遍

词没问题，但句子结构让读者卡住——这是仓库文章最普遍、最隐蔽的问题。句子没超过 35 字，没有"不是…而是…"，用词也不书面，但读完就是要在脑子里倒一下。

这一层的问题来自中文的结构特性，不注意就会滑进去。

### 4.1 先出主角，后给修饰

中文把所有修饰语堆在名词前面。修饰链一长，读者在到达核心名词之前，必须把所有修饰内容悬在工作记忆里。

**规则：名词前最多放 2 层修饰。超过 2 层，把核心名词提到前面，细节用后续分句补。**

改写前：

```md
很多思想文本会给你价值方向，卷一还要逼你回答另一件事：现实到底该怎么拆，行动到底该先押在哪一点上。
```

改写后：

```md
很多思想文本给方向就结束了。卷一多逼你一步：现实怎么拆？行动从哪一点开始？
```

### 4.2 砍"的"字链

每个"的"引入一层嵌套。2 层以内大脑自动处理；3 层需要有意识追踪；4 层以上需要回读。

**规则：同一句里连续的"的"不超过 2 个。超过 2 个，拆中间的修饰关系为独立分句。**

改写前：

```md
本书对于现代组织管理中的领导力发展的理论框架的适用边界的讨论尤为深入。
```

改写后：

```md
领导力发展有一套理论框架，但不是哪里都管用。这本书讨论的就是它的适用边界。
```

### 4.3 一句话一个因果

因果链是线性依赖的——理解 C 必须先理解 B，理解 B 必须先理解 A。一句话里超过 2 个因果环节，理解准确率显著下降。

**规则：一句话里最多一个因果连接。因果链超过 2 环，拆成多句。**

改写前：

```md
很多判断之所以越做越偏，是因为问题一开始没有分清层次，后面补进来的证据和分析都会继续替一个歪掉的入口服务。
```

改写后：

```md
很多判断从起手那一问就已经偏了。入口歪了，后面补的材料只会继续替那个偏差服务。
```

### 4.4 重心别埋在从句里

核心判断应该在主句。但书面中文常把最重要的意思藏在"在…的情况下""通过…的方式""基于…的考量"后面，主句反而是空壳。

**规则：写完一句话，问自己——最重要的意思在主干还是在从句？如果在从句，翻到主干来。**

改写前：

```md
因为结构整理真正整理的，从来不是材料，而是问题和回答之间的关系。
```

改写后：

```md
结构整理的核心是问题和答案的逻辑关系，不是材料本身。
```

### 4.5 一句话只过一个弯

每个转折、否定或条件切换都要求读者修改心理模型。一句话里修改两次以上，读者就不确定最终的意思该是什么。

这条比"反向建构"更底层——它管的是句内方向变化的总次数，不只是"不是…而是…"这一种模式。

**规则：一句话里最多一次方向变化。两次以上，拆成两句，每句处理一个弯。**

改写前：

```md
虽然作者的论证在大方向上基本成立，但部分关键环节的证据其实并不充分，不过这并不影响整体框架的参考价值。
```

改写后：

```md
论证的大方向基本成立，整体框架可以参考。但有几个关键环节的证据不太够。
```

**反向建构（"不是…而是…"）的限频规则也归在这里：**

- 全篇不超过 3 次
- 连续两段不能都出现
- 变体都算同一种消耗：`不只是…更是…`、`并不是…其实是…`、`不在于…在于…`、`与其说…不如说…`
- 保留条件：这个误解读者真的会犯，且不纠正会严重影响后文理解

改写前：

```md
零售经营真正反复出现的，不是某一个零售绝招，而是一条经营顺序。
```

改写后：

```md
零售经营里反复出现的是一条顺序：先看顾客变了什么，再试，再改。
```

### 4.6 直说，不绕路

间接表达是仓库文章第二大问题。它不是句子太长或用词太书面，而是说一件事时先绕一个弯。

常见形式：

- 用"另一件事"代替直接说那件事是什么
- 用"它解决的问题是"开头，而不是直接说解决方案
- 用"这也是为什么…"做跳板，而不是直接给判断
- 用"这套组织方式都围着一个判断转"，而不是直接说那个判断

**规则：如果删掉导入句，直接从内容开始，意思不变，那导入句就是多余的。**

改写前：

```md
这套组织方式都围着一个判断转: 先把回答交出去，再把支撑它的东西一层层往下摆。
```

改写后：

```md
核心判断很简单：先给答案，再分层展开理由。
```

改写前：

```md
所以顺序不是装饰。它在替你把"这些材料之间是什么关系"多说半句话。
```

改写后：

```md
顺序本身就在说明材料之间的关系。
```

### 4.7 指代要明确

每一个"它""这""那个"都需要读者回头找指代对象。找一次没关系，反复找就变成搜索任务。

**规则：如果"它"或"这"的指代对象不在同一句或紧邻的上一句，换成具体名词。**

改写前：

```md
这也是它在工作场景里格外耐用的原因。
```

改写后：

```md
金字塔原理在工作场景里格外耐用，原因就在这里。
```

## 5. 段的层面：不让读者走神退出

词和句子都没问题，但段落的组织方式让读者的注意力一断再断。

### 5.1 记忆过载

一段里塞太多东西，读者必须全部 hold 住才能走到终点。

**并列项过多**——4 个以上并列项没有分组。

规则：同一句里并列项不超过 3 个。超过 3 个，先分组再展开。

改写前：

```md
选址、配送、订货、单品管理、信息系统、门店沟通，这些看起来分散的动作，其实都在服务同一件事。
```

改写后：

```md
选址和配送是后台，订货和单品管理是现场，信息系统把两头接起来。看着分散，其实都在服务同一件事。
```

### 5.2 防御性写作

每个判断后面跟两三层"但也要注意""常见误读是""边界在于"。读者不是在接收内容，而是在旁听作者跟想象中的反对者辩论。

**规则：**
- 一个判断后面最多跟一层补充
- `常见误读是…` 全篇最多出现 1 次
- 觉得需要两层以上防守，回去改判断本身

改写前：

```md
理论只有回到调查和实践里，才有资格继续被相信。

这条原则约束的是书本替代调查、概念替代接触、引用替代理解。常见误读是把它读成"经验比理论高明"，于是顺手滑到反智。其实它约束的是脱离现实的理论，不是否定理论本身。
```

改写后：

```md
理论当然需要。但理论只有在现实里碰过、被修正过，才从借来的确定感变成自己的判断力。

一旦现实给出反证，你还只拿原来的概念去压，理论就已经僵了。
```

### 5.3 节奏锁死

连续四五段长度一模一样，开头结构一模一样。读者的大脑检测到模板后会自动降速——不再处理语义，只在做模式匹配。

**规则：**
- 不允许连续 4 段以上长度相近
- 不允许连续 3 段以上用同一种开头结构
- 长段后面插一个 1-2 行的短段

### 5.4 抽象先行

段落总是先给判断，后给场景。但人脑的理解顺序是反过来的——先有处境，才有共鸣。

**规则：每 3 段里至少 1 段从具体处境起笔。**

处境不是抒情。就三件事：人卡在哪里、人为什么会默认这么做、继续这么做会怎样。

改写前：

```md
很多经营者一焦虑，就盯着竞争对手看。
```

改写后：

```md
业绩掉了，你打开手机先看的是隔壁店上了什么新品。看完心更慌，回来赶紧也跟一个。
```

## 6. 复合诊断

真正让人累的，往往不是单个问题，而是同一段里三个层面的问题同时出现。

**诊断规则：先把每一段过一遍，标出它在词、句、段三个层面各命中了什么。**

- 只命中一个层面的一个问题：局部微调
- 命中两个层面：这一段需要认真改
- **三个层面都中：整段重写，不要修补**

例子——一段同时在三个层面出问题的典型累段落：

```md
零售经营真正反复出现的，不是某一个零售绝招，而是一条经营顺序: 先重新看顾客哪里变了，再把判断写成可验证假设，随后用门店动作和单品反馈去试，最后把有效经验沉到订货、商品、配送和信息系统里。
```

- ❌ 词：隐喻动词"沉到"
- ❌ 句：反向建构 + 62 字长句 + 因果压缩
- ❌ 段：并列过载（先…再…随后…最后… + 订货、商品、配送、信息系统）+ 零处境

重写：

```md
铃木敏文做了四十年便利店，招数换了无数，但经营顺序从来没变过。

先看顾客最近变了什么。看完不急着动，先把猜测写成一句能被推翻的话。然后找一个小动作去试——改一个货架、换一组商品。看几天数据，确认有效，再写进系统。

这条顺序本身，比任何一招零售绝招都耐用。
```

## 7. 按这个顺序重写

### 第一步：词——扫书面词和空壳动词

全文搜书面虚词（由于/因此/以便/从而/以及）和名词化空壳（进行/做出/加以/开展）。替换掉密度过高的。标记造词和隐喻动词，确认是否需要换回字面表达。

### 第二步：句——逐句检查结构

对每个超过 25 字的句子做五项检查：

1. 主语在句子前 1/3 位置出现了吗？
2. "的"超过 2 个了吗？
3. 因果链超过 1 环了吗？
4. 方向变化超过 1 次了吗？
5. 最重要的意思在主干还是从句里？

命中任一项就改。

### 第三步：句——砍反向建构

全文搜 `不是`、`而是`、`并不是`、`不在于`、`与其说`。数总量。超过 3 次，从密度最高的段落开始逐个替换成直接陈述。

### 第四步：段——删防御层

找所有"常见误读""边界在于""当然这不是说"。一个判断后面只留一层补充。

### 第五步：段——破节奏和补处境

看段落长度分布。连续 4 段以上长度差不多的，插短段或换段首结构。连续 3 段以上抽象判断开头的，至少换 1 段为具体处境起笔。

### 第六步：通读——检查自然度

出声读一遍。读起来卡的地方，就是读者也会卡的地方。不要用"写作者"的眼睛检查，用"第一次读的人"的耳朵检查。

## 8. 保真门禁

命中任一条，默认不过稿：
- 为了顺口删掉限定条件
- 把 `llm-draft` 写成定论
- 把"可能 / 往往 / 更像"改成绝对判断
- 把具体内容改成更空的漂亮话
- 把槽位边界改坏
- 改完以后判断密度下降了——多了很多"你""我们"，但内容更空了

## 9. 终检清单

提交前逐条确认。

**词的层面：**
- 书面虚词（由于/因此/以便/从而）同一段不超过 1 个
- 没有"进行/做出/加以/开展 + 名词"残留
- 造词和隐喻动词已确认——要么有上下文自解释，要么已换成字面表达

**句的层面：**
- 主语在每句前 1/3 位置出现
- 同一句里连续"的"不超过 2 个
- 一句话里因果连接不超过 1 个
- 一句话里方向变化不超过 1 次
- 核心判断在主干，不在从句
- `不是…而是…` 及其变体全篇不超过 3 次
- 没有需要回头才能确认的模糊指代
- 35 字以上的句子不超过全篇 15%

**段的层面：**
- 同一段里并列项不超过 3 个（超过的已分组）
- 没有判断后面跟 2 层以上防御性补充
- `常见误读是…` 全篇不超过 1 次
- 没有连续 4 段以上长度相近
- 没有连续 3 段以上用同一种开头结构
- 每 3 段里至少 1 段有具体处境

**保真指标：**
- 核心判断没有被冲淡
- 限定条件没有被删
- 没有用漂亮但更空的话替代原来的具体判断

## 10. 输出规则

- 默认直接改正文，不只给建议
- 先做三层复合诊断，找出三层都中的段落，优先整段重写
- 只中一两个问题的段落，局部微调
- 向用户汇报时，说清这篇主要累在哪个层面，改动集中在哪几步
