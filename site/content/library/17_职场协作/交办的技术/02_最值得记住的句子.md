{{< sentence >}}
你越忙，越说明你没在做管理者该做的事。
{{< /sentence >}}

忙到加班还觉得"团队离不开我"的时候拿出来。忙不是勤奋的证据，是交办不足的症状。管理者的产出不是自己的工作量，是团队的产出量。

{{< sentence >}}
交办不是把事丢出去，是把判断权交给别人。
{{< /sentence >}}

用来纠正"我已经在交办了"的自我感觉。如果交出去的只是执行步骤，对方还是在用你的脑子——那叫分派。

{{< sentence >}}
说清楚要什么结果，比说清楚怎么做重要十倍。
{{< /sentence >}}

每次布置任务之前默念一遍。你花在描述过程上的时间越多，对方思考的空间就越小。反过来，只要结果标准清楚，对方有机会找到比你更好的做法。

{{< sentence >}}
对方犯错的成本，就是培养的学费。
{{< /sentence >}}

交出去之后看到对方做错了想接回来的时刻。接回来你省了今天的返工，赔上的是对方下一次还做不对。

{{< sentence >}}
分派让人做事，交办让人成长。
{{< /sentence >}}

两者都是"把事给别人做"，区别在于你给出去的是动作指令还是结果目标。一个管理者如果只会分派，团队规模再大也只有一个脑子在运转。

{{< sentence >}}
跟进不是监视，是确认对方没有卡在不该卡的地方。
{{< /sentence >}}

交出去之后不敢问也不敢看的时候读。不跟进不是信任，是放任。跟进的判断标准只有一条：对方需不需要你现在帮忙清除障碍。

{{< sentence >}}
交办之后最难的不是对方做不好，是你忍不住插手。
{{< /sentence >}}

管理者最大的交办障碍通常不在下属那边。你插手一次，下属就学会等。等你来改，等你来定，等你替他做判断。

{{< sentence >}}
不是所有事都值得交出去，但大多数管理者交出去的太少。
{{< /sentence >}}

适合放在季度工作盘点时用。逐条过一遍手上的事，"只有我能做"的比例超过 30% 就值得警惕——大概率不是真的只有你能做，是你没认真想过谁能接。
