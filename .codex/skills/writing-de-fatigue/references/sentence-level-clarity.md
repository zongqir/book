# 通俗易懂：句子级原理与操作手册

## 0. 这份文档要解决什么问题

现有的 writing-de-fatigue skill 按词、句、段三个层面处理阅读疲劳。其中句的层面已经覆盖了修饰前置、的字链、因果压缩、方向变化、重心错位、间接表达和模糊指代。

但用户反馈："还有一种更基础的累。"

那种累发生在单句内部。一个句子 30 个字，没超过 35 字上限，没有"不是…而是…"，没有防御层，用词也不算太书面——但读完就是要在脑子里倒一下才能理解。

这是句子结构本身在消耗认知资源。

本文档提供这些句子级原则的深层认知机制解释、更多改写示例，以及 SKILL.md 未详展的补充原则（用人做主语、顺着思维写、拆四字格墙、被字句转回来）。

## 1. 通俗易懂到底是什么

先说它不是什么：

- 不是用简单词替换专业词（那叫词汇降级）
- 不是把长句拆成短句（那只是限长）
- 不是把深度内容写浅（那叫降质）
- 不是模仿口语（那叫口语化表演）

通俗易懂的真实定义是：**读者的大脑在处理这个句子时，不需要回头、不需要重组、不需要在脑内翻译，一遍就能接住意思。**

它对应的是认知科学里的三个变量：

| 变量 | 含义 | 通俗易懂的要求 |
| --- | --- | --- |
| 工作记忆负载 | 读到句尾时，读者需要同时记住多少个未完成的语义单元 | 尽量少。不超过 3 个 |
| 期望确认延迟 | 读者建立预期后，多久才能得到确认或反转 | 尽量短。不让读者悬着 |
| 表层-深层映射成本 | 读者需要几步才能从字面意思到达作者的实际意思 | 尽量直接。一步最好 |

一句话同时在三个维度都低成本，读者就觉得"好懂"。任何一个维度成本高，读者就会感到"卡了一下"。卡一下没关系，连卡三四次，读者就退出了。

### 通俗 ≠ 浅

通俗是降低表层的穿透成本。浅是降低内容本身的深度。

王小波写"人的一切痛苦，本质上都是对自己无能的愤怒"——意思一点不浅，但表层穿透成本极低。每个词都是日常词，语序就是说话的顺序，一遍就懂。

相反，"人类所经历的各类痛苦体验，其根本性的驱动因素在于对自身能力局限性的深层认知所引发的情绪反应"——意思一样，但穿透成本高了五倍。

通俗易懂追求的是：保持深度，降低穿透成本。

## 2. 为什么中文特别容易写得不通俗

中文有几个结构特性，使得书面语和口语的距离比很多语言都大。这不是中文的缺陷，而是中文的特点——但写作者如果不有意识地管理这些特性，就会不自觉地滑向高认知成本的句子。

### 特性一：修饰语全部前置

中文是强左分支语言。所有修饰成分（定语、状语）都放在被修饰词前面。英语可以用后置定语从句分担压力，中文不行——全部压在前面。

这意味着：修饰链越长，读者等待核心名词的时间越长，工作记忆里积压的未完成语义单元越多。

### 特性二：书面语有独立的一套虚词系统

中文口语说"因为""所以""但是""用""和"。书面语说"由于""因此""然而""使用""以及"。这不只是风格差异——读者在脑内必须做一次翻译才能进入理解。

### 特性三：四字格是压缩包

四字成语和四字短语把一个完整的判断或场景压缩到四个字里。单个使用时效率极高。但连续多个四字格排列时，读者要连续解压多个压缩包，处理速度急剧下降。

### 特性四：名词化是正式感的来源

中文书面语靠名词化来制造"严肃感"。"讨论"变成"进行讨论"，"调整"变成"做出调整"，"分析"变成"开展分析"。每一次名词化都让读者多做一步拆包动作。

### 特性五：话题-评论结构 vs 主谓结构

中文口语天然是话题-评论结构："这件事，我觉得不行。"书面语强行改成主谓结构："我认为此事不可行。"两种都合法，但前者处理成本更低，因为话题先出来，框架立刻建立。

这五个特性叠加在一起，就是为什么中文正式写作特别容易滑向"意思没错，但读着累"。

## 3. 十条句子级操作原则

### 原则一：先出主角，再给修饰

**问题：** 中文把所有定语堆在名词前面。定语链一长，读者在到达核心名词之前，必须把所有修饰内容悬在工作记忆里，不知道它们在描述谁。这叫"花园路径问题"——读者走了很远才发现自己在修饰什么。

**认知机制：** 工作记忆里每多悬挂一个未锚定的修饰语，理解失败的概率就上升约 15%。超过 3 个未锚定修饰语，大多数读者需要回读。

**操作规则：** 名词前最多放 2 层修饰。超过 2 层，把核心名词提到前面，细节用后续分句补。

**改写前：**

```
在当前充满不确定性的国际竞争格局下努力提升自身核心竞争力的中国科技企业正面临着前所未有的挑战。
```

读者要扛过 26 个字的修饰链，才知道主语是"中国科技企业"。

**改写后：**

```
中国科技企业正面临一个大挑战。国际竞争格局充满不确定性，核心竞争力必须跟上。
```

主语第一个词就出现。读者立刻知道在说谁，后面的细节是补充，不是悬念。

**再看一个书评场景的例子：**

改写前：

```
作者在第三章中基于对大量失败创业案例的深入分析后提出的关于团队决策机制的核心判断非常值得注意。
```

改写后：

```
第三章有一个核心判断值得注意：团队决策机制出问题，往往不是能力差，而是分析失败案例时发现的共性模式。
```

### 原则二：砍"的"字链

**问题：** "的"是中文里最高频的结构助词，用来连接修饰语和被修饰语。每个"的"都是一个嵌套边界。连续 3 个"的"以上，读者就要在脑内建多层嵌套结构，处理速度骤降。

**认知机制：** 每个"的"引入一个新的修饰层级。2 层嵌套时，人脑能自动处理；3 层需要有意识地追踪结构；4 层以上，大多数人需要回读。这跟编程中的嵌套缩进一个道理——超过 3 层就该重构。

**操作规则：** 同一句话里连续的"的"不超过 2 个。超过 2 个，把中间的修饰关系拆成独立分句。

**改写前：**

```
这是作者对中国企业的管理实践的深层逻辑的一次系统性的梳理。
```

4 个"的"连击。读者到最后才知道这句话在说"梳理"，而且每个"的"的归属都不直觉。

**改写后：**

```
作者做了一件事：把中国企业管理实践背后的深层逻辑理了一遍。
```

1 个"的"。意思没变，穿透成本降了一半。

**书评场景：**

改写前：

```
本书对于现代组织管理中的领导力发展的理论框架的适用边界的讨论尤为深入。
```

改写后：

```
领导力发展有一套理论框架，但不是哪里都管用。这本书讨论的就是它的适用边界。
```

---

### 原则三：让动词当动词

**问题：** 中文书面语有一个习惯：用"进行 / 做出 / 加以 / 予以 / 给予 / 实施 / 开展"这类空壳动词，把真正的动作变成名词宾语。"讨论"变成"进行讨论"，"分析"变成"开展分析"，"调整"变成"做出调整"。

**认知机制：** 读者看到"进行"时，知道有个动作要来，但不知道是什么——必须等到宾语位置才能知道。这就是信息的延迟交付。而且读者还要做一步解包：从"进行讨论"还原成"讨论"。每一步都是认知税。

**操作规则：** 看到"进行 / 做出 / 加以 / 予以 / 实施 / 开展 + 名词"结构，直接用那个名词的动词形式替换整个结构。

**改写前：**

```
团队需要对当前的业务流程进行全面的梳理，并对发现的问题做出及时的调整。
```

**改写后：**

```
团队得把业务流程全面梳理一遍，发现问题就及时调。
```

从 32 字到 22 字。信息量一样，处理步骤少了两步。

**书评场景：**

改写前：

```
作者对传统阅读方法进行了系统性批判，并对新的阅读框架做出了详细阐述。
```

改写后：

```
作者系统批判了传统阅读方法，然后详细讲了自己的新框架。
```

### 原则四：一句话一个因果

**问题：** 中文允许在一个句子里串联多个因果关系："A 导致 B，引起 C，从而造成 D。"语法上合法，但读者的工作记忆要同时追踪整条因果链的每个环节。

**认知机制：** 因果链是线性依赖的——理解 C 必须先理解 B，理解 B 必须先理解 A。每多一环，工作记忆负载不是线性增加，而是加速增加，因为读者必须同时保持前面所有环节的激活状态。超过 2 个因果环节的单句，理解准确率显著下降。

**操作规则：** 一句话里最多一个因果连接。因果链超过 2 环，拆成多句，每句处理一个因果环节。

**改写前：**

```
由于市场竞争加剧导致利润率持续下降，使得企业被迫压缩研发预算，从而严重影响了产品创新能力的提升。
```

4 环因果链在一句话里：竞争加剧 → 利润下降 → 研发缩减 → 创新受损。

**改写后：**

```
市场竞争加剧，利润率一直在掉。研发预算被压缩，产品创新跟不上了。
```

同样的因果链，分两句各处理两环。每句的负担都在可控范围内。

**书评场景：**

改写前：

```
正是因为作者坚持从实践出发收集案例，使其论证建立在坚实的经验基础上，从而使整套方法论获得了超越纯理论推演的说服力。
```

改写后：

```
作者的案例全从实践里来，论证就有了经验基础。这比纯靠理论推演要有说服力得多。
```

### 原则五：用人做主语

**问题：** 书面中文偏爱抽象名词做主语。"技术进步推动了""制度创新促进了""方法论的演进导致了"——主语是一个没有面孔的概念，读者无法在脑内构建画面。

**认知机制：** 人脑处理有生命主体（人、动物、团队）的速度显著快于处理抽象概念主体。这叫"生命性优势"（animacy advantage）。有生命主体做主语时，读者自动启动动作模拟（action simulation），理解速度更快，记忆留存更好。

**操作规则：** 优先用人（或具体的组织、团队）做主语。如果原句主语是抽象概念，看能不能找到背后的人。找不到具体的人时，至少用一个读者能想象出画面的主语。

**改写前：**

```
经济体制改革的持续深化推动了基层社会治理模式的全面创新与发展。
```

主语是"经济体制改革的持续深化"——没有人，没有画面。

**改写后：**

```
经济体制一直在改，基层管理社会的方式也跟着变了。
```

虽然还是没有明确的"人"，但"经济体制"和"基层"变成了读者能感知的具体对象，而不是嵌套的抽象名词短语。

**书评场景：**

改写前：

```
领导力理论的范式转型为组织变革的实践路径提供了新的分析框架。
```

改写后：

```
领导力理论变了一轮以后，组织搞变革终于有了新的分析思路。
```

### 原则六：顺着思维的语序写

**问题：** 中文口语天然是"话题-评论"结构：先说"关于什么"，再说"怎么样"。书面语经常强行改成主谓宾结构，或把条件状语前置太多层。两种都合法，但话题-评论结构的处理成本更低。

**认知机制：** 话题-评论结构之所以处理快，是因为话题先出场，立刻建立了语义框架。读者知道"我们在聊什么"，后续内容就有了着陆点。而主谓宾结构里，如果主语是一个长修饰链（见原则一），读者在建立框架之前就已经被灌了一堆信息。

另一个维度是信息流：人的思维先处理已知信息（旧信息），再处理新信息。话题通常是旧信息，评论是新信息。这个顺序符合认知的自然流向。

**操作规则：** 当句子主语很长或很抽象时，看看能不能把它改成话题-评论结构——先用短话题锚定框架，再给评论。

**改写前：**

```
尽管在实际操作过程中团队可能会面临来自多个层面的阻力，但通过循序渐进地推进和及时的方案调整，最终达成预期目标是完全可以实现的。
```

条件从句太长，读者到主句时已经忘了前面在限定什么。

**改写后：**

```
阻力肯定有，会从好几个方向来。但一步步推，边做边调，目标能达成。
```

先抛话题（阻力），再给评论（能达成）。框架立刻建立。

**书评场景：**

改写前：

```
虽然该书在理论深度方面具有显著优势，但对于缺乏相关领域专业背景的普通读者而言，部分章节的论述可能存在一定的理解门槛。
```

改写后：

```
理论深度是这本书的强项。但没有专业背景的读者，碰到有些章节会觉得吃力。
```

### 原则七：拆四字格墙

**问题：** 四字格（成语、四字短语）是中文的压缩包——四个字里装一个完整的意象或判断。一个四字格，精炼有力；三四个连排，就变成一堵墙。读者要连续解压多个压缩包，认知带宽很快耗尽。

**认知机制：** 每个四字格要求读者做两件事——识别这个固定表达，然后解压它的含义。单个四字格的解压成本很低（熟悉的成语几乎是自动处理的）。但连续多个时，解压任务排队，处理速度下降，而且四字格之间的逻辑关系（并列？递进？对比？）经常是隐含的，读者还要猜连接方式。

**操作规则：** 同一句话里四字格不超过 2 个。超过 2 个时，拆开部分四字格，用日常话把意思说出来。保留最精准或最有画面感的那个，其余展开。

**改写前：**

```
管理者应当与时俱进、居安思危、未雨绸缪、防微杜渐，才能确保组织的可持续发展。
```

4 个四字格连排。它们的意思有重叠（"居安思危"和"未雨绸缪"差不多），读者处理完发现信息增量很小。

**改写后：**

```
管理者得跟上变化，日子好的时候想想坏日子怎么来。小问题刚露头就处理，别等它长大。
```

意思一样，但每个词都不需要解压。

**书评场景：**

改写前：**

```
作者深入浅出、旁征博引、条分缕析、鞭辟入里地剖析了现代管理的核心困境。
```

改写后：

```
作者分析现代管理的核心困境，讲得透，例子多，条理清楚。
```

### 原则八：被字句转回来

**问题：** 中文的被动句（"被"字句）是有标记的（marked construction），处理成本比主动句高。而且"被"字句在中文传统语感里带贬义色彩（被打、被骗、被开除），用在中性或正面语境里会产生微妙的语义冲突。

**认知机制：** 主动句的认知流程是"谁 → 做了什么 → 对谁"，顺着行动方向走。被动句是"谁 → 被 → 做了什么"，读者先遇到承受者，再反向追溯施动者。这个反向追溯是额外的处理步骤。

另外，中文的"被"还会和"使得""令""导致"组成被动链："A 被 B 所导致的 C 使得 D 受到影响"——读者要同时处理被动和因果，双重负担。

**操作规则：** 默认用主动句。只在两种情况下保留被动：（1）施动者确实不知道或不重要，（2）承受者才是话题焦点。

**改写前：**

```
这一观点已被众多学者广泛引用，并被视为该领域最具影响力的理论贡献之一。
```

**改写后：**

```
很多学者引用了这个观点，不少人把它看作这个领域最有影响力的理论贡献之一。
```

**书评场景：**

改写前：

```
读者极易被作者流畅的叙事节奏所吸引，以至于核心论证的薄弱之处容易被忽略。
```

改写后：

```
作者叙事节奏太流畅，读者容易跟着走，反而忽略了核心论证的薄弱处。
```

### 原则九：重心别埋在从句里

**问题：** 一句话的信息重心应该在主句，不在从句。但书面中文经常把最重要的判断藏在"在…的情况下""通过…的方式""基于…的考量"这类框架性从句里，主句反而是一个空洞的包装。

**认知机制：** 这叫"重心错位"。读者的注意力分配策略是：主句 > 从句。如果核心信息在从句里，读者要么跳过了它（因为从句被当作背景处理），要么理解了但印象很浅（因为从句在注意力分配中权重低）。

**操作规则：** 写完一句话后，问自己：这句话最重要的意思在主干里还是在从句里？如果在从句里，把它翻到主干来。

**改写前：**

```

```

核心判断是"这个方案最可操作"，但它被埋在一个"被认为是"的被动结构里，前面还顶了一个条件从句。

**改写后：**

```
这个方案最能落地。它是在盘清了现有资源和能力边界之后选出来的。
```

核心判断提到第一句。条件变成第二句的补充说明。

**书评场景：**

### 原则十：一句话只过一个弯

**问题：** 有些句子语法上没毛病，长度也没超标，但读完需要在脑子里"转"一下才能理解。原因通常是句子里同时包含了两个方向的信息——一个转折、一个因果、一个条件，或者把"其实不是 X"和"而是 Y"压在同一句里。

**认知机制：** 理解一个句子时，读者会建立一个心理模型（mental model）。每个转折、否定或条件切换都要求修改这个模型。修改一次，处理成本可控。一句话里修改两次以上，读者的模型就不稳了——他不确定最终的模型应该是什么形状。

这跟反向建构（"不是 A 而是 B"）不完全一样。反向建构是一种特定的修辞模式。"一句话只过一个弯"是更底层的原则——它管的是句内方向变化的总次数。

**操作规则：** 一句话里最多一次方向变化（一个"但""却""虽然""不过"或隐含的转折）。如果一句话里有两次方向变化，拆成两句，每句处理一个弯。

**改写前：**

```
虽然作者的论证在大方向上基本成立，但部分关键环节的证据其实并不充分，不过这并不影响整体框架的参考价值。
```

三个方向变化：肯定 → 否定 → 再肯定。读者的心理模型被翻了三次。

**改写后：**

```
论证的大方向基本成立，整体框架可以参考。但有几个关键环节的证据不太够。
```

先把结论给清楚（成立、可以参考），再单独说不足。读者的模型只修改一次。

## 4. 三条判断标准的操作化

用户提出的三条核心标准，和上面十条原则的映射关系：

### 不要过于书面化的词语

这一条现有 skill 已经覆盖了词汇层面（使用→用，由于→因为）。但"书面化"不只是词汇问题，还有结构问题：

- 名词化结构本身就是书面化的（原则三）
- 被动句本身就是书面化的（原则八）
- 长定语链本身就是书面化的（原则一、二）
- 四字格密排本身就是书面化的（原则七）

**词汇降级做完还觉得书面味重，往往是这四个结构没处理。**

### 好读

好读的核心是：读者的处理资源不被浪费在"理解结构"上，而是全部用在"理解内容"上。

- 修饰语前置太多 → 资源浪费在等主语（原则一、二）
- 因果链压缩 → 资源浪费在追踪依赖（原则四）
- 方向变化过多 → 资源浪费在修改心理模型（原则十）
- 重心埋在从句 → 资源浪费在定位重点（原则九）

### 直白不晦涩

晦涩的来源不只是用了难词。更常见的是：

- 抽象主语让读者不知道在说谁（原则五）
- 语序不符合思维的自然流向（原则六）
- 一句话同时说两三件事，读者不确定重点是哪个（原则十）
- 核心判断藏在从句里，读者跳过了也不知道（原则九）

**直白 = 读者不需要在脑内重新排列信息就能抓到重点。**

## 5. 通俗易懂不是什么

### 不是口语化表演

把"在这个框架下"改成"说白了就是"，把"值得注意的是"改成"你猜怎么着"——这不是通俗，是在模仿一种语气。通俗是降低结构的处理成本，不是把书面语cosplay成脱口秀。

### 不是把所有东西说两遍

"也就是说""换句话说""简单来说"——这些不是通俗的标志，是作者不确定第一遍说清楚了没有的标志。如果第一遍就说清了，不需要第二遍。如果第一遍没说清，改第一遍，不要加第二遍。

### 不是降低信息密度

通俗不是把一个段落拉长到三个段落。信息密度可以高，只要每个信息单元的处理成本低。王小波的信息密度不低，但每句话的穿透成本极低。

### 不是避开专业概念

专业概念该用就用。通俗的要求是：首次出现时，上下文足以让它自解释。读者不需要查词典，但也不需要你写一段定义。

### 不是永远用短句

短句不等于好读。"他来了。他看了。他走了。"——全是短句，但读起来像打字机。通俗要求的是每句话处理成本低，不是每句话字数少。长句只要结构清晰（主语早出、一个因果、一个方向），也可以很好读。

### 不是没有节奏

通俗易懂的文章有节奏。长短交替，紧松交替，判断和场景交替。没有节奏的平铺直叙，读起来不累但也不想读——因为无聊。
